# Sigma (Σ) - 自主工作流

## 自主循环机制

> 核心理念：汇聚各方经验，持续生产可复用的 Skills 和 Agents
> 定位：AI Agent 开发框架，给其他项目提供 Skills/Agents/Workflows

---

## 一、每次接收到任务的循环流程

```
┌─────────────────────────────────────────────────────────────┐
│  接收任务                                                    │
│  输入：新技能需求 / 优化现有技能 / 扩展 Agents                │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  步骤1: 理解任务                                             │
│  - 读 ARCHITECTURE.md 理解当前 Skills/Agents 库            │
│  - 问自己：这个技能是给哪个项目用的？什么场景？                │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  步骤2: 规划设计                                            │
│  - 设计 Skill 的触发条件、输入输出、Prompt                   │
│  - 参考现有 Skills 的格式和结构                              │
│  - 更新 ARCHITECTURE.md（新增 Skill/Agent）                   │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  步骤3: GitHub/社区研究（如果需要）                        │
│  - 去 _reference/ 研究参考实现                               │
│  - 搜索 GitHub / HackerNews 找灵感和最佳实践                 │
│  - 记录学到的东西到 _LEARNING.md                            │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  步骤4: 实现代码                                            │
│  - 按 Sigma Skills 规范实现                                  │
│  - 遵循 ARCHITECTURE.md 的格式写注释                        │
│  - 确保 Skill 可被其他项目直接引用                           │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  步骤5: 测试验证                                            │
│  - 运行 npm run build / python -m pytest                   │
│  - 功能正常运行？                                            │
│  - Skill 格式是否符合规范？                                 │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  步骤6: 更新文档                                            │
│  - 更新 ARCHITECTURE.md（新增模块）                          │
│  - 更新 _LEARNING.md（记录学到的）                           │
│  - 更新 PROGRESS.md（记录本轮做了什么）                      │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  步骤7: 自我审计                                            │
│  - Skill 质量：是否符合规范？可复用？                        │
│  - 文档完整：SKILL.md、ARCHITECTURE 都更新了？               │
│  - 是否需要通知相关项目来使用新 Skill？                       │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  步骤8: 版本管理                                            │
│  - git add . && git commit -m "描述本次完成的工作"        │
│  - git push 推送到 GitHub 仓库                            │
│  - 确保所有重要更改都已提交和推送                          │
└─────────────────────────────────────────────────────────────┘
                        ↓
                        ↓ 循环结束
```

## 二、定期自主循环（无新任务时）

当没有新任务时，每轮自动执行：

```
┌─────────────────────────────────────────────────────────────┐
│  定期循环 (每轮自动执行)                                    │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  1. GitHub 趋势扫描                                        │
│  - 搜索 AI agent framework / LangGraph skills 相关项目      │
│  - 找有没有新的 Agent 设计模式或 Skill 范式                  │
│  - 有就 clone 到 _reference/                               │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  2. 现有 Skills 审视                                       │
│  - 检查有没有可以抽象复用的模式                              │
│  - 检查现有 Skills 是否有改进空间                             │
│  - 发现好的模式，更新到对应 Skill 或创建新 Skill             │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  3. 其他项目需求收集                                       │
│  - 看其他项目的 PROGRESS.md，了解他们需要什么                │
│  - 主动思考哪些 Skill 可以被他们复用                        │
│  - 如果有明确需求，优先实现                                 │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  4. 框架优化                                                │
│  - 检查 Skills 规范是否需要更新                              │
│  - 优化 Skill 触发机制                                      │
│  - 改进文档模板                                             │
└─────────────────────────────────────────────────────────────┘
```

## 四、动态跟踪各项目需求

**注意**: 其他项目会动态提出 Skill 需求，需要持续跟踪。

### 各项目定位
```
D:/PM-AI-Workstation/01-ai-agents/
├── pm-assistant/              # Pi - PM助理 → 需要 PM 相关 Skills
├── private-operation-agent/    # Nu - 私域运营 → 需要 运营/营销 Skills
├── investment-research-agent/  # Rho - 投研 → 需要 金融/分析 Skills
├── podcast-research-agent/    # Echo - 播客研究 → 需要 数据/教育 Skills
├── content-gen-agent/        # Vox - 内容生成 → 需要 营销/内容 Skills
└── vertical-browser-agent/    # Spider - 垂直浏览器 → 需要 自动化 Skills
```

### 需求发现机制
- 每轮定期检查其他项目的 WORKFLOW.md 或 PROGRESS.md
- 如果发现某项目在自主循环中需要某 Skill，主动为他们创建
- 优先创建多个项目可能复用的通用 Skill

### Skill 被使用后
- 记录是哪个项目在使用
- 根据使用反馈优化 Skill

## 五、Skills 规范

### Skill 目录结构
```
skills/
├── [domain]/
│   ├── [skill-name]/
│   │   ├── SKILL.md           # Skill 定义
│   │   ├── src/
│   │   │   └── index.ts      # Skill 实现
│   │   └── tests/
│   │       └── skill.test.ts
```

### SKILL.md 格式
```markdown
# [Skill Name]

## 触发条件
- 什么场景下调用这个 Skill

## 输入
- 输入参数及格式

## 输出
- 输出结果及格式

## Prompt 模板
```
[Prompt 内容]
```

## 示例
### 示例1
- 输入：xxx
- 输出：xxx

## 注意事项
- 关键的注意点
```

## 七、文档更新规范

### ARCHITECTURE.md 更新
新增 Skill/Agent 时：
```markdown
## [skill-name] Skill

### 类型
- `InputType` - 输入类型
- `OutputType` - 输出类型

### 函数
- `execute(input: InputType) -> OutputType`

### 所在目录
- `skills/[domain]/[skill-name]/`

### 被使用项目
- (待补充)
```

### _LEARNING.md 更新
学到新东西时：
```markdown
## 第N轮学习

### GitHub 新发现
- [项目名] - 简介 - 可用于：xxx

### Skill 改进
- 改进了 xxx Skill - 原因：xxx

### 框架优化
- 优化了 xxx 机制 - 原因：xxx
```

### PROGRESS.md 更新
每轮结束时：
```markdown
## 第N轮

### 完成
- [新增 Skill1]
- [优化 Skill2]

### 被使用
- [项目名] 使用了 [Skill名]

### 学习
- [学到的1]

### 下轮计划
- [计划1]
```

## 八、循环终止条件

可以终止的情况：
1. 用户说"满意了"或"可以了"
2. 循环 20 次后自动暂停（Sigma 迭代较慢）
3. 遇到无法解决的技术难题，记录并暂停

---

## 九、与其他项目的协作

### 核心原则
- Sigma 负责**生产** Skills/Agents
- 其他项目负责**使用** Skills/Agents
- 发现通用功能 → 抽取到 Sigma → 各项目复用

### 通知机制
当 Sigma 新增或更新 Skill 时：
1. 在 PROGRESS.md 中记录
2. 其他项目在自主循环中会发现并使用
