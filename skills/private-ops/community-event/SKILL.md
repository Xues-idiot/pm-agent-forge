# Community Event Skill

私域社群活动策划与执行，提升用户活跃度和粘性。

## 何时使用

- 社群活动策划
- 用户活跃度提升
- 线上/线下活动组织
- 节日活动设计
- 社群氛围营造

## 活动策划流程

```
1. 目标设定 → 2. 活动设计 → 3. 预热引流 → 4. 执行监控 → 5. 复盘优化
```

### 1. 目标设定

**活动目标矩阵：**

```markdown
## 活动目标

### 核心目标
| 目标 | 指标 | 定义 |
|------|------|------|
| 拉新 | 新增用户 | 活动带来新成员 |
| 促活 | 活跃率 | 参与用户/总用户 |
| 转化 | 付费率 | 付费用户/参与用户 |
| 留存 | 回访率 | 次日/7日回访 |

### 目标设定SMART
```markdown
## 目标模板

具体：[活动主题]带来[具体结果]
可衡量：[X]人参与，[Y]%转化
可达成：基于历史数据合理设定
相关性：支撑[季度/月度]目标
时间：[活动周期]

示例：
"618活动带来500新用户参与，转化率15%，高于平时50%"
```

### 3. 预热引流

**预热策略：**

```markdown
## 预热引流

### 预热节奏
| 时间 | 动作 | 内容 |
|------|------|------|
| 活动前7天 | 悬念发布 | 活动预告，引发好奇 |
| 活动前3天 | 详细说明 | 规则介绍，预约参与 |
| 活动前1天 | 再次提醒 | 最后提醒，刺激参与 |
| 活动当天 | 高潮推广 | 集中曝光，引导参与 |

### 预热话术
```markdown
## 预热话术

### 悬念期

```markdown
## 应急预案

### 场景1：参与量超预期
- 问题：服务器压力
- 对策：临时扩容 + 排队机制

- 问题：无法领取
- 对策：人工处理 + 补偿方案

## 社群活动检查清单

- [ ] 目标清晰可衡量
- [ ] 规则简洁易懂
- [ ] 奖励有吸引力
- [ ] 预热充分到位

- [ ] 经验沉淀归档

## 红牌警告

- 虚假宣传欺骗用户
- 奖励承诺不兑现
- 活动规则不透明
- 奖池被内部薅羊毛
- 用户隐私泄露
- 过度打扰用户
- 活动效果数据注水
- 不复盘不改进
