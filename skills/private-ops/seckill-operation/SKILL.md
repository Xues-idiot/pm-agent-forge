# Flash Sale / Seckill Operation

秒杀运营，设计稀缺营销和秒杀活动。

## 何时使用

- 策划秒杀活动
- 设计限时抢购
- 制定稀缺营销策略
- 处理秒杀峰值
- 分析秒杀效果

## 秒杀运营流程

### 前期准备

```markdown

| 时间 | 动作 | 内容 |

### 活动执行

```markdown
## 秒杀执行

### 开抢前
- [ ] 库存提前置入
- [ ] 服务器扩容
- [ ] 客服准备就绪

### 开抢中
| 监控项 | 指标 | 阈值 |
|--------|------|------|
| 下单成功率 | >95% | 低于90%预警 |
| 响应时间 | <200ms | 超过500ms预警 |
| 订单量 | QPS监控 | 异常峰值预警 |

### 异常处理
| 问题 | 处理 |
|------|------|
| 超卖 | 立即停售、道歉、补偿 |
| 系统慢 | 限流、降级、扩容 |
| 恶意下单 | 风控拦截、人工处理 |
```

## 红牌警告

- 超卖导致无法发货
- 系统扛不住峰值崩溃
- 价格太低亏太多
- 虚假秒杀欺骗用户
- 没有预热流量不足

## 好/坏对比

**好**：
```
秒杀准备：
- 选引流爆款
- 价格有噱头
- 预热7天
- 技术扩容
- 客服就绪
→ 售罄率高、用户体验好
```

**坏**：
```
秒杀准备：
- 选品不热门
- 价格没诚意
- 没预热
- 系统崩溃
→ 流量来了接不住
```
