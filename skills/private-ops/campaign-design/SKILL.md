# Private Ops Campaign Design Skill

设计有效的私域营销活动，提升用户参与度和转化率。

## 何时使用

- 新用户激活活动
- 老用户召回活动
- 大促活动策划（双十一、周年庆）
- 节日营销活动
- 用户分层运营活动

## 活动设计流程

```
1. 明确活动目标 → 2. 分析目标用户 → 3. 设计活动机制 → 4. 制定触达计划 → 5. 预估效果 → 6. 复盘优化
```

### 1. 明确活动目标

**活动目标类型：**

| 目标类型 | 具体指标 | 适用场景 |
|---------|---------|---------|
| 拉新 | 新增用户数、新用户转化率 | 拓展用户池 |
| 促活 | DAU/MAU、互动率 | 提升用户粘性 |
| 转化 | GMV、付费率、客单价 | 提升变现 |
| 召回 | 召回率、召回用户留存 | 挽回流失用户 |

**SMART 目标设定：**
```markdown
## 活动目标示例

❌ 不好：提升用户活跃度
✓ 更好：30天内 DAU 从 1000 提升到 1500（+50%）

❌ 不好：增加收入
✓ 更好：活动期间 GMV 达到 100 万，转化率提升 20%
```

### 2. 分析目标用户

**用户分层基础：**

| 分层维度 | 分层标准 | 运营策略 |
|---------|---------|---------|
| RFM | 最近消费、频率、金额 | 差异化触达 |
| 用户生命周期 | 新用户/活跃/沉睡/流失 | 针对性策略 |
| 用户价值 | 高价值/中价值/低价值 | 资源倾斜 |
| 行为偏好 | 购买型/浏览型/互动型 | 内容匹配 |

### 触达偏好
- 偏好渠道：

### 3. 设计活动机制

**活动机制类型：**

| 机制类型 | 原理 | 适用目标 |
|---------|------|---------|
| 优惠券/红包 | 短期利益刺激 | 转化、召回 |
| 任务体系 | 完成行为获得奖励 | 促活、拉新 |
| 竞赛/排名 | 竞争心理驱动 | 促活、UGC |
| 会员专属 | 特权感驱动 | 留存、转化 |
| 邀请有礼 | 社交裂变 | 拉新 |
| 签到打卡 | 习惯养成 | 促活 |

**活动机制设计模板：**
```markdown
## 活动机制设计

### 活动名称：[名称]
### 活动主题：[主题文案]
### 活动时间：[开始-结束]

### 核心机制
[详细描述活动规则]

### 参与流程
1. 用户进入活动页
2. [行为A] → 获得[奖励A]
3. [行为B] → 获得[奖励B]
4. 完成全部任务 → 获得[最终奖励]

### 奖励体系
| 行为 | 奖励 | 成本 |
|------|------|------|
| 每日签到 | 积分X | ¥0.1 |
| 分享邀请 | 优惠券¥10 | ¥10 |
| 单笔消费满100 | 返现5% | ¥5 |

### 防刷策略
- [策略1]
- [策略2]
```

### 4. 制定触达计划

**触达渠道选择：**

| 渠道 | 触达率 | 打开率 | 适用场景 |
|------|-------|-------|---------|
| 公众号 | 中 | 低 | 日常触达、内容型 |
| 小程序弹窗 | 高 | 中 | 活跃用户 |
| 短信 | 中 | 低 | 沉睡用户、正式通知 |
| 朋友圈广告 | 高 | 中 | 拉新、召回 |
| 社群 | 高 | 高 | 活跃用户互动 |
| 私聊 | 最高 | 中 | 重要通知、高价值用户 |

**触达时间策略：**

| 用户类型 | 最佳时间 | 频率建议 |
|---------|---------|---------|
| 上班族 | 12:00-13:00, 20:00-22:00 | 每周2-3次 |
| 学生 | 18:00-21:00 | 每周3-4次 |
| 自由职业 | 10:00-12:00 | 每周1-2次 |

**触达计划模板：**
```markdown
## 触达计划

### 预热期（活动前3天）
- Day 1：社群预告，神秘活动
- Day 2：公众号推送预告
- Day 3：私聊高价值用户预热

### 爆发期（活动期间）
- 每日固定时间提醒
- 实时播报活动进度
- 限时加倍奖励刺激

### 收尾期（活动最后1-2天）
- 倒计时提醒
- 未完成用户定向触达
```

## 活动设计检查清单

- [ ] 活动目标明确且可衡量
- [ ] 目标用户清晰
- [ ] 活动机制简单易懂
- [ ] 奖励有吸引力且成本可控
- [ ] 触达计划覆盖目标用户
- [ ] 防刷机制到位
- [ ] 数据埋点完整
- [ ] 客服话术准备

## 红牌警告

- 目标贪多，什么都想提升
- 活动机制过于复杂
- 奖励成本超预算
- 忽略防刷导致被薅羊毛
- 没有数据埋点无法复盘
- 触达频率过高打扰用户
- 虚假宣传夸大奖励

## 活动文案示例

**好文案：**
```
【限时3天】邀请1位好友得¥50无门槛券，多邀多得！
你邀请的好友下单后立即到账，可叠加使用。
还剩X个名额 →
```

**坏文案：**
```
好消息！活动开始了！快来参加！很多优惠！
```
