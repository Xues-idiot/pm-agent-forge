# Customer Relationship

客户关系管理，提升客户满意度和忠诚度。

## 何时使用

- 客户反馈管理
- 投诉处理
- 客户满意度
- 关系维护
- 忠诚度提升

## 反馈管理

### 反馈渠道

```markdown
## 反馈渠道

### 主动反馈
| 渠道 | 特点 | 响应时效 |
|------|------|----------|
| 客服 | 即时沟通 | 实时 |
| 电话 | 直接沟通 | 即时 |
| 邮件 | 正式沟通 | 24h |

| 渠道 | 特点 |
|------|------|
| 评价 | 公开可见 |

### 反馈处理

```markdown
## 反馈处理

### 处理流程
1. 接收反馈 → 2. 分类评估 → 3. 分发处理 → 4. 回复客户 → 5. 归档跟进

### 处理时效
| 类型 | 响应时间 | 解决时间 |
|------|----------|----------|
| 紧急 | 1小时 | 24小时 |
| 重要 | 4小时 | 48小时 |
| 一般 | 24小时 | 7天 |

### 回复模板
```markdown
## 回复模板

### 致歉+解决
"非常抱歉给您带来不便，我们已采取XX措施解决您的问题。如有其他问题请随时联系。"

### 感谢+跟进
"感谢您的反馈，我们非常重视您的问题，已转交相关部门处理，预计XX时间给您回复。"

## 满意度管理

### 满意度指标

```markdown
## 满意度指标

### 常用指标
| 指标 | 计算 | 目标 |
|------|------|------|
| CSAT | 满意/总数 | >90% |
| NPS | 推荐-贬损 | >50 |
| CES | 费力程度 | <5分 |
| 投诉率 | 投诉/总数 | <1% |

### NPS计算
```markdown
## NPS计算

### 评分标准
- 9-10分：推荐者
- 7-8分：被动者
- 0-6分：贬损者

### 计算公式
NPS = 推荐者% - 贬损者%
```
```

### 满意度调查

```markdown
## 满意度调查

### 调查类型

## 投诉处理

### 投诉分类

```markdown
## 投诉分类

### 按类型
| 类型 | 说明 | 处理 |
|------|------|------|
| 产品问题 | 质量、功能 | 退换/赔偿 |
| 服务问题 | 态度、响应 | 道歉/改进 |
| 物流问题 | 延迟、破损 | 赔偿/补偿 |
| 虚假宣传 | 承诺不符 | 澄清/赔偿 |

### 按严重度
| 级别 | 定义 | 处理 |
|------|------|------|
| P0 | 大规模/媒体 | 紧急处理+公关 |
| P1 | 高价值客户 | 优先处理 |
| P2 | 普通投诉 | 正常处理 |
| P3 | 一般反馈 | 记录优化 |
```
```

### 处理原则

```markdown
## 处理原则

### 处理态度
- 认真倾听
- 不争辩
- 同理心
- 真诚道歉

### 解决标准
```markdown
## 解决标准

### 解决标准
| 级别 | 解决时间 | 解决标准 |
|------|----------|----------|
| P0 | 4小时 | 问题解决+补偿 |
| P1 | 24小时 | 问题解决+补偿 |
| P2 | 48小时 | 问题解决 |
| P3 | 7天 | 解释+改进 |

### 补偿标准
- 轻微：积分/优惠券
- 中度：退款/赔偿
- 严重：退款+赔偿+额外补偿

## 红牌警告

- 反馈不处理
- 投诉升级
- 服务态度差
- 过度打扰
- 无差异化维护
- 不做流失预警

## 好/坏对比

**好**：
```
客户关系：
- 分层清晰
- 反馈及时
- 投诉妥善
- 满意度高
- 关系维护好
→ 忠诚度高、口碑好
```

**坏**：
```
客户关系：
- 分层模糊
- 反馈石沉大海
- 投诉不处理
- 满意度低
- 不做维护
→ 流失高、负口碑
```
