## 社群危机

### 危机处理

```markdown
## 危机处理

### 危机类型
| 类型 | 表现 | 紧急程度 |
|------|------|----------|
| 广告刷屏 | 多人发广告 | 高 |
| 负面舆情 | 成员负面情绪 | 高 |
| 竞争对手 | 竞品潜入 | 中 |
| 成员纠纷 | 成员间冲突 | 中 |
| 系统故障 | 机器人失效 | 低 |

### 处理流程
```markdown
## 危机处理

1. 发现：及时发现异常
2. 评估：判断影响程度
3. 处理：执行应对方案
4. 跟进：监控后续发展
5. 总结：复盘改进
```

### 应急预案
```markdown
## 应急预案

### 广告刷屏
- 立即移除广告者
- 开启入群验证
- 加强巡逻

### 负面舆情
- 先安抚情绪
- 私聊了解情况
- 公开回应解决

### 成员纠纷
- 分别私聊沟通
- 必要时移除一方
- 事后复盘规则
```

## 红牌警告

- 没有价值输出
- 变成广告群
- 管理缺位
- 过度商业化
- 忽视成员反馈

## 好/坏对比

**好**：
```
社群运营：
- 定位清晰：垂直人群
- 规则完善：违规必究
- 内容有价值：干货+互动
- 激励到位：荣誉+物质
- 商业化适度：先信任后卖货
→ 高活跃、高变现
```

**坏**：
```
社群运营：
- 没有定位：什么人都要
- 没有规则：广告刷屏
- 没有内容：整天就是卖货
- 没有激励：成员自生自灭
- 过度商业化：一上来就卖
→ 死群、退群率高
```
