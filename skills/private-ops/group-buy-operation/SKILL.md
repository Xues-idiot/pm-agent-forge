# Group Buy Operation

拼团运营，设计邀请购买和社交裂变。

## 何时使用

- 策划拼团活动
- 设计邀请机制
- 运营社群团购
- 分析拼团效果
- 优化拼团转化

## 拼团模式

### 拼团类型

```markdown
## 拼团类型

### 按参与方式
| 类型 | 说明 | 示例 |
|------|------|------|
| 自主开团 | 用户主动发起 | 电商拼团 |
| 强制拼团 | 必须凑人数 | 团购网站 |
| 阶梯拼团 | 人数越多越便宜 | 阶梯定价 |

### 按社交范围
| 类型 | 说明 | 特点 |
|------|------|------|
| 私域拼团 | 社群/企微 | 转化高、可控 |
| 公域拼团 | 朋友圈/好友 | 曝光大、难转化 |

### 拼团商品
```markdown
## 拼团商品选择

| 类型 | 特点 | 适合场景 |
|------|------|----------|
| 高频刚需 | 消耗快、决策低 | 食品、日用品 |
| 高性价比 | 颜值高、价格低 | 引流拉新 |
| 社交话题 | 有噱头、可分享 | 传播裂变 |
| 服务体验 | 线下服务 | 教育、健身 |
```

## 拼团流程

### 标准拼团流程

```markdown
## 拼团流程

用户A发起拼团
    ↓
分享给好友/社群
    ↓
好友B/C/D参与拼团
    ↓
凑齐人数
    ↓
拼团成功
    ↓
发货/核销
```

### 拼团时间设置

```markdown
## 拼团时间

### 开团有效期
| 类型 | 时长 | 适用 |
|------|------|------|
| 短时拼团 | 2-4小时 | 紧迫感强 |
| 中时拼团 | 12-24小时 | 常规 |
| 长时拼团 | 3-7天 | 高决策商品 |

### 成团期限
- 建议：24-48小时
- 太短：凑不够人
- 太长：失去紧迫感
```

## 拼团机制设计

### 定价策略

```markdown
## 拼团定价

### 定价原则
- 单买价 > 拼团价 > 成本价
- 拼团价 = 成本 + 合理利润
- 单买价与拼团价差 ≥ 10%

### 定价示例
| 商品 | 单买价 | 拼团价(2人) | 折扣 |
|------|--------|-------------|------|
| 商品A | ¥99 | ¥79 | 8折 |
| 商品B | ¥199 | ¥149 | 75折 |
| 商品C | ¥399 | ¥299 | 75折 |

### 阶梯拼团
| 成团人数 | 2人 | 3人 | 5人 |
|----------|-----|-----|-----|
| 价格 | ¥79 | ¥69 | ¥59 |
| 折扣 | 8折 | 7折 | 6折 |
```

### 成团机制

```markdown
## 成团机制

### 满人即开
- 凑齐人数立即成团
- 最常用，效率高

### 定时开奖
- 无论是否满人，到时开奖
- 适合稀缺商品

### 拼团失败
| 处理方式 | 说明 | 适用 |
|----------|------|------|
| 原路退款 | 24h内退款 | 常规 |
| 转为购买 | 可选单买价购买 | 减少流失 |
| 补偿券 | 发放优惠券 | 用户关怀 |
```

## 检查清单

- [ ] 选品适合拼团
- [ ] 定价有吸引力
- [ ] 时间设置合理
- [ ] 分享素材精美
- [ ] 社群运营到位
- [ ] 数据追踪完善

## 红牌警告

- 虚假拼团刷单
- 价格低于成本
- 货不对板
- 拼团不发货
- 打扰用户过频繁

## 好/坏对比

**好**：
```
拼团设计：
- 选品刚需高频
- 定价合理(8折)
- 海报精美有噱头
- 社群运营精细
→ 高成团率、高复购
```

**坏**：
```
拼团设计：
- 选品冷门
- 价格太低亏本
- 海报土气
- 发完不运营
→ 开团多成团少
```
