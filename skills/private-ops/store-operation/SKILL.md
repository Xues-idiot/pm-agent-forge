# Store Operation

门店运营，管理零售运营和设计线上线下融合。

## 何时使用

- 门店日常运营
- 库存管理
- 店员培训
- OMO融合策略
- 门店数据化

## 库存管理

### 库存策略

```markdown
## 库存策略

### 库存分类
| 类型 | 策略 | 补货周期 |
|------|------|----------|
| A类爆款 | 高库存 | 1周 |
| B类主力 | 中库存 | 2周 |
| C类长尾 | 低库存 | 1月 |

### 补货流程
```markdown
## 补货流程

1. 每日盘点：检查A类、B类库存
2. 生成订单：系统自动计算
3. 审核确认：店长审批
4. 总部配送：统一配送
5. 验收入库：门店验收
```

## 门店数据

### 核心指标

```markdown
## 门店KPI

### 销售指标
| 指标 | 计算 | 目标 |
|------|------|------|
| 销售额 | Σ商品×数量 | ¥X万/月 |
| 客单价 | 销售额/客数 | ¥X |
| 客流 | 进店人数 | X人/天 |
| 转化率 | 成交/进店 | X% |

### 效率指标
| 指标 | 计算 | 目标 |
|------|------|------|
| 人效 | 销售额/人数 | ¥X/人 |
| 坪效 | 销售额/面积 | ¥X/㎡ |
| 库存周转 | 销售额/平均库存 | X次 |

### 服务指标
| 指标 | 计算 | 目标 |
|------|------|------|
| 客户满意度 | 调研评分 | >90% |
| 投诉率 | 投诉/客流 | <1% |
| 复购率 | 回头客/总客 | >30% |
```

### 数据分析

```markdown
## 数据分析

### 销售分析
| 分析维度 | 内容 |
|----------|------|
| 时间维度 | 日/周/月/年 |
| 品类维度 | 各品类销售占比 |
| 客户维度 | 新客/老客 |
| 员工维度 | 个人业绩 |

### 异常分析
| 情况 | 可能原因 |
|------|----------|
| 销售额下降 | 客流少/转化低 |
| 客单价下降 | 主推产品变化 |
| 库存积压 | 补货过多 |
| 库存不足 | 预测不准 |
```

## 检查清单

- [ ] 日常运营流程标准化
- [ ] 库存管理规范
- [ ] 人员培训到位
- [ ] OMO策略清晰
- [ ] 数据指标明确
- [ ] 服务标准统一

## 红牌警告

- 库存积压或短缺
- 服务态度差
- 数据不准确
- OMO割裂
- 员工流失高

## 好/坏对比

**好**：
```
门店运营：
- 标准化流程
- 库存周转健康
- 员工培训到位
- OMO数据打通
→ 效率高、体验好
```

**坏**：
```
门店运营：
- 缺货严重
- 服务参差不齐
- 数据不准确
- 线上线下割裂
→ 效率低、体验差
```
