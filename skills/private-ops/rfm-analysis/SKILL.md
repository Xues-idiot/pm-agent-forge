# Private Ops RFM Analysis Skill

基于 RFM 模型进行用户分层，制定精准运营策略。

## 何时使用

- 建立用户分层体系
- 识别高价值用户
- 设计差异化运营策略
- 优化运营资源分配
- 用户生命周期管理

## RFM 模型基础

### 什么是 RFM

| 维度 | 全称 | 定义 | 含义 |
|------|------|------|------|
| R | Recency | 最近一次消费时间 | 用户活跃度 |
| F | Frequency | 消费频率 | 用户粘性 |
| M | Monetary | 消费金额 | 用户价值 |

### RFM 分层标准

**分值计算：**
```markdown
## RFM 分值计算

### R 分值（最近消费距今天数）
- 5分：0-30天
- 4分：31-60天
- 3分：61-90天
- 2分：91-180天
- 1分：180天以上

### F 分值（消费频次）
- 5分：10次以上
- 4分：6-10次
- 3分：3-5次
- 2分：2次
- 1分：1次

### M 分值（消费金额）
- 5分：5000元以上
- 4分：2000-5000元
- 3分：500-2000元
- 2分：100-500元
- 1分：100元以下
```

**RFM 用户分群：**

| 分群 | R值 | F值 | M值 | 用户特征 | 运营策略 |
|------|-----|-----|-----|---------|---------|
| 重要价值用户 | 高 | 高 | 高 | 核心用户 | VIP服务、专属权益 |
| 重要发展用户 | 高 | 低 | 高 | 有潜力 | 提升频次 |
| 重要保持用户 | 低 | 高 | 高 | 流失风险 | 召回触达 |
| 重要挽留用户 | 低 | 低 | 高 | 即将流失 | 强召回 |
| 一般价值用户 | 高 | 高 | 低 | 活跃度低 | 提升客单价 |
| 一般发展用户 | 高 | 低 | 低 | 有潜力 | 提升频次和金额 |
| 一般保持用户 | 低 | 高 | 低 | 流失风险 | 召回 |
| 一般挽留用户 | 低 | 低 | 低 | 边缘用户 | 减少投入 |

## RFM 分析流程

```
1. 数据准备 → 2. RFM 计算 → 3. 用户分群 → 4. 画像分析 → 5. 策略制定 → 6. 执行监控
```

**数据查询示例：**
```sql
-- 基础 RFM 数据提取
SELECT
    user_id,
    MAX(order_date) as last_order_date,  -- R
    COUNT(order_id) as order_count,        -- F
    SUM(order_amount) as total_amount      -- M
FROM orders
WHERE order_date >= DATE_SUB(CURDATE(), INTERVAL 365 DAY)
GROUP BY user_id
```

### 2. RFM 计算

**Python 实现：**
```python
import pandas as pd
from datetime import datetime

def calculate_rfm(df, reference_date=None):
    """
    计算 RFM 分值

    参数:
        df: 包含 user_id, order_date, order_amount 的 DataFrame
        reference_date: 参考日期，默认今天
    """
    if reference_date is None:
        reference_date = datetime.now()

    # 计算每个用户的 RFM 值
    rfm = df.groupby('user_id').agg({
        'order_date': lambda x: (reference_date - x.max()).days,  # R
        'order_id': 'count',  # F
        'order_amount': 'sum'  # M
    }).reset_index()

    rfm.columns = ['user_id', 'recency', 'frequency', 'monetary']

    # 计算分值（1-5分）
    rfm['R_score'] = pd.cut(rfm['recency'],
                            bins=[0, 30, 60, 90, 180, float('inf')],
                            labels=[5, 4, 3, 2, 1]).astype(int)

    rfm['F_score'] = pd.cut(rfm['frequency'],
                            bins=[0, 1, 2, 3, 6, float('inf')],
                            labels=[1, 2, 3, 4, 5]).astype(int)

    rfm['M_score'] = pd.cut(rfm['monetary'],
                            bins=[0, 100, 500, 2000, 5000, float('inf')],
                            labels=[1, 2, 3, 4, 5]).astype(int)

    return rfm
```

### 3. 用户分群

**分群规则：**
```markdown
## 用户分群定义

def segment_user(row):
    r, f, m = row['R_score'], row['F_score'], row['M_score']

    if r >= 4 and f >= 4 and m >= 4:
        return '重要价值用户'
    elif r >= 4 and f <= 2 and m >= 4:
        return '重要发展用户'
    elif r <= 2 and f >= 4 and m >= 4:
        return '重要保持用户'
    elif r <= 2 and f <= 2 and m >= 4:
        return '重要挽留用户'
    elif r >= 4 and f >= 4 and m <= 2:
        return '一般价值用户'
    elif r >= 4 and f <= 2 and m <= 2:
        return '一般发展用户'
    elif r <= 2 and f >= 4 and m <= 2:
        return '一般保持用户'
    else:
        return '一般挽留用户'
```

### 5. 策略制定

**差异化运营策略：**

| 分群 | 运营目标 | 策略 | 触达频率 | 渠道 |
|------|---------|------|---------|------|
| 重要价值用户 | 保持忠诚 | VIP服务、专属权益 | 低（1次/月） | 私聊 |
| 重要发展用户 | 提升频次 | 提升F的激励 | 中（2次/月） | 私聊+社群 |
| 重要保持用户 | 防止流失 | 召回激励 | 高（1次/周） | 私聊+短信 |
| 重要挽留用户 | 激活留存 | 强利益刺激 | 高（1次/周） | 全部渠道 |
| 一般价值用户 | 提升价值 | 提升M的激励 | 中 | 社群+弹窗 |
| 一般发展用户 | 培育转化 | 成长体系 | 低 | 公众号+社群 |
| 一般保持用户 | 防止流失 | 轻度召回 | 中 | 社群 |
| 一般挽留用户 | 减少投入 | 低成本维护 | 低 | 公众号 |

### 6. 执行监控

**监控指标体系：**

| 指标 | 计算方式 | 目标 | 监控频率 |
|------|---------|------|---------|
| 分群健康度 | 各分群用户数占比变化 | 稳定 | 每周 |
| 价值迁移率 | 高价值用户向下迁移比例 | <5% | 每周 |
| 升级率 | 低价值用户向上迁移比例 | >10% | 每月 |
| 策略响应率 | 触达后产生行为的用户比例 | >20% | 每次活动 |

## RFM 分析检查清单

- [ ] 数据时间范围合理（12个月以上）
- [ ] RFM 分值标准与业务匹配
- [ ] 分群定义清晰可执行
- [ ] 各分群策略差异化
- [ ] 权益成本可控
- [ ] 监控指标完整
- [ ] 定期复盘调整

## 红牌警告

- 用脏数据做分析（退货、测试单未过滤）
- 分群标准一成不变
- 所有用户用同一套策略
- 只关注高价值用户，忽略潜力用户培养
- 运营动作过于频繁打扰用户
- 不监控策略效果

## 好/坏对比

**好分析：**
```
重要价值用户占5%却贡献50% GMV，需要重点维护。
重要发展用户R高但F低，建议设计提升频次的活动。
```

**坏分析：**
```
高价值用户多发短信，召回率高。
（没有具体标准，什么是高价值？发多少？）
```
