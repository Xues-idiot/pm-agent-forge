# A/B Testing

私域运营A/B测试，设计实验验证假设、优化转化。

## 何时使用

- 优化转化率
- 验证运营假设
- 测试不同策略效果
- 降低运营风险
- 数据驱动决策

## 测试变量类型

| 变量 | 测试内容 | 核心指标 |
|------|---------|----------|
| **标题** | 不同类型标题 | 打开率 |
| **正文** | 长短/风格/结构 | 阅读完成率 |
| **CTA** | 按钮文案/颜色/位置 | 点击率 |
| **时机** | 发送时间 | 触达率 |
| **人群** | 不同分层 | 转化率 |

```markdown

**因为**: [原因]

| 项目 | 对照组A | 实验组B |
|------|--------|--------|
| 样本量 | 50% | 50% |
| 分流方式 | 随机 | 随机 |

### 样本计算
- 基准转化率: X%
- 最小可检测变化: X%
- 统计功效: 80%
- 显著性水平: 95%
- **所需样本量**: 每组X人

### 测试周期
- 最短测试天数: X天
- 建议测试天数: X天

### 核心指标
- 指标: [如点击率]
- 目标提升: X%

### 次要指标
- [指标1]
- [指标2]

### 风险控制
- 止损条件: 转化率下降>X%
- 回滚方案: 切换回A版本
```

## 变体设计原则

### 1. 变量单一化
```
❌ 一次改多个变量
✓ 每次只改一个变量
```

### 2. 差异显著性
```
❌ 变体之间差异太小
✓ 变体之间差异明显
```

### 3. 样本均匀性
```
❌ 各变体样本量差异大
✓ 各变体样本量相当
```

### 4. 测试周期
```
❌ 只测1-2天
✓ 至少3-7天
```

## 统计分析

### 统计显著性判断

```markdown
## 统计显著性判断

### 结果数据
| 指标 | 对照组A | 实验组B |
|------|--------|--------|
| 样本量 | X | X |
| 转化数 | X | X |
| 转化率 | X% | X% |

### 统计检验
- 检验方法: Z检验/卡方检验
- p值: X.XXX
- 置信度: XX%
- **结论**: [显著/不显著]

### 效果评估
- 相对提升: +X%
- 置信区间: [X%, X%]
```

### 显著性判断标准

| 置信度 | p值 | 结论 |
|--------|-----|------|
| 95% | <0.05 | 统计显著 |
| 90% | <0.10 | 边缘显著，需验证 |
| <90% | >0.10 | 不显著 |

## 检查清单

### 测试设计
- [ ] 假设明确
- [ ] 变量单一
- [ ] 样本量足够
- [ ] 测试周期合理

### 测试执行
- [ ] 分流均匀
- [ ] 无数据污染
- [ ] 实时监控

### 结果分析
- [ ] 统计显著性判断
- [ ] 无误判风险
- [ ] 有可操作结论

## 红牌警告

- 不做统计显著性判断
- 测试周期太短数据不准
- 同时测多个变量
- 不设置止损条件
- 看到正向就停止测试

## 好/坏对比

**好**：
```
测试假设: 如果标题加数字，CTR提升
设计: 只改标题，固定其他变量
样本: 每组1000人
周期: 7天
结果: +15%, p<0.01, 显著
→ 全量上线
```

**坏**：
```
测试假设: 改标题+改图片+改文案
结果: CTR提升10%
→ 不知道哪个变量起作用
```
