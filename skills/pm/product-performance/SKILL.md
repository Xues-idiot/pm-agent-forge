# Product Performance

产品绩效，分析产品表现和推动改进。

## 何时使用

- 产品数据分析
- 性能评估
- 问题发现
- 优化机会识别
- 复盘分析

## 性能评估

### 评估框架

```markdown
## 性能评估

### 评估维度
| 维度 | 指标 | 说明 |
|------|------|------|
| 用户 | 活跃、留存 | 用户健康度 |
| 业务 | 转化、收入 | 商业价值 |
| 产品 | 功能使用 | 产品力 |
| 技术 | 性能、稳定性 | 技术健康 |

### 评估频率
| 类型 | 频率 | 内容 |
|------|------|------|
| 日常监控 | 每日 | 核心指标 |
| 周度复盘 | 每周 | 趋势分析 |
| 月度评估 | 每月 | 综合评估 |
| 季度评审 | 每季 | 战略复盘 |
```

### 核心指标

```markdown
## 核心指标

### 用户指标
| 指标 | 计算 | 趋势 |
|------|------|------|
| DAU | 日活跃用户 | ↑ |
| MAU | 月活跃用户 | ↑ |
| 次日留存 | 次日回访 | >40% |
| 7日留存 | 7日回访 | >20% |
| 流失率 | 流失/总数 | ↓ |

### 业务指标
| 指标 | 计算 | 趋势 |
|------|------|------|
| 转化率 | 转化/访客 | ↑ |
| GMV | 成交总额 | ↑ |
| 客单价 | GMV/订单 | ↑ |
| 付费率 | 付费/活跃 | ↑ |
```

## 归因分析

### 问题：GMV下跌20%

## 检查清单

- [ ] 指标体系完整
- [ ] 监控机制健全
- [ ] 问题发现及时
- [ ] 归因分析准确
- [ ] 优化落地有效
- [ ] 定期复盘总结

## 红牌警告

- 指标定义混乱
- 只看表面数据
- 问题不深究
- 有问题不解决
- 不做复盘
- 归因不准确

## 好/坏对比

**好**：
```
产品绩效：
- 指标清晰
- 监控到位
- 问题发现早
- 归因准确
- 优化有效
→ 持续改进
```

**坏**：
```
产品绩效：
- 指标混乱
- 问题发现晚
- 归因不清
- 有问题不改
- 从不复盘
→ 问题积累
```
