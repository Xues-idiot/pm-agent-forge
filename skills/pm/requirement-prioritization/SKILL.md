# Requirement Prioritization Skill

产品需求优先级排序，在资源有限时做出正确的取舍。

## 何时使用

- 需求优先级排序
- 迭代计划制定
- 资源争抢裁决
- 路线图规划
- MVP范围定义

## 优先级框架

```
1. 价值评估 → 2. 成本估算 → 3. 优先级排序 → 4. 迭代计划
```

### 1. 价值评估

**用户价值矩阵：**

```markdown
## 用户价值评估

### 价值维度
| 维度 | 说明 | 权重 |
|------|------|------|
| 用户覆盖面 | 多少用户受影响 | 25% |
| 使用频率 | 用户多频繁使用 | 20% |
| 痛点强度 | 解决多痛的问题 | 25% |
| 战略匹配度 | 符合产品方向 | 15% |
| 商业价值 | 收入/转化影响 | 15% |

### 价值评分
| 分值 | 定义 |
|------|------|
| 5 | 核心痛点，所有人都需要 |
| 4 | 重要需求，多数人需要 |
| 3 | 中等需求，部分人需要 |
| 2 | 次要需求，少数人需要 |
| 1 | 边缘需求，极少人需要 |
```

```markdown

### 3. 优先级排序

**ICE评分法：**

```markdown
## ICE评分

### 公式
ICE = Impact × Confidence × Ease

| 维度 | 说明 | 分数 |
|------|------|------|
| Impact | 影响程度 | 1-10 |
| Confidence | 确定性 | 1-10 |
| Ease | 难易度 | 1-10 |

### 计算示例
| 功能 | Impact | Confidence | Ease | ICE |
|------|--------|------------|------|-----|
| A | 9 | 8 | 3 | 216 |
| B | 7 | 6 | 7 | 294 |
| C | 8 | 9 | 4 | 288 |
```

**RICE评分法：**

```markdown
## RICE评分

### 公式
RICE = (Reach × Impact × Confidence) / Effort

| 维度 | 说明 |
|------|------|
| Reach | 影响的用户数 |
| Impact | 对用户的影响 |
| Confidence | 确定程度(%) |
| Effort | 人力(人月) |

### 计算示例
| 功能 | Reach | Impact | Confidence | Effort | RICE |
|------|-------|--------|------------|--------|------|
| A | 10000 | 0.5 | 80% | 3 | 1333 |
| B | 5000 | 0.8 | 70% | 2 | 1400 |
| C | 8000 | 0.3 | 90% | 4 | 540 |
```

**四象限法则：**

```markdown
## 四象限矩阵

        高价值
           │
    紧急   │   紧急+重要
    ───────┼────────
    不紧急 │   不紧急
           │
        低价值

### 优先级定义
| 象限 | 行动 | 说明 |
|------|------|------|
| 紧急+重要 | 立即做 | P0 |
| 重要不紧急 | 计划做 | P1 |
| 紧急不重要 | 尽量简化 | P2 |
| 不紧急不重要 | 不做 | P3 |
```

**资源冲突处理：**

```markdown
## 资源争抢

### 决策原则
1. 战略优先于战术
2. 核心路径优先于辅助功能
3. 高ROI优先于低ROI
4. 团队能力匹配

### 冲突解决
```markdown
## 解决方案

### 方案1：串行执行
- 优点：资源专注
- 缺点：周期长
- 适用：强依赖

### 方案2：增加资源
- 优点：周期可控
- 缺点：成本增加
- 适用：重要但不紧急

### 方案3：缩减范围
- 优点：周期可控
- 缺点：功能不完整
- 适用：时间紧迫

### 方案4：延迟交付
- 优点：保完整
- 缺点：晚上线
- 适用：不紧急
```

## 红牌警告

- 拍脑袋定优先级
- 忽略用户声音
- 政治因素干扰
- 范围蔓延失控
- 忽略技术债务
- 优先级反复变更
- 无视团队能力
