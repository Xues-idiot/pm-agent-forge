# User Journey

用户旅程，绘制用户旅程地图和分析用户触点。

## 何时使用

- 用户旅程绘制
- 触点分析
- 体验优化
- 需求挖掘
- 转化分析

## 用户旅程基础

### 核心概念

```markdown
## 用户旅程

### 定义
用户旅程是用户完成目标所经历的全过程

### 组成要素
| 要素 | 说明 |
|------|------|
| 阶段 | 用户所处的阶段 |
| 目标 | 用户想达成什么 |
| 行为 | 用户实际做了什么 |
| 触点 | 用户接触产品的点 |
| 想法 | 用户在想什么 |
| 情绪 | 用户感受如何 |
| 痛点 | 用户遇到的问题 |
| 机会 | 优化改进点 |

### 与用户旅程的区别
| 概念 | 关注点 |
|------|--------|
| 用户旅程 | 从用户视角看 |
| 业务流程 | 从企业视角看 |
| 用户地图 | 全面呈现+洞察 |
```

## 旅程绘制

### 绘制步骤

```markdown
## 绘制步骤

### 步骤
1. 确定目标用户
2. 确定目标场景
3. 收集用户数据
4. 绘制旅程阶段
5. 补充各阶段细节
6. 识别问题和机会
7. 制定优化方案

### 数据来源
- 用户访谈
- 问卷调研
- 数据分析
- 客服记录
- 社交媒体
```

## 情绪曲线

### 情绪分析

```markdown
## 情绪曲线

### 绘制方法
| 阶段 | 情绪(1-5) | 说明 |
|------|------------|------|
| 认知 | 2 | 不了解 |
| 考虑 | 3 | 犹豫不决 |
| 决策 | 4 | 倾向购买 |
| 开箱 | 3 | 期望落差 |
| 使用 | 5 | 超出预期 |
| 复购 | 4 | 满意 |

### 情绪触点
- 高情绪点：强化
- 低情绪点：修复
- 情绪拐点：重点关注

### 情绪优化

```markdown
## 情绪优化

### 提升策略

|------|------|

| 峰终定律 | 结尾要好 |

## 问题识别

### 痛点分类

```markdown
## 痛点分类

### 痛点层级
| 层级 | 说明 | 优先级 |
|------|------|--------|
| 表层痛点 | 表面问题 | 低 |
| 深层痛点 | 根本原因 | 高 |
| 隐性痛点 | 用户未意识到 | 中 |

### 痛点分析
```markdown
## 痛点分析

### 分析框架
1. 这是什么痛点？
2. 痛点有多痛？
3. 用户现在怎么解决？
4. 我们能解决吗？
5. 解决后价值多大？
```
```

### 问题优先级

```markdown
## 问题优先级

### 重要度-紧迫度矩阵
| 象限 | 特征 | 行动 |
|------|------|------|
| 重要+紧急 | 核心问题 | 立即做 |
| 重要+不紧急 | 关键优化 | 规划做 |
| 不重要+紧急 | 表面问题 | 尽快做 |
| 不重要+不紧急 | 可做可不做 | 尽量不做 |

### 决策标准
```markdown
## 决策标准

### 做/不做
- [ ] 影响多少用户
- [ ] 痛点多深
- [ ] 解决成本
- [ ] 商业价值
```
```

## 优化方向

### 优化类型
| 类型 | 说明 | 投入 |
|------|------|------|
| 消除 | 去掉痛点 | 高 |
| 减轻 | 减少痛点 | 中 |
| 转化 | 换方式解决 | 低 |

### 优化优先级
1. 高频+高影响
2. 高频+低影响
3. 低频+高影响
```

## 红牌警告

- 只看表面问题
- 忽视用户想法
- 痛点分析不深入
- 优化方案不落地
- 不做效果验证
- 与业务流程脱节
