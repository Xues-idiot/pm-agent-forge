# Tracking Design Skill

埋点设计，支持产品数据采集和分析。

## 何时使用

- 埋点方案设计
- 事件规范定义
- 埋点需求撰写
- 数据口径统一
- 埋点验收测试

## 埋点设计流程

```
1. 需求分析 → 2. 指标拆解 → 3. 事件设计 → 4. 文档输出 → 5. 验收上线
```

| 指标 | 定义 | 目标值 |
|------|------|--------|
| | | |

### 分析维度

### 2. 指标拆解

**指标体系：**

```markdown
## 指标拆解

### 核心指标
```markdown
## GMV拆解

GMV = 访问用户数 × 转化率 × 客单价

访问用户数 = 新用户 + 老用户
转化率 = 下单数 / 访问数
客单价 = GMV / 下单数
```

### 漏斗指标
```markdown
## 购买漏斗

访问 → 浏览 → 加购 → 下单 → 支付 → 成交
100%   80%    30%    10%    8%    6%
```
```

### 3. 事件设计

**事件命名规范：**

```markdown
## 事件命名

### 命名规则
[页面]_[动作]

### 示例
- home_page_view
- product_detail_view
- button_click
- add_cart_success
- order_create
- pay_success

### 禁止
- ❌ 中划线：product-detail-view
- ❌ 拼音：shouye_dianji
- ❌ 缩写：pv/ctr
- ❌ 无意义：event1/page2
```

**事件设计模板：**

```markdown
## 事件定义

### 事件基本信息
| 字段 | 内容 |
|------|------|
| 事件ID | [唯一标识] |
| 事件名称 | [英文名] |
| 中文名称 | [中文名] |
| 触发时机 | [何时触发] |

### 事件参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| | | | |

### 代码示例
```javascript
// 伪代码
track('event_name', {
  param1: 'value1',
  param2: 'value2'
});
```

### 采集示例
```json
{
  "event": "event_name",
  "time": 1234567890,
  "params": {
    "param1": "value1",
    "param2": "value2"
  }

```
```

**常用事件模式：**

```markdown
## 通用事件

### 页面事件
```markdown
## page_view

触发时机：页面加载完成
参数：
- page_id: 页面标识
- page_name: 页面名称
- refer: 来源页面
- duration: 停留时长（离开时）
```

### 点击事件
```markdown
## button_click

触发时机：按钮点击
参数：
- button_id: 按钮ID
- button_name: 按钮文案
- position: 按钮位置
- page: 所属页面
```

### 表单事件
```markdown
## form_submit

触发时机：表单提交
参数：
- form_id: 表单ID
- form_name: 表单名称
- success: 是否成功
- error_msg: 错误信息（如有）
```

### 交易事件
```markdown
## order_pay

触发时机：支付成功
参数：
- order_id: 订单ID
- amount: 支付金额
- product_count: 商品数量
- payment_method: 支付方式
```
```

## 红牌警告

- 重复埋点/遗漏埋点
- 参数定义混乱
- 数据口径不统一
- 敏感信息采集
- 命名不规范难理解
- 只埋不用
- 从不验收数据准确性
- 文档缺失难维护
