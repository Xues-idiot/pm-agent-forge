# PM Competitive Analysis Skill

系统性地分析竞争对手，找到差异化机会。

## 何时使用

- 制定产品策略前
- 开发新功能前确认市场空白
- 竞品发布新版本
- 融资/战略汇报需要

## 竞品分析流程

```
1. 确定竞品范围 → 2. 收集信息 → 3. 多维度对比 → 4. 识别机会 → 5. 制定策略
```

### 1. 确定竞品范围

**竞品分类：**
| 类型 | 定义 | 例子 | 分析重点 |
|------|------|------|---------|
| 直接竞品 | 相同目标用户，相同需求 | 微信 vs 钉钉 | 功能对比 |
| 间接竞品 | 相同目标用户，不同解决方案 | 微信 vs 抖音 | 用户时间争夺 |
| 替代品 | 不同产品满足相同需求 | 音乐 vs 播客 | 场景替代 |

**竞品选择原则：**
- 目标用户重叠
- 在细分市场有影响力
- 有值得学习的地方

### 2. 收集信息

**信息来源：**
| 来源 | 内容 | 可靠性 |
|------|------|--------|
| 竞品官网 | 功能、定价、文档 | 高 |
| 产品体验 | 亲自使用 | 高 |
| App Store/评论 | 用户反馈 | 中 |
| 行业报告 | 市场份额、趋势 | 中 |
| 社交媒体 | 用户讨论 | 低 |
| 招聘网站 | 技术栈、发展方向 | 中 |

### 3. 多维度对比

**对比维度：**
```markdown
## 功能对比矩阵

| 功能 | 我们的产品 | 竞品A | 竞品B | 竞品C |
|------|----------|-------|-------|-------|
| 功能1 | ✓ | ✓ | ✓ | ✗ |
| 功能2 | ✓ | ✗ | ✓ | ✓ |
| 功能3 | ✗ | ✓ | ✗ | ✓ |
```

**深度对比维度：**

| 维度 | 具体指标 |
|------|---------|
| 核心功能 | 功能完整性 |
| 用户体验 | 界面设计、交互流畅度 |
| 性能 | 响应速度、稳定性 |
| 定价 | 性价比、收费模式 |
| 生态 | 第三方集成、API |
| 客服 | 支持渠道、响应速度 |
| 安全 | 认证、加密、合规 |

### 4. 识别机会

**机会识别框架：**

```
                  用户需求强度
                     ↑
                     │
        蓝海机会      │      差异化机会
    （无竞品满足）    │    （竞品做不好）
                     │
  ────────────────────┼──────────────────→
                     │
        避免进入      │      跟进即可
    （红海竞争）      │    （成熟功能）
                     │
                     └──────────────→
                              竞争激烈程度
```

**差异化机会矩阵：**

| 机会类型 | 描述 | 策略 |
|---------|------|------|
| 功能差异化 | 竞品没有的功能 | 创新 |
| 体验差异化 | 同样的功能，更好的体验 | 极致体验 |
| 价格差异化 | 更低的价格 | 成本领先 |
| 垂直差异化 | 专注特定行业 | 专业化 |
| 规模差异化 | 大规模定制 | 平台化 |

## 竞品分析方法论

### SWOT 分析

| | 有利 | 不利 |
|--|------|------|
| **内部** | Strengths（优势） | Weaknesses（劣势） |
| **外部** | Opportunities（机会） | Threats（威胁） |

### 护城河分析

| 护城河类型 | 判断标准 | 例子 |
|-----------|---------|------|
| 转换成本 | 用户离开有多难 | 数据迁移、习惯 |
| 网络效应 | 用户越多价值越大 | 社交、平台 |
| 成本优势 | 更低成本提供服务 | 规模、专利 |
| 无形资产 | 品牌、资质 | 品牌、许可证 |

### 竞品监控机制

```markdown
## 竞品监控计划

### 定期监控（每月）
- 产品更新日志
- 社交媒体动态
- 用户评价变化

### 事件触发监控
- 竞品融资新闻
- 重大功能发布
- 高管变动

### 季度深度分析
- 全面功能对比
- 策略调整分析
- 用户调研对比
```

## 竞品分析检查清单

- [ ] 竞品范围合理（不多不少）
- [ ] 信息来源可靠
- [ ] 功能对比客观
- [ ] 差异化机会有数据支撑
- [ ] 策略建议可执行

## 红牌警告

- 只分析直接竞品，忽略间接替代
- 信息来源单一
- 只看功能对比，忽略用户体验
- 假设竞品策略不变
- 结论与产品实际能力脱节

## 好/坏对比

**好分析：**
```
竞品A的审批功能使用率只有20%，因为其移动端体验差。
我们的移动端体验更好，可以强化这个点。
```

**坏分析：**
```
竞品A有审批功能，我们也要有。
```
