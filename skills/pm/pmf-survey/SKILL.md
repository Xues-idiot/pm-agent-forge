# PMF Survey

产品市场匹配度测量，用Rahul Vohra的Superhuman框架。

## 何时使用

- 评估产品是否有PMF
- 创建PMF调查问卷
- 分析已有调查结果
- 判断产品是否值得继续投入

## 核心公式

```
>40% 的用户回答"非常失望" = 有PMF
<40% = 还没有PMF，需要继续迭代
```

---

## 创建PMF调查

### 前置条件检查

- [ ] 有真实使用过产品的用户？
- [ ] 用户最近2周至少使用2次？

### 四问题调查问卷

**Q1: 核心问题**
> 如果你不能再使用这个产品，你会多失望？
- 非常失望
- 有点失望
- 不失望

**Q2: 用户画像**
> 什么类型的人最能从这个产品受益？

**Q3: 主要收益**
> 你从这个产品得到的主要好处是什么？

**Q4: 改进建议**
> 你觉得什么可以改进？

### 调查实施指南

**发送对象：**
- 活跃用户（不是所有人）
- 最近2周使用过至少2次
- 至少收集100份回复

**调查工具：**
- Typeform
- Google Forms
- 邮件发送

## 分析PMF结果

### PMF分数计算

```markdown
## PMF分数计算

总回复数: N
回答"非常失望": X

PMF分数 = X / N × 100%

## 判定标准
| PMF分数 | 状态 | 行动 |
|---------|------|------|
| >40% | 有PMF | 加速增长 |
| 25-40% | 接近PMF | 专注改进 |
| <25% | 没有PMF | 重新审视产品 |
```

### 用户分层

```markdown
## 用户分层

### Very Disappointed (支持者)
- 回答"非常失望"
- 你的核心用户
- 重点关注他们的反馈

### Somewhat Disappointed (中立者)
- 回答"有点失望"
- 需要转化
- 关注他们的障碍

### Not Disappointed (反对者)
- 回答"不失望"
- 可以忽略
- 他们的需求不代表主流
```

### 行动方案

```markdown
## PMF分析报告

### PMF分数: XX%

### 支持者特征 (Very Disappointed)
从Q2汇总：
- 用户类型1
- 用户类型2
- 用户类型3

### 他们喜欢什么 (Q3答案汇总)
1. [主要收益1]
2. [主要收益2]
3. [主要收益3]

### 他们希望改进什么 (Q4答案汇总)
1. [改进点1]
2. [改进点2]

### 下一步行动

#### 应该加倍投入
- [基于支持者反馈的功能/方向]

#### 可以忽略
- [反对者的需求]

#### 转化中立者
- [针对障碍的改进]
```

## 检查清单

- [ ] 确认调查对象是活跃用户
- [ ] Q1是必答
- [ ] 收集足够样本量(>100)
- [ ] 正确计算PMF分数
- [ ] 区分支持者/中立者/反对者
- [ ] 基于支持者反馈制定行动

## 红牌警告

- 对所有用户发调查（包括从未使用的）
- PMF分数<40%却停止迭代
- 关注反对者需求而忽视支持者
- 样本量不足(<50)就下结论

## 好/坏对比

**好**：
```
PMF调查100人，42%非常失望
→ 有PMF，加速增长
→ 重点投入支持者喜欢的功能
→ 忽略不失望的用户
```

**坏**：
```
对所有注册用户发调查（包括从不使用的）
PMF 15%，但产品继续按原计划开发
→ 没有真正理解用户需求
```
