# Pricing Optimization

定价策略优化，订阅模型设计和价格敏感性分析。

## 何时使用

- 制定新定价
- 优化现有定价
- 设计订阅模型
- 分析价格敏感度
- 竞品定价对标

## 定价框架

### 成本基础定价

```markdown
## 成本基础定价

### 成本结构
| 成本类型 | 金额 | 说明 |
|----------|------|------|
| 固定成本 | ¥X | 研发、人员 |
| 变动成本 | ¥X/单 | 带宽、服务 |
| 单位成本 | ¥X | 总成本/用户数 |

### 定价计算
- 目标利润率：X%
- 最低价格：单位成本 / (1 - 利润率)
```

### 价值基础定价

```markdown
## 价值基础定价

### 价值量化
| 价值点 | 用户感知价值 | 量化依据 |
|--------|-------------|----------|
| 节省时间 | ¥100/月 | 用户时薪×节省小时 |
| 提升收入 | ¥500/月 | 用户收入增量 |
| 降低风险 | ¥200/月 | 风险敞口×概率 |

### 定价区间
- 最低价：成本 + 合理利润
- 最高价：用户感知价值
- 建议价：区间中点或偏向用户价值
```

### 竞品对标定价

```markdown
## 竞品对标

| 竞品 | 价格 | 包含功能 | 我们的定位 |
|------|------|----------|-----------|
| 竞品A | ¥99/月 | 功能1,2,3 | 略低 |
| 竞品B | ¥199/月 | 功能1,2,3,4 | 相同 |
| 我们 | ¥149/月 | 功能1,2,3,4,5 | 更高性价比 |

### 差异化定价策略
- 功能差异化：提供竞品没有的功能
- 价格差异化：同等功能更低价格
- 服务差异化：更好的支持
```

## 订阅模型设计

### 订阅层级设计

```markdown
## 订阅层级

### 常见模式：Freemium + 3层订阅

| 层级 | 价格 | 目标用户 | 核心功能 | 限制 |
|------|------|----------|----------|------|
| Free | ¥0 | 试用用户 | 基础功能 | 功能限制 |
| Pro | ¥29/月 | 个人用户 | 核心功能 | 无限制 |
| Team | ¥99/月 | 小团队 | 协作功能 | 5人 |
| Enterprise | ¥299/月 | 企业 | 全功能+支持 | 不限 |

### 升级路径
Free → Pro → Team → Enterprise

### 定价心理学
- 锚定效应：设置高价锚点
- 损失厌恶：强调"不买会损失什么"
- 套餐打包：将相关功能打包
```

### 订阅周期策略

```markdown
## 订阅周期

### 周期选项
| 周期 | 折扣 | 用户选择理由 |
|------|------|--------------|
| 月付 | 0% | 低门槛尝试 |
| 年付 | 15-20% | 长期承诺折扣 |
| 多年付 | 25-30% | 锁定长期客户 |

### 推荐策略
- 新用户：月付优先，降低决策门槛
- 续费：引导年付，提升LTV
- 高价值用户：推多年付，锁定竞争
```

## 价格敏感度分析

### Van Westendorp模型

```markdown
## 价格敏感度测试

### 问卷问题
1. 价格贵不贵？（便宜-太贵）
2. 价格便不便宜？（太便宜-太贵）
3. 什么价格会觉得贵？
4. 什么价格会觉得便宜？

### 分析
- 最低价格 (Too Cheap)
- 较低价格 (Inexpensive)
- 较高价格 (Expensive)
- 最高价格 (Too Expensive)

### 合理价格区间
最优价格点在"不觉得贵"和"不觉得便宜"的交叉区域
```

### A/B测试定价

```markdown
## 定价A/B测试

### 测试设计
| 组 | 价格 | 样本 | 时长 |
|----|------|------|------|
| A组 | ¥99 | 50% | 2周 |
| B组 | ¥129 | 50% | 2周 |

### 评估指标
- 转化率：购买用户/访客
- revenue per visitor：总收入/访客
- 取消率：首月取消率

### 决策
| 场景 | 转化率变化 | revenue per visitor | 决策 |
|------|------------|---------------------|------|
| A优 | -5% | +3% | B价格更好 |
| B优 | +10% | +8% | B价格更好 |
```

## 定价策略场景

### 新产品定价

```markdown
## 新产品定价流程

1. 成本核算
   - 最低价格 = 成本 / (1 - 目标利润率)

2. 价值评估
   - 用户调研：愿意付多少
   - 竞品对标：市场行情

3. 定价测试
   - MVP测试：小规模验证
   - A/B测试：最优价格

4. 监控优化
   - 转化率
   - 客诉率
   - 竞品动态
```

### 提价策略

```markdown
## 提价策略

### 何时提价
- 成本上涨
- 功能大幅增强
- 供不应求
- 竞品提价

### 提价方法
| 方法 | 适用场景 | 风险 |
|------|----------|------|
| 直接提价 | 忠诚用户多 | 流失风险 |
| 版本迭代提价 | 老版本不涨价 | 需新产品 |
| 隐性提价 | 减少功能 | 用户感知差 |
| 锚定提价 | 新高端版本 | 引导中端 |

### 沟通策略
✓ 提前通知
✓ 强调价值增量
✓ 给老用户优惠
✓ 提供替代选项
```

## 检查清单

- [ ] 成本核算清楚
- [ ] 用户价值评估
- [ ] 竞品分析完成
- [ ] 定价测试验证
- [ ] 监控指标建立

## 红牌警告

- 定价低于成本
- 不考虑用户价值
- 完全照搬竞品
- 定价后不监控

## 好/坏对比

**好**：
```
定价前：
- 核算成本
- 调研用户价值
- 竞品对标
- A/B测试验证
→ 找到最优价格
```

**坏**：
```
"竞品收99，我们收79"
→ 陷入价格战
→ 用户可能觉得便宜没好货
```
