# Product Analytics Skill

分析产品指标，理解用户行为，做出数据驱动的产品决策。

## 何时使用

- 产品数据分析
- 用户行为分析
- 漏斗分析
- 留存分析
- 指标异动分析
- 数据驱动决策

## 产品分析流程

```
1. 指标搭建 → 2. 数据采集 → 3. 分析诊断 → 4. 洞察发现 → 5. 决策建议
```

### 1. 指标搭建

**北极星指标：**

```markdown
## 北极星指标

### 选择标准
- [ ] 反映用户价值
- [ ] 可衡量可追踪
- [ ] 与业务目标强相关
- [ ] 全团队可理解

### 常见北极星指标
| 产品类型 | 北极星指标 |
|----------|-----------|
| 社交 | DAU / 用户时长 |
| 电商 | GMV / 支付用户数 |
| SaaS | 付费转化率 / 续费率 |
| 游戏 | DAU / 付费率 |
| 内容 | 用户时长 / 完播率 |

### 指标分解
```markdown
## 指标分解

北极星指标 → 先行指标 → 代理指标

例：GMV
    ↓
日活 × 转化率 × 客单价
    ↓
新增用户 + 留存用户
```
```

**指标体系：**

```markdown
## 指标体系

### AARRR指标
| 阶段 | 核心指标 | 细分指标 |
|------|-----------|-----------|
| Acquisition | CAC, 渠道量 | 曝光, CTR, 注册率 |
| Activation | AHA时刻达成率 | 首次体验分, 核心功能使用 |
| Retention | 留存率, churn | 次留, 7留, 30留 |
| Referral | K-factor, NPS | 邀请率, 转化率 |
| Revenue | ARPU, LTV | 付费率, 客单价 |

### 产品健康度指标
| 指标 | 定义 | 预警阈值 |
|------|------|----------|
| 核心功能使用率 | 使用核心功能用户/DAU | <50% |
| 用户满意度 | 调研评分 | <4.0 |
| 核心路径转化 | 完成核心路径用户/DAU | <30% |
```

### 2. 数据采集

**埋点设计：**

```markdown
## 埋点设计

### 埋点原则
- [ ] 核心事件必埋
- [ ] 关键路径埋点
- [ ] 属性要完整
- [ ] 不埋敏感信息

### 事件设计
```markdown
## 事件规范

### 事件命名
[页面]_[动作]

### 事件示例
- page_view
- button_click
- form_submit
- search_query
- item_add_cart
- order_create
- order_pay
```

### 事件属性
```javascript
// 事件上报格式
{
  event: "order_pay",
  properties: {
    page: "checkout_page",
    product_id: "SKU123",
    product_name: "商品名称",
    price: 99.00,
    payment_method: "wechat",
    user_type: "new" // 新老用户
  }
}
```
```

### 3. 分析诊断

**漏斗分析：**

```markdown
## 漏斗分析

### 核心漏斗
```markdown
## 注册漏斗

访问首页 100%
    ↓ 50%
点击注册 50%
    ↓ 70%
开始注册 35%
    ↓ 80%
完成注册 28%
    ↓ 60%
完成新手引导 17%
    ↓ 50%
完成首次体验 8.5%
```

### 漏斗诊断
| 步骤 | 转化率 | 行业对比 | 问题 |
|------|---------|----------|------|
| 访问→点击 | | 偏高/偏低 | 入口不明显 |
| 点击→开始 | | 偏低 | 表单过长 |
| 开始→完成 | | 偏高/偏低 | 引导不清晰 |
```

- [ ] L型曲线：快速下降后稳定

|--------|------|------|------|
| 新用户 | | | |

### 留存分析维度
- 按渠道

### 路径1：完成核心需求

### 4. 洞察发现

**异动分析：**

```markdown
## 异动分析

### 异动识别
| 指标 | 昨日 | 今日 | 变化 | 正常波动 |
|------|------|------|------|-----------|
| DAU | 100万 | 85万 | -15% | ±5% |

### 归因分析
```markdown
## 归因方法

### 维度拆解
DAU = 新用户 + 回流用户 + 留存用户

排查：哪个维度出现问题？

### 交叉分析
- 渠道维度
- 时间维度
- 用户维度
- 地域维度
```

**用户分群：**

```markdown
## 用户分群

### 行为分群
| 群体 | 定义 | 特征 |
|------|------|------|
| 高价值用户 | 近30天付费>500 | 高活跃, 高消费 |
| 潜力用户 | 高活跃未付费 | 转化潜力 |
| 沉默用户 | 7天未登录 | 流失风险 |
| 流失用户 | 30天未登录 | 已流失 |

### RFM分群
| 类型 | R(最近) | F(频率) | M(金额) | 运营策略 |
|------|---------|---------|---------|-----------|
| 重要价值 | 近 | 高 | 高 | VIP服务 |
| 重要发展 | 近 | 低 | 高 | 提升频率 |
| 重要保持 | 远 | 高 | 高 | 召回 |
| 重要挽留 | 远 | 低 | 高 | 重点召回 |
```

## 产品分析检查清单

- [ ] 指标定义清晰
- [ ] 埋点数据准确
- [ ] 分析维度全面
- [ ] 结论有数据支撑
- [ ] 建议可落地
- [ ] 后续跟踪到位

## 红牌警告

- 数据采集不准确
- 指标定义混乱
- 以偏概全
- 虚假数据
- 过度解读
- 忽视统计显著性
- 结论脱离业务
- 不跟踪效果
