# User Story Mapping Skill

创建用户故事地图，规划版本发布，可视化用户旅程。

## 何时使用

- 需求梳理
- 版本规划
- 用户旅程设计
- Sprint计划
- MVP规划

## 用户故事地图流程

```
1. 定义用户 → 2. 绘制地图骨架 → 3. 填充故事 → 4. 识别 MVP → 5. 版本规划
```

### 1. 定义用户

**用户角色：**

```markdown
## 用户角色

### 主要角色
| 角色 | 目标 | 痛点 | 使用频率 |
|------|------|------|---------|
| 角色A | | | |
| 角色B | | | |

### 角色详细描述
```markdown
**角色：[角色名称]**

- **人口特征：** 年龄、职业、使用场景
- **目标：** 用户想完成什么
- **动机：** 为什么有这个目标
- **障碍：** 什么阻止他们
- **当前方案：** 他们现在怎么做的
```
```

### 2. 绘制地图骨架

**活动-任务-步骤结构：**

```markdown
## 用户故事地图骨架

### 第一行：用户活动（Activities）
```
[活动1] → [活动2] → [活动3] → [活动4]
```

### 活动拆解
| 活动 | 用户任务 | 关键步骤 |
|------|----------|---------|
| 活动1 | 任务1.1 | 步骤1, 步骤2 |
| | 任务1.2 | 步骤3, 步骤4 |
| 活动2 | 任务2.1 | 步骤5, 步骤6 |
```

### 地图模板
```markdown
        步骤1  步骤2  步骤3  步骤4  步骤5  步骤6
活动1  [  ●  ] [  ●  ] [  ○  ] [  ○  ] [  ○  ] [  ○  ]
活动2  [  ○  ] [  ●  ] [  ●  ] [  ●  ] [  ○  ] [  ○  ]
活动3  [  ○  ] [  ○  ] [  ○  ] [  ●  ] [  ●  ] [  ●  ]
        ↑        ↑        ↑
      背部      腰部      腿部
```

### 3. 填充故事

**用户故事格式：**

```markdown
## 用户故事

### 标准格式
```markdown
**作为** [用户角色]
**我想要** [功能]
**以便** [实现的价值]
```

### 示例
```markdown
作为 [新用户]
我想要 [一键登录]
以便 [快速开始使用产品]
```

### 故事详情卡
```markdown
## 用户故事卡片

### ID：US-001
- **角色：** 新用户
- **故事：** 一键登录
- **价值：** 降低注册门槛

### 验收标准
- [ ] 支持微信一键登录
- [ ] 支持手机号一键登录
- [ ] 登录时间 < 3秒

### 优先级
- [ ] Must Have
- [ ] Should Have
- [ ] Could Have

### 技术备注
- 依赖：微信/支付宝SDK
- 数据需求：手机号、昵称、头像
```
```

**故事仓库：**

```markdown
## 故事仓库

| ID | 用户角色 | 故事 | 活动 | 优先级 | 故事点 |
|----|----------|------|------|--------|--------|
| US-001 | 新用户 | 一键登录 | 注册 | P0 | 3 |
| US-002 | 新用户 | 浏览首页 | 了解 | P1 | 2 |
| US-003 | 老用户 | 搜索内容 | 查找 | P0 | 5 |
| US-004 | 老用户 | 收藏内容 | 管理 | P1 | 3 |
| US-005 | 付费用户 | 升级会员 | 付费 | P0 | 5 |
```

### 4. 识别 MVP

**MVP 边界：**

```markdown
## MVP 定义

### MVP 原则
- 只做最小功能集
- 解决核心问题
- 可交付可用版本

### MVP vs 非MVP
```markdown
## MVP 功能（必做）
| 功能 | 用户价值 | 开发成本 | 优先级 |
|------|----------|---------|--------|
| 登录注册 | 高 | 中 | P0 |
| 浏览核心内容 | 高 | 低 | P0 |
| 基础搜索 | 高 | 低 | P0 |

## 后续功能（可延后）
| 功能 | 用户价值 | 开发成本 | 优先级 |
|------|----------|---------|--------|
| 高级筛选 | 中 | 中 | P1 |
| 收藏功能 | 中 | 低 | P1 |
| 分享功能 | 低 | 低 | P2 |
```
```

### 5. 版本规划

**版本路线图：**

```markdown
## 版本规划

### Version 1.0 - MVP
**目标：** 核心用户可完成主流程

| 故事ID | 功能 | 故事点 | Sprint |
|--------|------|--------|--------|
| US-001 | 登录注册 | 3 | Sprint 1 |
| US-002 | 浏览首页 | 2 | Sprint 1 |
| US-003 | 搜索内容 | 5 | Sprint 2 |
| **总计** | | **10** | **2 Sprint** |

### Version 1.1 - 基础互动
**目标：** 用户可互动和留存

| 故事ID | 功能 | 故事点 | Sprint |
|--------|------|--------|--------|
| US-004 | 收藏功能 | 3 | Sprint 3 |
| US-005 | 评论功能 | 5 | Sprint 3 |
| US-006 | 分享功能 | 3 | Sprint 4 |

### Version 2.0 - 商业化
**目标：** 实现商业变现

| 故事ID | 功能 | 故事点 | Sprint |
|--------|------|--------|--------|
| US-007 | 会员体系 | 8 | Sprint 5 |
| US-008 | 支付功能 | 5 | Sprint 5 |
| US-009 | 订单管理 | 5 | Sprint 6 |
```

## 用户故事地图检查清单

- [ ] 用户角色清晰
- [ ] 活动流程完整
- [ ] 覆盖主路径和分支路径
- [ ] 每个故事有验收标准
- [ ] 优先级合理
- [ ] MVP 边界明确
- [ ] 版本规划可行
- [ ] 团队已对齐

## 红牌警告

- 遗漏关键用户路径
- 故事过于复杂
- 优先级与业务目标脱节
- MVP 定义过于庞大
- 忽视非功能需求
- 故事点估算偏差大
- 技术债务未考虑
- 用户角色假设未验证
