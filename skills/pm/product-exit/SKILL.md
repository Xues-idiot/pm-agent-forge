# Product Exit

产品退出管理，产品终止决策和退出执行。

## 何时使用

- 产品关停决策
- 退出策略制定
- 数据迁移
- 用户沟通
- 遗留资产处理

## 退出评估

### 退出信号

```markdown
## 退出信号

### 定性信号
| 信号 | 说明 |
|------|------|
| 用户负增长 | 连续6个月下滑 |
| 竞品碾压 | 市场份额被蚕食 |
| 技术老旧 | 架构无法演进 |
| 战略放弃 | 不再是核心业务 |

### 定量信号
| 指标 | 阈值 |
|------|------|
| MAU下降 | 连续3月<-20% |
| 收入下降 | 连续2季度亏损 |
| ROI为负 | 持续投入无回报 |
| 用户留存 | <10% |

### 退出评估
```markdown
## 退出评估

### 评估维度
| 维度 | 问题 | 权重 |
|------|------|------|
| 财务影响 | 退出损失多少 | 25% |
| 用户影响 | 多少用户受影响 | 25% |
| 品牌影响 | 对品牌影响多大 | 20% |
| 竞争影响 | 放弃什么市场 | 15% |
| 团队影响 | 人员如何安置 | 15% |
```
```

## 退出策略

### 退出路径

```markdown
## 退出路径

### 选项
| 策略 | 说明 | 适用 |
|------|------|------|
| 直接关闭 | 立即停止服务 | 无替代品 |
| 逐步关闭 | 停止新用户+功能 | 有迁移方案 |
| 卖给其他 | 业务出售 | 有买家 |
| 转型 | 改变产品定位 | 找到新方向 |

### 决策树
```markdown
## 决策树

是否有替代产品？
    │
    ├── 有 → 引导迁移
    │
    └── 无 → 评估用户价值
                │
                ├── 高价值 → 继续运营/找替代
                │
                └── 低价值 → 逐步关闭
```
```

### 时间规划

```markdown
## 退出时间表

### 典型周期
| 阶段 | 时间 | 内容 |
|------|------|------|
| 决策期 | 1-2月 | 评估和决策 |
| 准备期 | 2-3月 | 方案准备 |
| 通知期 | 1-3月 | 用户沟通 |
| 执行期 | 1-2月 | 技术实施 |
| 收尾期 | 1月 | 数据存档 |

### 里程碑
- D-day: 宣布退出
- D-30: 开始迁移
- D-15: 强制迁移
- D-0: 服务关闭
```

## 用户迁移

### 迁移方案

```markdown
## 用户迁移

### 迁移类型
| 类型 | 说明 | 用户接受度 |
|------|------|------------|
| 引导迁移 | 推荐替代产品 | 高 |
| 数据导出 | 允许导出数据 | 中 |
| 补偿方案 | 付费用户补偿 | 中 |
| 退款 | 直接退款 | 低 |

### 迁移激励
```markdown
## 迁移激励

### 激励措施
| 措施 | 说明 | 成本 |
|------|------|------|
| 专属优惠 | 替代品优惠券 | 低 |
| 优先体验 | 新功能优先 | 低 |
| 专属客服 | 1v1服务 | 中 |

```
```

## 红牌警告

- 决策犹豫不决
- 用户突然告知
- 无迁移方案
- 数据不处理
- 员工最后才知道
- 不做复盘

## 好/坏对比

**好**：
```
产品退出：
- 评估充分
- 策略清晰
- 沟通透明
- 迁移平滑
- 收尾干净
→ 用户理解、损失最小
```

**坏**：
```
产品退出：
- 决策拖延
- 用户突然炸锅
- 无替代方案
- 数据丢失
- 员工不满
→ 用户投诉、品牌受损
```
