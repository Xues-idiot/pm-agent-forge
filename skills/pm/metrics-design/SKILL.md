# Product Metrics Design Skill

设计产品指标体系，构建衡量框架，定义产品成功标准。

## 何时使用

- 产品指标体系设计
- KPI制定
- 数据看板设计
- 增长指标定义
- 产品质量衡量

## 指标设计流程

```
1. 业务理解 → 2. 目标分解 → 3. 指标定义 → 4. 数据采集 → 5. 监控体系
```

### 5. 监控体系

**看板设计：**

```markdown
## 数据看板

### 核心看板（每日）
| 指标 | 图表类型 | 刷新频率 |
|------|---------|---------|
| DAU/WAU/MAU | 趋势图 | 日 |
| 留存率 | 漏斗图 | 周 |
| 收入 | 数值+趋势 | 日 |

### 运营看板（实时）
| 指标 | 图表类型 | 刷新频率 |
|------|---------|---------|
| 在线用户 | 数值 | 实时 |
| 请求量 | 折线图 | 5分钟 |
| 错误率 | 仪表盘 | 5分钟 |

### 分析看板（按需）
- 用户路径分析
- 转化漏斗分析
- 留存 cohort 分析
```

**告警规则：**

```markdown
## 告警规则

### P0 紧急告警
| 指标 | 条件 | 通知方式 |
|------|------|---------|
| 服务不可用 | 可用性<99% | 电话+短信 |
| 重大Bug | 错误率>5% | 电话+短信 |

### P1 重要告警
| 指标 | 条件 | 通知方式 |
|------|------|---------|
| 指标异常 | 日环比<-10% | 企微/钉钉 |
| 转化率下降 | 下降>15% | 企微/钉钉 |

### P2 关注告警
| 指标 | 条件 | 通知方式 |
|------|------|---------|
| 趋势异常 | 连续3天偏离 | 企微群 |
| 新功能表现 | 低于预期 | 邮件 |
```

## 指标审查清单

- [ ] 指标与业务目标一致
- [ ] 计算口径清晰无歧义
- [ ] 数据可采集
- [ ] 目标值合理
- [ ] 监控覆盖关键路径
- [ ] 告警阈值设置合理
- [ ] 埋点完整

## 红牌警告

- 指标过多（超过15个核心指标）
- 指标定义模糊
- 口径不一致
- 数据无法采集
- 目标值拍脑袋
- 无监控告警
- 重数量轻质量
- 指标与分析脱节
