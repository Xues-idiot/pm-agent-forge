# User Growth Skill

规划用户增长策略，设计获客漏斗，优化增长实验。

## 何时使用

- 用户增长策略
- 获客漏斗设计
- 增长实验设计
- 裂变增长
- 渠道优化
- 增长黑客

## 增长管理流程

```
1. 增长诊断 → 2. 策略制定 → 3. 漏斗优化 → 4. 实验迭代 → 5. 效果复盘
```

### 1. 增长诊断

**增长指标体系：**

```markdown
## 增长指标

### 漏斗指标
| 阶段 | 指标 | 定义 | 行业基准 |
|------|------|------|----------|
| 曝光 | 曝光量 | 广告展示次数 | - |
| 点击 | CTR | 点击/曝光 | 1-3% |
| 注册 | 注册率 | 注册/点击 | 5-15% |
| 激活 | 激活率 | 激活/注册 | 30-60% |
| 留存 | 次留/7留 | 留存/注册 | 30%/15% |
| 付费 | 付费率 | 付费/活跃 | 2-5% |
| 推荐 | K-factor | 邀请/被邀请×转化 | >1 |

### 核心比率
| 比率 | 计算 | 目标 |
|------|------|------|
| CAC | 获客成本/获客数 | 越低越好 |
| LTV | 用户生命周期价值 | >3×CAC |
| LTV/CAC | 生命周期价值/获客成本 | >3 |
| Payback | 回本周期 | <12个月 |
```

**增长诊断模型：**

```markdown
## 增长诊断

### 海盗指标分析（AARRR）
```markdown
## AARRR诊断

### Acquisition（获客）
- 获客渠道：哪些有效？
- CAC趋势：上升/下降？
- 渠道ROI：哪个渠道最优？

### Activation（激活）
- 激活漏斗：卡点在哪里？
- AHA时刻：用户何时体验到价值？
- 激活率：是否满足目标？

### Retention（留存）
- 留存曲线：是否平稳？
- 流失原因：为什么会流失？
- 留存策略：如何提升留存？

### Referral（推荐）
- K-factor：是否>1？
- 推荐漏斗：哪个环节出问题？
- 推荐动机：为什么用户愿意推荐？

### Revenue（收入）
- ARPU：是否增长？
- 付费转化：哪个环节流失？
- 付费场景：用户为什么付费？
```
```

### 3. 漏斗优化

**漏斗分析：**

```markdown
## 漏斗分析

### 漏斗绘制
```markdown
## 漏斗图

步骤1：曝光 100万
    ↓ 2%
步骤2：点击 2万
    ↓ 30%
步骤3：注册 6000
    ↓ 50%
步骤4：激活 3000
    ↓ 40%
步骤5：留存 1200
    ↓ 10%
步骤6：付费 120
```

### 流失分析
| 步骤 | 流失率 | 流失原因 | 优化方向 |
|------|--------|----------|----------|
| 曝光→点击 | 98% | 素材不吸引 | 优化创意 |
| 点击→注册 | 70% | 注册门槛高 | 简化流程 |
| 注册→激活 | 50% | 引导不足 | 优化引导 |
| 激活→留存 | 60% | 价值不明显 | 优化激活 |
```
```

### 4. 实验迭代

**增长实验：**

```markdown
## 增长实验

### 实验类型
| 类型 | 说明 | 周期 | ROI |
|------|------|------|------|
| A/B测试 | 对照实验 | 1-2周 | 高 |
| 灰度发布 | 渐进发布 | 1-4周 | 中 |
| 问卷调研 | 用户反馈 | 1周 | 低 |
| 访谈 | 深度理解 | 2周 | 中 |

### 实验流程
```markdown
## 实验流程

1. 提出假设
   "如果[改什么]，那么[期望结果]，
   因为[假设依据]"

2. 设计实验
   - 对照组：A方案
   - 实验组：B方案
   - 流量分配：50/50
   - 成功指标：
   - 护栏指标：

3. 执行监控
   - 样本量计算
   - 统计显著性
   - 异常监控

4. 分析结论
   - 显著 → 全量上线
   - 不显著 → 迭代假设
   - 有害 → 回滚
```

## 用户增长检查清单

- [ ] 增长指标定义清晰
- [ ] 漏斗各环节可衡量
- [ ] 策略与数据匹配
- [ ] 实验设计严谨
- [ ] 结论有数据支撑
- [ ] 迭代快速
- [ ] 复盘到位
- [ ] 经验沉淀

## 红牌警告

- 只看新增忽视留存
- CAC持续上升
- LTV/CAC < 1
- 裂变系数 < 1
- 虚假流量
- 实验无统计显著性
- 数据造假
- 忽视用户质量
