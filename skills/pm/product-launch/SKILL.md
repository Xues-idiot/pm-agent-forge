# Product Launch

产品发布管理，规划上线、GTM策略和发布协调。

## 何时使用

- 新产品发布
- 功能迭代上线
- 市场进入策略
- 发布计划制定
- 上线后复盘

## 发布类型

```markdown
## 发布类型

### 按规模
| 类型 | 说明 | 示例 |
|------|------|------|
| 小型发布 | 单功能/局部用户 | Bug修复、体验优化 |
| 中型发布 | 多功能/部分用户 | 功能迭代 |
| 大型发布 | 全新产品/全量用户 | 全新产品线 |

### 按方式
| 类型 | 说明 | 风险 |
|------|------|------|
| 灰度发布 | 逐步放量 | 低 |
| 尝鲜发布 | 用户主动参与 | 中 |
| 全量发布 | 直接上线所有用户 | 高 |
```

## 灰度发布

### 灰度策略

```markdown
## 灰度发布策略

### 灰度节奏
| 阶段 | 时间 | 比例 | 监控重点 |
|------|------|------|----------|
| 内部 | 1-2天 | 员工 | 功能稳定性 |
| 内测 | 3-5天 | 1-5% | 核心指标 |
| 小范围 | 1周 | 5-20% | 全面监控 |
| 中范围 | 1周 | 20-50% | 转化指标 |
| 全量 | - | 100% | 稳定后放开 |

### 灰度规则
```markdown
## 灰度规则

### 用户选择
| 方式 | 说明 | 适用 |
|------|------|------|
| 随机 | 按比例随机 | 通用 |
| 标签 | 特定用户群 | 精准 |
| 白名单 | 指定用户 | KOL/客户 |

### 开启条件
- 上一阶段无P0/P1问题
- 核心指标无异常
- 人工确认可继续
```

### 监控指标

```markdown
## 灰度监控

### 核心监控
| 指标 | 阈值 | 处置 |
|------|------|------|
| 错误率 | <0.1% | >0.5%回滚 |
| 响应时间 | P95<500ms | >1s排查 |
| CPU使用率 | <70% | >80%扩容 |

### 业务监控
| 指标 | 阈值 | 处置 |
|------|------|------|
| 转化率 | 波动<10% | 异常分析 |
| 功能使用率 | 符合预期 | - |
| 用户反馈 | 无批量投诉 | - |

## 红牌警告

- 没有测试就发布
- 没有回滚方案
- 没有灰度直接全量
- 出现问题不监控
- 发布后不跟进

## 好/坏对比

**好**：
```
发布流程：
- 灰度发布，逐步放量
- 每阶段监控核心指标
- 准备回滚方案
- 发布后持续跟踪
→ 问题可控，快速迭代
```

**坏**：
```
发布流程：
- 直接全量发布
- 没有监控告警
- 没有回滚方案
- 发完就结束
→ 问题爆发，无法控制
```
