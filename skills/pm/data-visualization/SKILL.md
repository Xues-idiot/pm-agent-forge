# Data Visualization

数据可视化，设计数据看板和报告。

## 何时使用

- 数据看板设计
- 数据报告制作
- 图表选择
- 数据故事讲述
- 指标可视化

## 可视化原则

### 设计原则

```markdown
## 设计原则

### 核心原则
| 原则 | 说明 |
|------|------|
| 目标明确 | 清楚想表达什么 |
| 选对图表 | 不同数据用不同图 |
| 简化设计 | 减少视觉干扰 |
| 层次分明 | 重点信息突出 |

### 减少干扰
- 去除不必要的网格线
- 减少配色数量
- 去掉多余的图例
- 标签直接标注
```

### 图表选择

```markdown
## 图表选择

### 按用途
| 用途 | 推荐图表 |
|------|----------|
| 趋势 | 折线图 |
| 比较 | 柱状图 |
| 构成 | 饼图、环形图 |
| 分布 | 直方图、散点图 |
| 关系 | 气泡图 |
| 流程 | 漏斗图 |

### 常见错误
| 错误 | 问题 | 正确做法 |
|------|------|----------|
| 饼图太多片 | 难以比较 | 用柱状图 |
| 3D图表 | 视觉扭曲 | 用2D图 |
| 双Y轴 | 容易混淆 | 分离图表 |
| 截断坐标 | 夸大差异 | 从0开始 |
```
```

### 讲故事步骤

## 检查清单

- [ ] 图表选择正确
- [ ] 设计简洁清晰
- [ ] 信息层次分明
- [ ] 数据准确无误
- [ ] 交互友好
- [ ] 受众导向

## 红牌警告

- 图表选择错误
- 设计过于复杂
- 数据不准确
- 信息过载
- 缺乏洞察
- 自说自话

## 好/坏对比

**好**：
```
数据可视化：
- 图表选择正确
- 设计简洁
- 信息清晰
- 受众导向
- 有洞察有建议
→ 决策支持、效率提升
```

**坏**：
```
数据可视化：
- 图表滥用
- 设计混乱
- 数据错误
- 信息过载
- 没有洞察
→ 误导决策
```
