# PM User Research Skill

系统性地了解目标用户的需求、行为和痛点。

## 何时使用

- 需求定义前需要理解用户
- 设计新功能前确认用户真实需求
- 用户反馈问题需要归因
- 验证产品方向是否正确

## 用户研究流程

```
1. 明确研究目标 → 2. 选择研究方法 → 3. 执行研究 → 4. 分析归纳 → 5. 输出洞察
```

### 1. 明确研究目标

在开始前，先确认：
- 要回答什么问题？
- 决策是什么？
- 需要什么程度的确定性？

**好例子：**
```
目标：了解中小企业主在财务管理中的最大痛点
决策：是否开发自动记账功能
确定性：中等（定性验证即可）
```

**坏例子：**
```
目标：了解用户
决策：无
确定性：无所谓
```

### 2. 选择研究方法

| 方法 | 适用场景 | 样本量 | 时间成本 |
|------|---------|--------|---------|
| 用户访谈 | 深度理解需求 | 5-10人 | 高 |
| 问卷调查 | 量化验证 | 100+ | 中 |
| 可用性测试 | 验证交互 | 5-8人 | 中 |
| 数据分析 | 验证行为 | N/A | 低 |
| 竞品分析 | 了解市场 | 3-5个 | 中 |

### 3. 执行研究

**用户访谈模板：**
```markdown
## 开场（2分钟）
- 自我介绍
- 说明目的和时间
- 强调没有正确答案

## 背景了解（5分钟）
- 日常工作是什么？
- 使用什么工具？
- 典型的一天是怎样的？

## 核心问题（15分钟）
- 描述一个遇到的困难...
- 上次遇到这个问题怎么解决的？
- 如果有一个魔法棒...

## 验证假设（5分钟）
- 确认关键洞察
- 追问细节

## 收尾（3分钟）
- 还有什么想说的？
- 可以推荐其他人吗？
```

### 4. 分析归纳

**用户访谈分析：**
1. 转录所有访谈记录
2. 提取关键Quote
3. 识别模式和主题
4. 归类为：痛点、需求、期望
5. 优先级排序

**分析模板：**
```markdown
## 用户访谈分析：[主题]

### 关键发现

| 主题 | 出现次数 | 代表Quote | 优先级 |
|------|---------|----------|--------|
| 记账麻烦 | 8/10 | "每天手动记账太痛苦" | P0 |
| 报表看不懂 | 5/10 | "数字太多，不知道代表什么" | P1 |

### 用户画像

**画像A：[名称]**
- 特征：...
- 核心需求：...
- 痛点：...

### 结论
- 用户真的需要...
- 建议优先...
```

### 5. 输出洞察

**用户洞察格式：**
```markdown
## 用户洞察报告：[主题]

### 核心洞察
1. [洞察1] - 来源：[N个用户]
2. [洞察2] - 来源：[N个用户]

### 用户故事
- 作为 [用户类型]
- 我想要 [功能/行为]
- 以便 [价值/收益]

### 产品建议
| 建议 | 影响 | 成本 | 优先级 |
|------|------|------|--------|
| 建议1 | 高 | 低 | P0 |

### 下一步
- [ ] 验证洞察（追加访谈/数据分析）
- [ ] 原型测试
- [ ] 需求文档
```

## 用户研究方法详解

### 问卷设计原则

| 原则 | 说明 | 反例 |
|------|------|------|
| 具体 | 问具体行为 | 你经常用吗？ |
| 定量 | 用数字评估 | 你觉得好不好？ |
| 场景化 | 描述情境 | 在紧急情况下 |
| 非引导 | 中立提问 | 你不认为X对吧？ |

### 可用性测试流程

1. **准备**（30分钟）
   - 准备任务清单
   - 招募用户（5-8人）
   - 设置记录工具

2. **执行**（45分钟/用户）
   - 欢迎和热身（5分钟）
   - 任务执行（30分钟）
   - 追问和收尾（10分钟）

3. **分析**（60分钟）
   - 观看录像
   - 记录问题
   - 分类严重程度

### 数据分析方法

**定量数据：**
- 用户行为漏斗分析
- 留存 cohort 分析
- 功能使用频率
- 任务完成率

**定性数据：**
- 访谈主题归纳
- NPS/CSAT 分类
- 支持工单聚类

## 用户研究检查清单

- [ ] 研究目标与业务决策关联
- [ ] 选择了合适的研究方法
- [ ] 样本具有代表性
- [ ] 数据可验证
- [ ] 洞察有足够支撑
- [ ] 输出了可行动的结论

## 集成其他技能

### 集成 PRD Writing
用户研究完成后，将洞察转化为 PRD 中的用户故事。

### 集成 Competitive Analysis
结合竞品分析，找出差异化机会。

## 红牌警告

- 样本量太小（少于3人）就下结论
- 用自己的假设代替用户声音
- 调研目标模糊
- 只收集支持自己观点的数据
- 调研结果与决策脱节
