# A/B Testing Skill

设计A/B测试，执行实验，通过测试验证产品假设。

## 何时使用

- 产品功能验证
- 交互方案选择
- 定价测试
- 文案优化
- 转化率提升

## A/B测试流程

```
1. 假设形成 → 2. 方案设计 → 3. 实施准备 → 4. 执行监控 → 5. 结果分析 → 6. 决策输出
```

**测试方案：**

```

**样本量计算：**

```markdown
## 样本量估算

### 参数输入
- 基线转化率：X%
- 最小可检测效应：X%
- 统计显著性：95%
- 功效：80%

### 估算公式
```python
from scipy import stats
import numpy as np

baseline = 0.05  # 基线转化率
mde = 0.10       # 最小可检测效应 (10%)
alpha = 0.05     # 显著性水平
power = 0.80     # 功效

# 计算每组所需样本量
p1 = baseline
p2 = baseline * (1 + mde)

effect_size = (p1 - p2) / np.sqrt((p1 + p2) / 2)
n = int(np.ceil(2 * (stats.norm.ppf(1 - alpha/2) + stats.norm.ppf(power))**2 / effect_size**2))

print(f"每组需要: {n} 样本")
print(f"总计需要: {n * 2} 样本")
```

### 6. 决策输出

**决策框架：**

```markdown
## 测试结论

### 情况1：实验组显著优于对照组
- **决策：** 全量上线实验组
- **条件：** p<0.05 且效果量有意义

### 情况2：实验组显著差于对照组
- **决策：** 保持对照组
- **后续：** 分析原因，改进假设

### 情况3：无显著差异
- **决策：** 选择成本更低/更简单的方案
- **后续：** 寻找更大改进空间

### 情况4：部分指标好部分差
- **决策：** 权衡利弊，或继续测试
- **讨论：** 召集利益相关者决策
```

## A/B测试检查清单

- [ ] 假设清晰可验证
- [ ] 样本量充足
- [ ] 分流随机均匀
- [ ] 埋点准确完整
- [ ] 监控告警到位
- [ ] 分析方法正确
- [ ] 结论有数据支撑
- [ ] 文档记录完整

## 红牌警告

- 样本量不足就下结论
- 测试期间修改方案
- 只看转化不看其他指标
- 忽视统计假阳性
- 选择"好看"的结果
- 测试周期不完整
- 多重比较问题（同时跑多个测试）
- 忽略细分用户群差异
