# Customer Development

客户探索，验证假设和建立客户关系。

## 何时使用

- 产品初期探索
- 验证用户假设
- 用户需求挖掘
- 客户关系建立
- 持续用户接触

## 客户探索流程

```markdown
## 客户探索流程

假设 → 验证 → 学习 → 迭代
  ↑                      │
  └──────────────────────┘
```

### 四步框架

| 步骤 | 内容 | 输出 |
|------|------|------|
| 1. 客户验证 | 找到目标用户 | 用户画像 |
| 2. 假设验证 | 验证需求假设 | 需求优先级 |
| 3. 解决方案 | 共同创造方案 | 解决方案 |
| 4. 市场验证 | 小规模测试 | 增长策略 |

## 用户访谈

### 访谈准备

```markdown
## 访谈准备

### 目标设定
- 验证哪些假设？
- 需要了解什么？
- 访谈对象是谁？

### 访谈对象
| 类型 | 选择标准 | 数量 |
|------|----------|------|
| 潜在用户 | 符合目标画像 | 5-10 |
| 现有用户 | 活跃用户 | 5-10 |
| 流失用户 | 曾使用但离开 | 3-5 |

### 访谈大纲
```markdown
## 访谈大纲

### 开场(5min)
- 自我介绍
- 访谈目的
- 感谢参与

### 背景了解(10min)
- 工作/生活背景
- 日常任务
- 技术使用情况

### 痛点挖掘(20min)
- 最大的挑战是什么？
- 现有解决方案？
- 改进空间？

### 现有方案(15min)
- 现在用什么工具？
- 优缺点？
- 切换成本？

### 价值验证(10min)
- 如果解决会怎样？
- 愿意付多少钱？
- 决策因素？

### 结束(5min)
- 总结确认
- 后续联系
- 感谢
```
```

### 访谈技巧

```markdown
## 访谈技巧

### 提问技巧
| 类型 | 示例 | 目的 |
|------|------|------|
| 开放式 | "能告诉我上次..." | 了解故事 |
| 5Why | 追问5次为什么 | 挖掘根因 |
| 假设验证 | "你会用..." | 验证假设 |
| 情境模拟 | "如果..." | 探索需求 |

### 倾听技巧
✓ 全神贯注，不要打断
✓ 记录关键词
✓ 追问细节
✓ 观察肢体语言

### 避免误区
✗ 不要问"你会想要X功能吗"
✗ 不要引导答案
✗ 不要一次问太多
✗ 不要只看表面
```

## 假设验证

### 假设类型

```markdown
## 假设类型

### 问题假设
| 假设 | 验证问题 |
|------|----------|
| 存在性 | 这个问题真的存在吗？ |
| 频率 | 多久发生一次？ |
| 严重性 | 足够痛吗？ |
| 现状 | 现在怎么解决的？ |

### 解决方案假设
| 假设 | 验证问题 |
|------|----------|
| 有效性 | 这个方案能解决问题吗？ |
| 可用性 | 用户能学会吗？ |
| 差异性 | 比现有方案好多少？ |

### 市场假设
| 假设 | 验证问题 |
|------|----------|
| 市场规模 | 有多少人愿意付钱？ |
| 获取成本 | 获客成本是多少？ |
| 竞争 | 差异化在哪里？ |
```

### 验证方法

```markdown
## 验证方法

### 定性验证
| 方法 | 成本 | 置信度 |
|------|------|--------|
| 用户访谈 | 中 | 高 |
| 现场观察 | 高 | 高 |
| 焦点小组 | 中 | 中 |

### 定量验证
| 方法 | 成本 | 置信度 |
|------|------|--------|
| 问卷调研 | 低 | 中 |
| 众筹验证 | 中 | 高 |
| Landing Page | 低 | 中 |
| MVP测试 | 高 | 高 |

### 验证优先级
1. 最重要假设优先验证
2. 风险最高假设优先验证
3. 成本最低先验证
```

## 检查清单

- [ ] 访谈目标明确
- [ ] 访谈对象选择正确
- [ ] 访谈技巧得当
- [ ] 假设验证系统化
- [ ] 痛点排序有依据
- [ ] 持续接触机制建立

## 红牌警告

- 只访谈不行动
- 访谈样本单一
- 假设不验证就开发
- 用户说啥就做啥
- 不建立持续反馈机制

## 好/坏对比

**好**：
```
用户探索：
- 系统化访谈目标用户
- 深入挖掘痛点和原因
- 验证核心假设
- 持续跟踪反馈
→ 产品解决真问题
```

**坏**：
```
用户探索：
- 不访谈，自己想当然
- 用户访谈走形式
- 假设没验证就开发
- 开发完不跟踪
→ 产品没人要
```
