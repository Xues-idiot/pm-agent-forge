# Feature Specification Skill

编写功能规格说明，定义技术需求，详细描述产品功能。

## 何时使用

- 功能PRD撰写
- 需求细化
- 技术方案设计
- 功能评审
- 验收标准定义

## 功能规格流程

```
1. 背景理解 → 2. 用户故事 → 3. 功能详述 → 4. 技术方案 → 5. 验收标准
```

### 1. 背景理解

**问题定义：**

```markdown
## 功能背景

### 问题描述
[用户/业务面临的什么问题]

### 问题影响
- 用户痛点：
- 业务影响：
- 紧急程度：

### 现有方案
[当前如何解决这个问题]

### 局限分析
[现有方案的问题]
```

### 2. 用户故事

**用户故事模板：**

```markdown

### 主故事
**作为** [用户角色]
**我想要** [功能]
**以便** [实现的价值]

### 子故事
| 故事 | 作为 | 我想要 | 以便 | 优先级 |
|------|------|--------|------|--------|
| | | | | |

### 用户场景
```markdown
**场景1：[场景名称]**
- 给定 [前置条件]
- 当 [操作]
- 然后 [期望结果]
```
```

### 5. 验收标准

**验收条件：**

```markdown
## 验收标准

### 功能验收
| 功能点 | 验收条件 | 测试方法 |
|--------|---------|---------|
| | | |

### 性能验收
- [ ] 页面加载 < X秒
- [ ] 接口响应 < Xms
- [ ] 支持 X 并发

### 兼容性验收
- [ ] Chrome/Firefox/Safari/Edge
- [ ] iOS/Android
- [ ] 最小分辨率

### 安全验收
- [ ] 权限验证
- [ ] 输入校验
- [ ] 敏感数据加密

### 埋点验收
- [ ] 关键事件已埋点
- [ ] 数据准确
```

## 功能评审检查清单

- [ ] 用户故事完整
- [ ] 功能范围清晰
- [ ] 流程无遗漏
- [ ] 边缘情况考虑
- [ ] 技术方案可行
- [ ] 验收标准明确
- [ ] 依赖关系说明
- [ ] 风险识别

## 红牌警告

- 功能范围模糊
- 用户故事缺乏价值
- 验收标准不具体
- 技术方案不可行
- 忽略边缘情况
- 依赖关系未说明
- 安全漏洞
- 性能要求不合理
