# Data-Driven Decision Making

数据驱动决策，分析产品指标和设计数据实验。

## 何时使用

- 分析产品数据
- 做关键业务决策
- 设计A/B测试
- 建立指标体系
- 评估功能效果

## 数据驱动决策流程

```
定义问题 → 收集数据 → 分析洞察 → 决策行动 → 验证效果
```

### 1. 定义问题

**问题定义模板**：
```markdown
## 问题定义

[具体描述业务问题]

- 谁做决策？
- 决策期限？

### 成功标准
- 怎样算"好"的决策？
- 有哪些约束条件？
```

### 4. 分析方法

**常用分析方法**：
| 方法 | 适用场景 | 工具 |
|------|----------|------|
| 漏斗分析 | 转化问题 | SQL/Excel |
| 趋势分析 | 变化追踪 | BI工具 |
| 群组分析 | 用户分群 | Python/SQL |
| 相关性分析 | 因素探索 | Python/R |
| 归因分析 | 价值分配 | 归因模型 |

**漏斗分析模板**：
```markdown
## 漏斗分析

### 漏斗
| 阶段 | 用户数 | 转化率 | 流失率 |
|------|--------|--------|--------|
| 访问 | 10000 | 100% | - |
| 注册 | 3000 | 30% | 70% |
| 首单 | 300 | 10% | 90% |
| 复购 | 100 | 33% | 67% |

### 问题识别
- 流失最大环节：注册→首单（流失90%）
- 可能原因：
  1. 注册到首单路径太长
  2. 新手优惠不够吸引
  3. 产品价值传达不够
```

```markdown
### 风险评估

- 预期：+Z%
```

## 统计基础

### 常用统计概念

```markdown
## 统计基础

### 描述统计
| 概念 | 含义 | 业务应用 |
|------|------|----------|
| 平均值 | 典型值 | 平均订单金额 |
| 中位数 | 中间值 | 收入分布 |
| 百分位数 | 位置 | P95延迟 |
| 标准差 | 离散程度 | 波动分析 |

### 统计显著性
| 概念 | 含义 | 判断标准 |
|------|------|----------|
| p值 | 偶然概率 | p<0.05 |
| 置信区间 | 估计范围 | 95%CI |
| 效应量 | 实际意义 | 业务判断 |
```

### 常见统计错误

```markdown
## 统计错误

### 相关性vs因果性
❌ 错误：冰淇淋销量和溺水人数相关，所以吃冰淇淋导致溺水
✓ 正确：两者都与"夏天"相关

### 样本偏差
❌ 错误：只调查活跃用户，他们代表所有用户
✓ 正确：分层抽样，覆盖所有用户群

### 幸存者偏差
❌ 错误：只看留存用户，发现他们都喜欢我们的产品
✓ 正确：分析流失用户，了解为什么离开

### 辛普森悖论
❌ 错误：整体数据A方案优于B
✓ 正确：分群体看，B在每个群体都优于A
```

## A/B测试设计

### 测试设计模板

```markdown
## A/B测试设计

### 假设
[明确假设]

### 分组
| 组 | 样本比例 | 处理 |
|----|----------|------|
| 对照组 | 50% | 现有方案 |
| 实验组 | 50% | 新方案 |

### 指标
| 类型 | 指标 | 最小检测效应 |
|------|------|-------------|
| 主要 | 转化率 | +5% |
| 次要 | 留存率 | +2% |

### 样本量
- 统计功效：80%
- 显著性水平：5%
- 所需样本量：每组X

### 测试周期
- 最小：X天（覆盖完整用户周期）
- 推荐：X天
```

### 结果分析

```markdown
## 结果分析

### 数据
| 组 | 样本量 | 转化数 | 转化率 |
|----|--------|--------|--------|
| 对照组 | 10000 | 300 | 3.0% |
| 实验组 | 10000 | 360 | 3.6% |

### 统计检验
- 检验方法：Z检验
- Z值：2.14
- p值：0.032
- 置信区间：[0.3%, 0.9%]

### 结论
✅ 统计显著：实验组显著优于对照组
✅ 实际意义：转化率提升0.6pp，相对提升20%
```

## 检查清单

- [ ] 问题定义清晰
- [ ] 指标定义准确
- [ ] 数据来源可靠
- [ ] 分析方法适当
- [ ] 考虑统计显著性
- [ ] 权衡成本与收益

## 红牌警告

- 没有数据就做决策
- 把相关性当因果性
- 不考虑样本偏差
- 忽视统计显著性
- 只看短期忽视长期

## 好/坏对比

**好**：
```
决策流程：
1. 定义问题：注册转化率低
2. 收集数据：漏斗分析
3. 提出假设：引导流程太长
4. A/B测试验证
5. 决策：简化引导流程
→ 有数据支撑的决策
```

**坏**：
```
"我觉得应该这样做"
→ 没有数据分析
→ 拍脑袋决策
→ 效果难以评估
```
