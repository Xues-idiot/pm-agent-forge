# Product Strategy

产品战略规划。

## 何时使用

- 制定产品战略
- 产品愿景定义
- 市场定位
- 战略规划
- 战略复盘

## 战略基础

### 战略层次

```markdown
## 战略层次

### 战略层级
| 层级 | 时间 | 内容 |
|------|------|------|
| 公司战略 | 3-5年 | 公司方向 |
| 业务战略 | 1-3年 | 业务目标 |
| 产品战略 | 1-2年 | 产品方向 |
| 职能战略 | 1年内 | 执行计划 |

### 各层关系
```markdown
## 战略对齐

公司战略 → 业务战略 → 产品战略 → 职能战略

### 对齐要点
- 产品战略支撑业务战略
- 产品战略承接公司战略
- 职能战略落地产品战略
```
```

### 战略要素

```markdown
## 战略要素

### 经典战略要素
| 要素 | 说明 | 问题 |
|------|------|------|
| 愿景 | 长期目标 | 我们要成为什么 |
| 使命 | 存在的意义 | 我们做什么 |
| 价值观 | 行为准则 | 我们怎么做 |
| 战略目标 | 中期目标 | 我们往哪去 |
| 战术 | 执行手段 | 具体怎么做 |

### 好的战略特征
- 聚焦
- 可执行
- 有时限
- 可衡量
- 有取舍
```

### 差异化策略

```markdown
## 差异化策略

### 差异化维度
| 维度 | 方式 | 示例 |
|------|------|------|
| 产品差异化 | 功能/体验 | 独特功能 |

2. 评估差异化价值

4. 传播差异化定位
5. 维护差异化优势

## 产品战略

## 产品愿景

### 愿景定义
```markdown
## 愿景模板

在 [时间范围] 内
[目标用户] 可以通过 [产品]
实现 [核心价值]

例如：
3年内
中国的中小企业主
可以通过我们的CRM
高效管理客户关系
```

### 愿景特征
- 清晰易懂
- 有感召力
- 可衡量
- 有时间性
- 可实现但有挑战
```

```markdown

```

## 战略地图

### 四个维度
- 财务：如何实现股东价值
- 客户：如何赢得客户满意

## 战略执行

### 执行要点

```markdown
## 战略执行

### 执行要点
| 要点 | 说明 | 实践 |
|------|------|------|
| 聚焦 | 资源集中 | 砍需求 |
| 节奏 | 快迭代 | 小步快跑 |
| 协作 | 跨团队 | 对齐 |
| 复盘 | 持续改进 | 周/月review |

### 战略执行障碍
| 障碍 | 表现 | 解决 |
|------|------|------|
| 执行偏差 | 做的不一样 | 及时纠偏 |
| 资源不足 | 人力/预算 | 优先级 |
| 组织壁垒 | 部门墙 | 协作机制 |
| 目标模糊 | 不知道方向 | 沟通对齐 |
```

### 战略评估

```markdown
## 战略评估

### 评估维度
| 维度 | 指标 | 频率 |
|------|------|------|
| 财务 | 收入/利润 | 季度 |
| 客户 | 流失/满意度 | 月度 |
| 内部 | 效率/质量 | 月度 |
| 能力 | 团队/技术 | 半年度 |

### 战略复盘
- 战略回顾：季度
- 执行评估：月度
- 及时调整：按需
```

- [ ] 市场分析充分

- [ ] 产品愿景明确

## 红牌警告

- 愿景空洞
- 不做市场分析
- 竞争分析不足
- 战略不落地
- 目标不对齐
- 执行无跟踪
- 不做复盘
