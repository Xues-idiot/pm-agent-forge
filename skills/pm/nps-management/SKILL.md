## NPS基础

### NPS概念

```markdown
## NPS概念

### 定义
净推荐值（Net Promoter Score）是衡量客户忠诚度的指标

### 计算方法
| 角色 | 评分 | 说明 |
|------|------|------|
| 推荐者 | 9-10分 | 忠诚客户 |
| 被动者 | 7-8分 | 中立满意 |
| 贬损者 | 0-6分 | 不满意 |

### 计算公式
NPS = 推荐者% - 贬损者%

### 分数范围
| NPS | 等级 | 说明 |
|-----|------|------|
| 70+ | 卓越 | 世界级 |
| 50-69 | 优秀 | 领先 |
| 30-49 | 良好 | 合格 |
| 0-29 | 一般 | 需改进 |
| <0 | 糟糕 | 危机 |

### NPS vs CSAT
| 指标 | 区别 | 适用场景 |
|------|------|----------|
| NPS | 未来行为预测 | 战略指标 |
| CSAT | 当前满意度 | 运营指标 |
| CES | 费力程度 | 任务指标 |
```

## 改进闭环

### 闭环流程

```markdown
## 闭环流程

### 标准流程
收集反馈 → 分类分析 → 分发处理 → 改进落实 → 用户通知

### 各环节
| 环节 | 负责人 | 时效 |
|------|--------|------|
| 反馈收集 | 系统 | 实时 |
| 分类分析 | 运营 | 24h |
| 分发处理 | 责任部门 | 48h |
| 改进落实 | 产品/运营 | 按计划 |
| 用户通知 | 客服 | 改进后 |
```

### 处理优先级

```markdown
## 处理优先级

### P0：批量问题
- 影响大量用户
- NPS下降>10
→ 立即处理

### P1：重要问题
- 高价值用户投诉
- 竞品对比劣势
→ 48h内处理

### P2：一般问题
- 个别用户反馈
- 体验优化建议
→ 按计划处理
```
