# Product Experimentation

产品实验，设计实验和A/B测试。

## 何时使用

- A/B测试设计
- 实验方案制定
- 统计显著性分析
- 功能灰度发布
- 实验结果判断

## 实验设计

### 假设生成

```markdown
## 假设生成

### 假设模板
```markdown
## 实验假设

【假设】
如果[改动]
那么[预期结果]
因为[原因]

示例：
如果把购买按钮从蓝色改成绿色
那么购买转化率会提升5%
因为绿色更符合用户对"确认"的认知
```
```

### 假设来源

```markdown
## 假设来源

### 来源渠道
| 来源 | 示例 |
|------|------|
| 用户反馈 | 抱怨某流程复杂 |
| 数据洞察 | 发现某个漏斗转化低 |
| 竞品分析 | 竞品有某个功能 |
| 行业最佳实践 | 某设计被验证有效 |
| 直觉经验 | PM的判断 |
```

## 方案设计

### 分流设计

```markdown
## 分流设计

### 分流原则
- 随机分配
- 分层实验
- 互不干扰
- 样本代表性

### 分流方式
```markdown
## 分流方式

### 简单分流
- 对照组：50%
- 实验组A：50%

### 多元分流
- 对照组：50%
- 实验组A：25%
- 实验组B：25%

### 分层分流
层1：流量分配给不同实验
层2：同层内再分配
```
```

### 样本量计算

```markdown
## 样本量计算

### 影响因素
| 因素 | 说明 |
|------|------|
| 基准转化率 | 当前转化率 |
| 最小可检测差异 | MDE |
| 显著性水平 | α (通常0.05) |
| 统计功效 | 1-β (通常0.8) |

### 计算工具
- 在线样本量计算器
- G Power
- 统计软件

### 估算方法
```markdown
## 样本估算

### 经验公式
n = 16 × σ² / δ²

σ = 标准差
δ = 预期差异

### 粗略估算
| 转化率 | MDE=5% | MDE=10% |
|--------|--------|--------|
| 5% | 10万 | 3万 |
| 10% | 5万 | 1.5万 |
| 20% | 2.5万 | 8千 |
```
```

### 注意事项

```markdown
## 注意事项

### 常见错误
| 错误 | 问题 | 避免 |
|------|------|------|
| 新手效应 | 新用户影响大 | 只看老用户 |
| 时间效应 | 周末/工作日差异 | 至少跑一周 |
| 选择偏差 | 分流不随机 | 检查分布 |
| 多重检验 | 多次看结果 | 固定看结果时间 |

### 最佳实践
- 先看分布是否一致
- 先看guardrail指标
- 避免中途看结果
- 保持实验环境一致
```

## 实验报告

### 实验概况
- 实验名称：
- 样本量：对照组X vs 实验组Y

### 核心指标
| 指标 | 对照组 | 实验组 | 差异 | p值 |

### 统计显著性

```markdown
## 统计显著性

### 判断标准
| p值 | 结论 |
|-----|------|
| p < 0.01 | 非常显著 |
| p < 0.05 | 显著 |
| p >= 0.05 | 不显著 |

### 置信区间
| 置信度 | 说明 |
|--------|------|
| 95% | 常用标准 |
| 99% | 重要决策 |

### 效果量
```markdown
## 效果量

### 常用指标
| 指标 | 计算 | 解释 |
|------|------|------|
| 相对提升 | (实验-对照)/对照 | 百分比提升 |
| 绝对提升 | 实验-对照 | 百分点提升 |
| Cohen's d | 均值差/标准差 | 标准化效果 |
```
```

## 灰度发布

### 灰度策略

```markdown
## 灰度发布

### 灰度方式
| 方式 | 说明 | 适用 |
|------|------|------|
| 百分比灰度 | 按用户比例 | 高风险 |
| 地域灰度 | 按地区 | 地域特性 |
| 圈选灰度 | 按用户群 | 高价值用户 |
| 白名单灰度 | 手动指定 | 特殊用户 |

### 灰度节奏
```markdown
## 灰度节奏

10% → 30% → 50% → 100%

每阶段观察1-2天
确认无异常后放量
```
```

### 灰度监控

```markdown
## 灰度监控

### 监控指标
| 指标 | 关注 | 阈值 |
|------|------|------|
| 核心转化 | 不应下降 | <-5% |
| 错误率 | 不应上升 | >1% |
| 性能 | 不应下降 | <-10% |
| 客服投诉 | 不应上升 | 异常 |

### 异常处理
- 发现异常立即回滚
- 保留流量做进一步分析
- 总结经验教训
```

## 检查清单

- [ ] 假设清晰明确
- [ ] 方案设计合理
- [ ] 样本量足够
- [ ] 分流随机均匀
- [ ] 分析方法正确
- [ ] 结论谨慎

## 红牌警告

- 假设不清晰
- 样本量不够
- 提前看结果
- 只看正向指标
- 忽视负面指标
- 过分解读不显著结果

## 好/坏对比

**好**：
```
实验设计：
- 假设清晰
- 设计严谨
- 执行规范
- 分析客观
- 结论谨慎
→ 数据驱动、决策科学
```

**坏**：
```
实验设计：
- 假设模糊
- 样本不够
- 提前看结果
- 过度解读
- 忽视风险
→ 误导决策
```
