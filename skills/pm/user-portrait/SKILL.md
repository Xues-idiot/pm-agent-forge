# User Portrait Skill

用户画像构建，支持产品决策和精准运营。

## 何时使用

- 用户画像构建
- 用户分群分析
- 人物角色( Persona )设计
- 用户标签体系
- 数据驱动决策

## 画像构建流程

```
1. 数据采集 → 2. 标签体系 → 3. 画像构建 → 4. 应用落地 → 5. 迭代优化
```

### 1. 数据采集

**数据来源矩阵：**

```markdown
## 用户数据来源

### 基础属性
| 数据类型 | 来源 | 说明 |
|----------|------|------|
| 人口统计 | 注册信息 | 年龄/性别/地区 |
| 职业信息 | 认证信息 | 职位/公司/行业 |
| 设备信息 | SDK采集 | 手机/系统/网络 |

### 行为数据
| 数据类型 | 来源 | 说明 |
|----------|------|------|
| 访问行为 |埋点系统 | 页面/时长/路径 |
| 点击行为 | 埋点系统 | 点击/停留 |
| 交易行为 | 订单系统 | 购买/加购/收藏 |

### 偏好数据
| 数据类型 | 来源 | 说明 |
|----------|------|------|
| 内容偏好 | 行为推断 | 品类/风格 |
| 价格偏好 | 交易数据 | 客单价/频次 |
| 渠道偏好 | 触达数据 | 响应渠道 |
```

**数据采集规范：**

```markdown
## 采集规范

### 必采数据
- [ ] 用户ID（唯一标识）
- [ ] 基础属性（年龄/性别/地区）
- [ ] 首次访问时间
- [ ] 核心行为（关键动作）

### 隐私合规
```markdown
## 合规要求

### 采集前
- [ ] 用户授权告知
- [ ] 隐私政策公示
- [ ] 退出机制明确

### 采集中
- [ ] 敏感信息脱敏
- [ ] 数据加密存储
- [ ] 访问权限控制

### 采集后
- [ ] 数据使用告知
- [ ] 删除机制完善
```
```

### 2. 标签体系

**标签分类：**

```markdown
## 标签体系

### 属性标签
| 标签类型 | 示例 | 更新频率 |
|----------|------|----------|
| 基础属性 | 女/北京/80后 | 静态 |
| 职业特征 | 互联网/金融/白领 | 季度 |
| 消费等级 | 高/中/低 | 月度 |

### 行为标签
| 标签类型 | 示例 | 更新频率 |
|----------|------|----------|
| 活跃度 | 高/中/低 | 每日 |
| 购买力 | 强/中/弱 | 每周 |
| 参与度 | 积极/一般/沉默 | 每日 |

### 偏好标签
| 标签类型 | 示例 | 更新频率 |
|----------|------|----------|
| 品类偏好 | 美妆/数码/服饰 | 实时 |
| 价格带 | 低端/中端/高端 | 实时 |
| 内容偏好 | 短视频/图文/直播 | 实时 |

### 预测标签
| 标签类型 | 示例 | 说明 |
|----------|------|------|
| 流失风险 | 高/中/低 | 预测流失概率 |
| 付费倾向 | 高/中/低 | 预测付费可能 |
| 升级潜力 | 高/中/低 | 预测升级可能 |
```

**标签定义模板：**

```markdown

### 标签定义
[如何计算/判断这个标签]

### 标签值
[枚举值及定义]

### 数据来源

### 更新频率
[实时/每日/每周]

[可选：0-1之间的权重值]
```

### 3. 画像构建

**用户分群：**

```markdown
## 用户分群

### 常用分群维度
| 维度 | 分群 | 说明 |
|------|------|------|
| 生命周期 | 新用户/成长/成熟/沉默/流失 | 用户阶段 |
| 价值分层 | 高价值/中价值/低价值/负价值 | RFM分层 |
| 行为特征 | 活跃型/目的型/浏览型 | 行为模式 |
| 需求类型 | 价格敏感型/品质型/便利型 | 决策因素 |

### 分群计算
```markdown
## 分群规则

### 新用户定义
首次访问时间在[7]天内

### 高价值用户
近30天购买金额>[X]元 且 购买频次>[Y]次

### 流失风险用户
近[30]天无访问 且 近[60]天无购买
```

### 4. 应用落地

**应用场景：**

```markdown
## 画像应用

### 产品应用
| 场景 | 应用方式 |
|------|----------|
| 功能设计 | 围绕核心用户需求 |
| 优先级 | 多数用户需求优先 |
| UI设计 | 匹配用户使用习惯 |
| 运营策略 | 针对不同用户群 |

### 运营应用
| 场景 | 应用方式 |
|------|----------|
| 精准推送 | 按偏好标签定向 |
| 营销活动 | 按用户分群设计 |
| 流失召回 | 按流失风险召回 |
| 会员运营 | 按价值分层服务 |

### 推荐应用
| 场景 | 应用方式 |
|------|----------|
| 个性化推荐 | 标签匹配推荐 |
| 搜索排序 | 偏好加权排序 |
| 广告投放 | 精准人群定向 |
```
```

### 5. 迭代优化

**效果追踪：**

```markdown

### 效果指标
| 指标 | 计算 | 目标 |
|------|------|------|
| 覆盖率 | 有标签用户/总用户 | >90% |
| 准确率 | 抽样验证准确 | >80% |

| 应用率 | 实际应用场景 | 持续增加 |

### 迭代优化
```markdown
## 优化流程

1. 数据验证 → 2. 标签调整 → 3. 应用评估 → 4. 持续优化

### 优化方向
- [ ] 补充缺失数据
- [ ] 修正错误标签
- [ ] 调整权重参数
- [ ] 新增应用场景
```
```

## 用户画像检查清单

- [ ] 数据来源清晰
- [ ] 隐私合规达标
- [ ] 标签定义明确
- [ ] 分群规则合理
- [ ] Persona具体可感
- [ ] 应用场景落地
- [ ] 效果可衡量
- [ ] 持续迭代优化

## 红牌警告

- 数据来源不合法
- 隐私信息滥用
- 标签定义模糊
- 分群逻辑混乱
- 画像脱离实际
- 只建不用
- 从不迭代更新
- 歧视性标签
