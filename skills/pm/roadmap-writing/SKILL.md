# Product Roadmap Writing Skill

制定产品路线图，规划功能时间表，统一利益相关者对产品方向的理解。

## 何时使用

- 产品路线图制定
- 功能优先级排序
- 季度/年度规划
- 利益相关者对齐
- 技术债务规划

## 路线图制定流程

```
1. 战略对齐 → 2. 能力盘点 → 3. 优先级排序 → 4. 时间规划 → 5. 沟通确认
```

```markdown
|------|---------|--------|
| | | |

### 季度主题
| 季度 | 主题 | 核心成果 |
|------|------|---------|
| Q1 | | |
| Q2 | | |
| Q3 | | |
```


## 路线图格式选择

**按受众选择格式：**

| 受众 | 推荐格式 | 特点 |
|------|---------|------|
| 内部团队 | 甘特图/时间线 | 详细、时间明确 |
| 高管 | OKR映射 | 目标导向、结果可衡量 |
| 客户 | 主题式路线图 | 按需求分组、无具体时间 |
| 销售 | 功能里程碑 | 关键节点、竞争差异 |

## 路线图审查清单

- [ ] 与公司战略对齐
- [ ] 资源可行性评估
- [ ] 技术可行性确认
- [ ] 依赖关系梳理
- [ ] 风险识别
- [ ] 利益相关者认可
- [ ] 沟通计划到位

## 红牌警告

- 路线图过于详细（锁定太久）
- 未考虑技术债务
- 资源估算不现实
- 利益相关者未对齐
- 缺乏应急缓冲
- 频繁取消已承诺功能
