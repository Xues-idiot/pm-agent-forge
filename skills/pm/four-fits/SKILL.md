# Four Fits Assessment

Brian Balfour的四匹配框架，评估规模化增长准备度。

## 何时使用

- 评估是否可以规模化增长
- 诊断增长阻塞问题
- 规模化前的验证
- 新市场segment分析
- 增长策略制定

## 核心原则

> "四个匹配都必须对齐才能实现$100M+增长。匹配不对齐就规模化只会烧钱。" — Brian Balfour

```
四个匹配:
1. Market ↔ Product (市场-产品匹配)
2. Product ↔ Channel (产品-渠道匹配)
3. Channel ↔ Model (渠道-商业模式匹配)
4. Model ↔ Market (商业模式-市场匹配)

任一匹配断裂 → 不要规模化 → 先修复
```

---

## 四个匹配详解

### 1. Market ↔ Product

**问题**: 客户真的想要这个产品吗？

**评估问题**:
- 用户有痛点吗？
- 产品解决了这个痛点吗？
- 比现有解决方案好吗？
- 用户会持续使用吗？

**验证证据**:
| 证据 | 要求 |
|------|------|
| 用户访谈 | 10+访谈确认痛点 |
| 留存数据 | 周留存>40% |
| 付费意愿 | >30%用户愿意付费 |
| NPS | >30 |

**状态判定**:
| 状态 | 说明 |
|------|------|
| Validated | 有数据支持 |
| Needs Work | 有迹象但不确定 |
| Misaligned | 产品不满足市场需求 |

---

### 2. Product ↔ Channel

**问题**: 产品能通过渠道高效触达用户吗？

**评估问题**:
- 用户在这些渠道吗？
- 产品展示方式适合渠道吗？
- 渠道触达的用户质量高吗？
- 渠道容量够大吗？

**验证证据**:
| 证据 | 要求 |
|------|------|
| 渠道转化率 | 各阶段转化率>行业均值 |
| 用户质量 | 来自该渠道的用户留存高 |
| CAC | 在该渠道CAC可接受 |

**状态判定**:
| 状态 | 说明 |
|------|------|
| Validated | 渠道效率高 |
| Needs Work | 渠道有效但需优化 |
| Misaligned | 渠道用户不适合产品 |

---

### 3. Channel ↔ Model

**问题**: 商业模式能支持获客成本吗？

**评估问题**:
- CAC能小于LTV的1/3吗？
- 渠道获客成本可持续吗？
- 规模越大CAC会降低吗？
- 渠道效率能保持吗？

**验证证据**:
| 证据 | 要求 |
|------|------|
| LTV/CAC | >3 |
| 回本周期 | <12个月 |
| 规模化预期 | CAC随规模下降 |

**状态判定**:
| 状态 | 说明 |
|------|------|
| Validated | 单元经济健康 |
| Needs Work | 经济可行但不确定 |
| Misaligned | CAC太高无法持续 |

---

### 4. Model ↔ Market

**问题**: 商业模式在这个市场能盈利吗？

**评估问题**:
- 市场能承受这个定价吗？
- 商业模式适合这个客户群吗？
- 市场增长潜力够吗？
- 有定价空间吗？

**验证证据**:
| 证据 | 要求 |
|------|------|
| 定价验证 | 用户接受当前定价 |
| 客户终身价值 | 商业模式可持续 |
| 市场容量 | TAM足够大 |

**状态判定**:
| 状态 | 说明 |
|------|------|
| Validated | 商业模式适合市场 |
| Needs Work | 模式可行但需调整 |
| Misaligned | 模式不适合市场 |

## 红牌警告

- 没有验证Market↔Product就烧钱获客
- Product↔Channel没验证就all-in某渠道
- Channel↔Model单元经济不健康
- 四个都没验证就喊规模化

## 好/坏对比

**好**：
```
四匹配评估:
- Market↔Product: ✅ Validated (留存>50%)
- Product↔Channel: ✅ Validated (SEO渠道CAC低)
- Channel↔Model: ✅ Validated (LTV/CAC=5)
- Model↔Market: ✅ Validated (定价被接受)

结论: 可以规模化
```

**坏**：
```
发现:
- 产品留存好 (Market↔Product ✅)
- 但CAC太高 (Channel↔Model ❌)

结论: 不要烧钱，先修复单元经济
```
