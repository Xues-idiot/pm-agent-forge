# User Feedback Management Skill

用户反馈收集、分析和管理，推动产品持续改进。

## 何时使用

- 用户反馈收集
- 反馈分类分析
- 需求池管理
- 反馈驱动迭代
- 用户声音体系搭建

## 反馈管理流程

```
1. 收集渠道 → 2. 分类整理 → 3. 分析洞察 → 4. 需求转化 → 5. 闭环追踪
```

### 1. 收集渠道

**反馈来源矩阵：**

```markdown
## 反馈渠道

### 主动反馈
| 渠道 | 形式 | 特点 |
|------|------|------|
| 客服工单 | 结构化 | 清晰但滞后 |
| 问卷调研 | 结构化 | 可量化但主观 |
| 用户访谈 | 非结构化 | 深入但成本高 |
| 应用商店 | 非结构化 | 真实但分散 |

### 被动反馈
| 渠道 | 形式 | 特点 |
|------|------|------|
| App评价 | 非结构化 | 真实但噪声大 |
| 社交媒体 | 非结构化 | 传播快但分散 |
| 社区论坛 | 结构化 | 聚集但有限 |
| 投诉数据 | 结构化 | 紧急但片面 |

### 反馈收集SOP
```markdown
## 收集规范

### 必做项
- [ ] 各渠道反馈统一汇总
- [ ] 24小时内首次响应
- [ ] 72小时内初步分类
- [ ] 7天内给出处理方案

### 收集频率
| 渠道 | 频率 |
|------|------|
| 客服工单 | 每日 |
| 应用商店 | 每周 |
| 社交媒体 | 每日 |
| 调研问卷 | 每月 |
```
```

### 2. 分类整理

**反馈分类体系：**

```markdown
## 反馈分类

### 类型分类
| 类型 | 定义 | 处理优先级 |
|------|------|----------|
| Bug问题 | 功能异常 | P0 |
| 性能问题 | 速度/卡顿 | P1 |
| 需求建议 | 新功能请求 | P2 |
| 操作困惑 | 不知怎么用 | P1 |
| 内容问题 | 文字/图片错误 | P2 |
| 投诉抱怨 | 不满情绪 | P1 |
| 好评表扬 | 正面反馈 | 记录 |

### 紧急度分类
| 等级 | 定义 | 响应时间 |
|------|------|----------|
| P0 | 大面积无法使用 | 1小时 |
| P1 | 核心功能受影响 | 4小时 |
| P2 | 功能异常但可用 | 24小时 |
| P3 | 体验问题 | 72小时 |

### 3. 分析洞察

**反馈分析方法：**

```markdown
## 反馈分析

### 定量分析
| 指标 | 计算 | 用途 |
|------|------|------|
| 反馈量 | 每日/周/月 | 趋势判断 |
| 问题率 | 问题数/DAU | 严重程度 |
| NPS | 推荐-贬损 | 整体满意 |
| 好评率 | 好评/总数 | 口碑状况 |
| 渠道占比 | 各渠道反馈占比 | 渠道优化 |

### 定性分析
```markdown
## 主题挖掘

### 步骤
1. 汇总同类反馈
2. 提取关键词
3. 归类到主题
4. 排序影响力

### 输出
| 主题 | 反馈数 | 占比 | 优先级 |
|------|--------|------|--------|
| 加载慢 | 45 | 25% | P1 |
| 功能难用 | 38 | 21% | P1 |
| 缺少XX | 30 | 17% | P2 |
```

### 4. 需求转化

**反馈驱动迭代：**

```markdown
## 需求转化

### 转化标准
| 条件 | 说明 |
|------|------|
| 反馈数量 | 同类反馈>10条 |
| 影响用户 | 覆盖>5%用户 |
| 业务价值 | 对核心指标有贡献 |
| 实现成本 | 合理范围内 |

### 转化流程
```markdown
## 反馈→需求

反馈汇总 → 需求池 → 优先级评审 → PRD → 开发 → 上线 → 反馈用户

### 关键节点
- [ ] 需求池登记
- [ ] PM评估确认
- [ ] 优先级确认
- [ ] PRD编写
- [ ] 开发上线
- [ ] 反馈用户
```
```

### 5. 闭环追踪

**反馈闭环管理：**

```markdown
## 闭环追踪

### 响应时效
| 阶段 | 时效 | 动作 |
|------|------|------|
| 接收确认 | 24h | 感谢+收到 |
| 处理告知 | 72h | 告知方案 |
| 处理完成 | 计划时间内 | 通知用户 |
| 满意度回访 | 完成后1周 | 确认满意 |

### 用户通知模板
```markdown
## 反馈回复模板

### 采纳类
"感谢您的建议！您关于[建议内容]的反馈我们已经采纳，将在[版本/时间]和大家见面，感谢您的支持！"

### 暂不采纳类
"感谢您的反馈！关于您提到的[建议内容]，我们经过评估，目前暂无计划调整，感谢您的理解。"

### Bug修复类
"非常抱歉给您带来困扰！您反馈的[问题]我们已经修复，请在[版本]更新，感谢您的耐心等待。"
```
```

## 反馈管理检查清单

- [ ] 渠道覆盖完整
- [ ] 响应时效达标
- [ ] 分类标准统一
- [ ] 标签体系规范
- [ ] 分析报告定期输出
- [ ] 需求转化有追踪
- [ ] 闭环率>80%
- [ ] 用户满意度>80%

## 红牌警告

- 反馈石沉大海
- 虚假反馈回复
- 问题反复出现
- 投诉升级发酵
- 反馈数据失真
- 需求池长期积压
- 修复后不通知用户
- 重采纳轻落地
