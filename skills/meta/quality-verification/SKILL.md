# Quality Verification Skill

验证 Pi/Nu 质量和功能正确性的技能。

## 何时使用

- 新功能开发完成后验证
- 定期质量检查
- 用户反馈问题后复现和验证
- 重大改动后的回归测试

## 验证流程

```
1. 明确验收标准 → 2. 准备测试环境 → 3. 执行测试 → 4. 记录结果 → 5. 问题修复 → 6. 回归测试
```

### 1. 明确验收标准

从 PRD 或需求中提取验收标准：
```markdown
## 验收标准
- [ ] 用户可以创建新项目
- [ ] 项目列表显示所有项目
- [ ] 删除项目后列表更新
```

### 2. 准备测试环境

```bash
# Pi 后端
cd D:/PM-AI-Workstation/01-ai-agents/pm-assistant
npm run dev  # 后端 5000

# Nu 后端
cd D:/PM-AI-Workstation/01-ai-agents/private-operation-agent
uv run python main.py  # 后端 8000
```

### 3. 执行测试

**手动测试**：
- 按照验收标准逐项测试
- 记录实际结果
- 标记通过/失败

**自动化测试**：
```bash
# Pi
cd pm-assistant && npm test

# Nu
cd private-operation-agent && pytest
```

### 4. 记录结果

```markdown
## 测试报告 - [日期]

### 环境
- Node: v20.x
- Python: 3.11
- API: http://localhost:5000

### 测试结果

| 功能 | 预期 | 实际 | 状态 |
|------|------|------|------|
| 创建项目 | 返回项目对象 | 返回项目对象 | ✅ |
| 项目列表 | 3个项目 | 2个项目 | ❌ |

### 问题

**问题 1**: 项目列表少一个
- 预期：3个项目
- 实际：2个项目
- 原因：删除接口返回空数组
```

### 5. 问题修复

遵循 TDD（来自 Superpowers）：
1. **RED**: 写测试复现问题
2. **GREEN**: 最小代码修复
3. **REFACTOR**: 清理代码

### 6. 回归测试

修复后重新运行所有测试，确保没有引入新问题。

## Pi 专项验证

### 功能清单

| 功能 | 验证方法 |
|------|---------|
| PRD 生成 | 输入主题，验证输出格式 |
| 周报生成 | 输入工作内容，验证结构 |
| 竞品分析 | 输入产品名，验证分析维度 |
| 用户调研 | 验证问卷生成和问题分析 |

### 质量标准

- 输出格式符合模板
- 内容专业、有深度
- 无语法错误
- 逻辑连贯

## Nu 专项验证

### 功能清单

| 功能 | 验证方法 |
|------|---------|
| 用户分层 | 验证 RFM 分群结果 |
| 触达模拟 | 验证消息模板和发送逻辑 |
| 追踪仪表盘 | 验证数据更新和图表 |
| 活动效果 | 验证指标计算 |

### 质量标准

- 数据准确性
- 图表可读性
- 响应速度
- 交互流畅

## 问题分类

| 类别 | 描述 | 优先级 |
|------|------|--------|
| **P0-Bug** | 核心功能完全不工作 | 立即修复 |
| **P1-Bug** | 功能有问题但部分可用 | 24h内修复 |
| **P2-缺陷** | 非核心功能问题 | 计划修复 |
| **P3-优化** | 体验改进 | 持续优化 |

- [ ] 手动测试完成
- [ ] 性能可接受
