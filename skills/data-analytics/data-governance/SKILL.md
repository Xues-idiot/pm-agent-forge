# Data Governance Skill

建立数据治理框架，管理数据资产，确保数据合规和安全。

## 何时使用

- 数据治理规划
- 数据资产管理
- 数据标准制定
- 数据质量管理
- 数据安全合规
- 数据架构优化

## 数据治理流程

```
1. 治理规划 → 2. 体系建设 → 3. 标准制定 → 4. 执行落地 → 5. 监控改进
```

### 1. 治理规划

**数据资产盘点：**

```markdown
## 数据资产盘点

### 盘点维度
| 资产类型 | 内容 | 负责人 | 敏感级别 |
|----------|------|--------|----------|
| 用户数据 | 用户信息/行为 | | 高 |
| 业务数据 | 订单/交易 | | 高 |
| 产品数据 | 商品/库存 | | 中 |
| 日志数据 | 行为日志 | | 低 |
| 第三方数据 | 外部数据 | | 中 |

### 资产分级
| 级别 | 定义 | 示例 | 保护要求 |
|------|------|------|----------|
| P4-高度敏感 | 机密 | 用户密码/支付信息 | 加密+严格权限 |
| P3-敏感 | 内部使用 | 用户画像/经营数据 | 加密+访问控制 |
| P2-内部 | 内部共享 | 统计数据/报表 | 访问控制 |
| P1-公开 | 可对外 | 公开数据/新闻 | 无限制 |
```

**组织架构：**

```markdown
## 数据治理组织

### 角色职责
| 角色 | 职责 | 部门 |
|------|------|------|
| 数据Owner | 数据资产责任人 | 业务部门 |
| 数据管理员 | 日常管理 | IT/业务 |
| 数据分析师 | 数据使用 | 数据部门 |
| 数据安全官 | 安全合规 | 合规部门 |

### 治理委员会
- 数据治理委员会：决策重大事项
- 数据标准组：制定标准
- 数据安全组：安全合规
- 数据质量组：质量管理
```

### 4. 执行落地

**数据安全落地：**

```markdown
## 数据安全

### 访问控制
| 措施 | 要求 | 实施 |
|------|------|------|
| 权限最小化 | 按需授权 | |
| 定期审计 | 季度审查 | |
| 离职回收 | 离职当天 | |
| 紧急权限 | 临时申请+审批 | |

### 数据脱敏
| 数据类型 | 脱敏方式 | 示例 |
|----------|----------|------|
| 手机号 | 加密/掩码 | 138****5678 |
| 身份证 | 掩码 | 310***********1234 |
| 银行卡 | 掩码 | ****1234 |
| 地址 | 模糊化 | 上海市浦东新区 |
```

**数据生命周期：**

```markdown
## 数据生命周期

### 保留策略
| 数据类型 | 保留周期 | 原因 |
|----------|----------|------|
| 行为日志 | 2年 | 业务分析 |
| 交易数据 | 永久 | 法规要求 |
| 用户资料 | 注销后180天 | 法规要求 |
| 临时数据 | 用完即删 | 安全 |
| 备份数据 | 30天 | 恢复目的 |

### 销毁流程
1. 申请销毁
2. 审批确认
3. 执行销毁
4. 记录归档
```

## 红牌警告

- 敏感数据泄露
- 数据标准不统一
- 数据质量低下
- 权限管理混乱
- 合规违规
- 数据滥用
- 标准形同虚设
- 治理责任不清
