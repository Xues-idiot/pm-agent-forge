## 样本量计算

### 样本量公式

```markdown
## 样本量计算

### 公式
n = (Zα + Zβ)² × (p1(1-p1) + p2(1-p2)) / (p2-p1)²

### 参数说明
| 参数 | 含义 | 典型值 |
|------|------|--------|
| Zα | 显著性水平(0.05) | 1.96 |
| Zβ | 统计功效(0.8) | 0.84 |
| p1 | 对照组转化率 | 5% |
| p2 | 期望转化率 | 5.5% |
| 最小检测效应 | MDE | +10% |

### 快速估算表
| p1 | MDE | 每组样本 |
|----|-----|----------|
| 5% | 10% | 15,000 |
| 5% | 20% | 4,000 |
| 10% | 10% | 15,000 |
| 10% | 20% | 4,000 |
```

### 计算工具

```markdown
## 样本量计算器

可使用以下工具：
- Evan Miller calculator
- Optimizely calculator
- R: power.prop.test()
- Python: statsmodels.stats.power

### 示例代码
```python
from statsmodels.stats.power import NormalIndPower
from statsmodels.stats.proportion import proportions_ztest

# 计算样本量
power = NormalIndPower()
n = power.solve_power(
    effect_size=0.1,  # 10%相对提升
    alpha=0.05,
    power=0.8,
    ratio=1.0
)
print(f"每组需要: {int(n)} 样本")
```
```

## 统计分析

### 统计检验

```markdown
## 统计检验

### Z检验（比例）
```python
from statsmodels.stats.proportion import proportions_ztest
import numpy as np

# 数据
conversions = np.array([300, 360])  # 对照组、实验组
nobs = np.array([10000, 10000])     # 样本量

# Z检验
z_stat, p_value = proportions_ztest(conversions, nobs)
```
```

### 结果解读

```markdown
## 结果解读

### 统计显著性
| p值 | 结论 | 行动 |
|-----|------|------|
| p < 0.05 | 显著 | 可上线 |
| 0.05 ≤ p < 0.1 | 边缘显著 | 继续观察 |
| p ≥ 0.1 | 不显著 | 需改进 |

### 置信区间
| 区间 | 含义 |
|------|------|
| 包含0 | 不显著 |
| 全正 | 显著正向 |
| 全负 | 显著负向 |

### 效应量
| 效应量 | 含义 |
|--------|------|
| < 1% | 微小 |
| 1-5% | 小 |
| 5-10% | 中 |
| > 10% | 大 |
```

## 测试常见问题

### 样本比率

```markdown
## 样本比率不均

### 正常情况
| 分流 | 比例 | 影响 |
|------|------|------|
| 50/50 | 50%/50% | 无 |
| 40/60 | 40%/60% | 可接受 |
| 10/90 | 10%/90% | 需分析 |

### 问题处理
- 检查分流逻辑
- 验证随机性
- 考虑时间因素
```

### 新奇效应

```markdown
## 新奇效应

### 定义
新版本带来新鲜感，短期内指标提升，长期回归

### 识别方法
- 看测试后期数据
- 长期跟踪指标
- 分群体分析

### 应对策略
- 延长测试周期
- 分新/老用户分析
- 谨慎决策
```

### 节假日影响

```markdown
## 节假日影响

### 问题
节假日用户行为异常，结果可能失真

### 识别
- 对照组也受影响
- 历史同期数据对比

### 建议
- 避开重大节日
- 延长测试周期
- 分工作日/节假日分析
```

## 检查清单

- [ ] 测试假设明确

## 红牌警告

- 不计算样本量就测试
- 看到正向就停止
- 不考虑新奇效应
- 只看统计显著不管实际意义
- 测试周期太短

## 好/坏对比

**好**：
```
测试设计：
- 假设明确：改标题→CTR提升
- 样本量计算：每组15000
- 测试周期：14天
- 统计检验：Z检验
→ 结论可靠
```

**坏**：
```
测试设计：
- 没有假设
- 每组500样本
- 测3天
- 看到+5%就上线
→ 可能误判
```
