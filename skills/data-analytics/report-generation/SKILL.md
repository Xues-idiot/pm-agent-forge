# Data Analytics Report Generation Skill

生成结构化的数据分析报告。

## 何时使用

- 需要撰写分析报告
- 汇报数据分析结果
- 定期数据报告
- 决策支持报告
- 数据驱动的提案

## 报告生成流程

```
1. 明确需求 → 2. 数据获取 → 3. 分析执行 → 4. 撰写报告 → 5. 审查优化 → 6. 输出交付
```

### 1. 明确需求

**需求收集：**

```markdown
## 报告需求

### 基本信息
- 报告标题：
- 报告类型：[定期/专项/决策]
- 目标读者：
- 截止时间：

### 分析要求
- 核心问题：
- 关注指标：
- 分析维度：
- 期望结论：

### 交付要求
- 格式：[PPT/Word/PDF/邮件]
- 页数：
- 语言：[中文/英文]
```

### 3. 分析执行

**分析框架：**

```markdown
## 分析框架

### 描述性分析
- 现状是什么？
- 关键指标数值？

### 诊断性分析
- 为什么是这样？
- 关键影响因素？

### 预测性分析
- 未来趋势如何？
- 预期目标能否达成？

### 建议性分析
- 应该怎么做？
- 优先采取什么行动？
```

### 6. 输出交付

**交付格式：**

| 格式 | 适用场景 | 工具 |

## 报告生成检查清单

- [ ] 需求理解准确
- [ ] 数据获取完整
- [ ] 分析方法正确
- [ ] 报告结构完整
- [ ] 数据准确无误
- [ ] 图表有效支撑
- [ ] 结论有数据依据
- [ ] 建议可执行

## 红牌警告

- 数据来源不明确
- 数据与分析结论不一致
- 样本偏差导致结论错误
- 过度解读相关性为因果性
- 建议缺乏数据支撑
- 忽略数据局限性
- 报告结构混乱难读
