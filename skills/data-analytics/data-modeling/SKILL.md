# 数据建模 (Data Modeling)

## Overview

数据建模是设计数据库结构或数据仓库模型的技能。涵盖概念模型、逻辑模型、物理模型、星型/雪花模型等核心环节。

## 何时使用

- 新系统数据库设计
- 数据仓库建设
- BI报表开发
- 数据架构优化

## 流程

1. **需求分析**：理解业务需求和数据使用
2. **概念模型**：设计实体和关系
3. **逻辑模型**：定义表结构和字段
4. **物理模型**：优化性能和维护性
5. **命名规范**：制定命名规范
6. **评审发布**：评审后实施
7. **文档维护**：维护数据字典

```
  - 普通索引: phone, create_time

## 命名规范
- 表命名: [业务模块_表含义]

```

## 检查清单

- [ ] 实体关系设计合理
- [ ] 主键外键关系正确
- [ ] 字段类型选择恰当
- [ ] 索引设计优化查询
- [ ] 命名规范统一
- [ ] 有数据字典文档

## 红牌警告

- 实体关系混乱
- 缺乏主键设计
- 字段类型选择不当
- 索引过多影响写入
- 缺乏文档传承
