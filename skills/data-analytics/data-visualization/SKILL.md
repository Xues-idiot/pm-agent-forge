# Data Visualization

数据可视化，创建图表、设计看板和展示数据洞察。

## 何时使用

- 创建数据报表
- 设计数据看板
- 选择图表类型
- 展示数据洞察
- 优化可视化效果

## 图表选择

### 按数据类型

```markdown
## 图表选择

### 适用数据类型

| 数据类型 | 推荐图表 | 说明 |
|----------|----------|------|
| 趋势数据 | 折线图、面积图 | 展示随时间变化 |
| 分类对比 | 柱状图、条形图 | 比较不同类别 |
| 占比构成 | 饼图、环形图 | 展示部分占比 |
| 相关关系 | 散点图 | 展示变量关系 |
| 分布情况 | 直方图、箱线图 | 展示数据分布 |
| 地理数据 | 地图、热力图 | 展示地理分布 |
```

### 图表选择决策树

```markdown
## 图表选择决策

用户想了解？
    │
    ├─ 时间变化 → 折线图
    │
    ├─ 类别对比 → 柱状图
    │
    ├─ 部分占比 → 饼图(≤5类) / 柱状图(>5类)
    │
    ├─ 相关关系 → 散点图
    │
    └─ 地理分布 → 地图
```

### 常见错误

```markdown
## 图表常见错误

### ❌ 饼图问题
- 类别太多(>7类)
- 占比接近难以区分
- 3D饼图扭曲比例

### ❌ 柱状图问题
- Y轴不从0开始
- 柱状图用于趋势
- 条形顺序混乱

### ❌ 折线图问题
- 数据点太少(<4个)
- 线条太多(>5条)
- Y轴刻度误导
```

## 图表设计原则

### 基础原则

```markdown
## 图表设计原则

### 简洁性
✓ 只展示必要信息
✓ 删除装饰元素
✓ 去除冗余标签

### 准确性
✓ Y轴从0开始(除非标注)
✓ 比例准确
✓ 不截断数据

### 可读性
✓ 标题清晰
✓ 图例完整
✓ 标注关键数据

### 美观性
✓ 配色协调
✓ 留白适当
✓ 字体易读
```

### 配色方案

```markdown
## 配色方案

### 单色系
- 深浅递进：强调→弱化
- 适用：同系列数据

### 对比色系
- 红/绿：好/坏
- 蓝/橙：冷/暖
- 适用：对立关系

### 品牌色系
- 主色+辅色
- 保持一致

### 色盲友好
- 避免红/绿
- 使用蓝/橙
- 加纹理区分
```

### 字体与排版

```markdown
## 排版规范

### 字体
- 标题：18-24px，加粗
- 坐标轴：10-12px
- 标签：9-11px
- 建议使用：Arial, Helvetica

### 间距
- 图表内边距：10-15%
- 元素间距：5-10px
- 标题与图表：15-20px
```

## 常用图表模板

### 折线图

```markdown
## 折线图

### 适用场景
- 趋势变化
- 时间序列
- 周期波动

### 模板要素
| 要素 | 内容 |
|------|------|
| 标题 | 清晰描述主题 |
| 坐标轴 | X:时间，Y:数值+单位 |
| 数据线 | 线条+数据点 |
| 图例 | 多系列时需要 |
| 标注 | 关键点标注 |

### 示例标题
"XX产品日活用户趋势 (2024年1-6月)"

### 优化技巧
- 数据点过多可用平滑曲线
- 多系列用不同线型区分
- 重要转折点加标注
```

### 柱状图

```markdown
## 柱状图

### 适用场景
- 类别对比
- 排名展示
- 目标对比

### 模板要素
| 要素 | 内容 |
|------|------|
| 排序 | 从大到小/时间顺序 |
| 数值标签 | 柱顶显示数值 |
| 基准线 | 目标线/平均线 |
| 颜色 | 区分/强调 |

### 示例标题
"各渠道获客数量对比 (2024年6月)"

### 优化技巧
- 柱子间距为柱宽的50-100%
- 数值标签放在柱顶
- 超过10类用横向柱状图
```

### 饼图

```markdown
## 饼图

### 适用场景
- 单一变量占比
- 类别≤5类
- 不需要精确比较

### 模板要素
| 要素 | 内容 |
|------|------|
| 标签 | 类别名+百分比 |
| 图例 | 颜色说明 |
| 标题 | 清晰描述 |

### 示例标题
"用户来源渠道分布"

### 优化技巧
- 从12点钟方向顺时针排列
- 最大扇形从12点开始
- 突出显示关键扇形
```

### 漏斗图

```markdown
## 漏斗图

### 适用场景
- 转化漏斗
- 流程步骤
- 逐级流失

### 模板要素
| 要素 | 内容 |
|------|------|
| 步骤 | 从上到下排列 |
| 数值 | 每步人数/金额 |
| 转化率 | 步与步之间 |
| 流失颜色 | 渐变标识 |

### 示例标题
"订单转化漏斗"

### 数据格式
| 阶段 | 数量 | 转化率 |
|------|------|--------|
| 访问 | 10000 | - |
| 加购 | 5000 | 50% |
| 下单 | 2000 | 40% |
| 支付 | 1500 | 75% |
```

### 仪表盘

```markdown
## 仪表盘设计

### 布局原则
```
┌─────────────────────────────────┐
│  标题栏(核心指标)                │
├───────────────┬─────────────────┤
│   指标卡1     │   指标卡2        │
│   KPI数值     │   KPI数值        │
├───────────────┼─────────────────┤
│   主图表      │   副图表         │
│   (折线图)    │   (柱状图)       │
├───────────────┴─────────────────┤
│   明细表格                          │
└─────────────────────────────────┘
```

### 配色分区
| 区域 | 颜色 | 含义 |
|------|------|------|
| 正向指标 | 绿色 | 好 |
| 负向指标 | 红色 | 差 |
| 中性指标 | 蓝/灰 | 中性 |
| 预警区域 | 黄色 | 注意 |

## 红牌警告

- 图表类型选错
- 数据与图表不匹配
- Y轴截断误导
- 颜色使用混乱
- 信息过载

## 好/坏对比

**好**：
```
图表设计：
- 标题："日活用户趋势"
- 折线图：清晰展示变化
- 配色：统一品牌色
- 标注：关键点有说明
→ 一眼看出趋势
```

**坏**：
```
图表设计：
- 标题："数据"
- 3D饼图：扭曲比例
- 图例缺失
- 颜色乱用
→ 看不出重点
```
