# Dashboard Design Skill

设计数据看板，创建可视化报告，规划分析展示。

## 何时使用

- 数据看板搭建
- 可视化设计
- 报表规划
- 数据故事
- 管理驾驶舱

## 数据看板设计流程

```
1. 需求分析 → 2. 指标梳理 → 3. 布局设计 → 4. 可视化实现 → 5. 迭代优化
```

### 2. 指标梳理

**指标体系规划：**

```markdown
## 指标体系

### 按层级分类
| 层级 | 指标类型 | 示例 | 更新频率 |
|------|-----------|------|----------|
| 战略层 | 北极星指标 | GMV, 用户数 | 日/周 |

| 执行层 | 细分指标 | 渠道, 品类 | 按需 |

### 核心指标选择（北极星指标）
- [ ] 指标1个，聚焦最重要
- [ ] 与业务目标强相关
- [ ] 可衡量可追踪
```

### 指标定义文档

```markdown
## 指标定义

### 指标详情模板
```markdown
## 指标：日活跃用户数（DAU）

### 基本定义
- 指标名称：DAU
- 计算公式：COUNT(DISTINCT user_id) WHERE date = 某日
- 数据来源：用户行为日志
- 统计周期：每日0点更新

### 业务定义
- 活跃定义：当天有任意行为的用户
- 包括：浏览、点击、下单等
- 不包括：仅注册未激活

### 限定条件
- 时间范围：当天0点-24点
- 用户范围：全量用户
- 排除：测试账号、内部用户
```
```

### 3. 布局设计

**看板布局原则：**

```markdown
## 布局原则

### 重要程度
- 左上：最重要（核心KPI）
- 右上：次重要（趋势对比）
- 中部：详情数据
- 下部：支撑信息

### 阅读习惯
- F型/ Z型 浏览习惯
- 关键信息靠上靠左
- 详情信息靠下靠右

### 留白设计
- 模块间距：16-24px
- 内边距：16px
- 不要过于拥挤
```

### 5. 迭代优化

**看板迭代检查：**

```markdown
## 迭代检查

### 用户反馈
- [ ] 用户能快速找到关键信息
- [ ] 数据更新及时
- [ ] 图表类型合适
- [ ] 交互流畅

### 数据质量
- [ ] 数据准确无误
- [ ] 计算逻辑正确
- [ ] 异常数据标注

### 性能优化
- [ ] 加载时间 < 3秒
- [ ] 无大量请求
- [ ] 缓存策略合理
```

## 数据看板检查清单

- [ ] 用户需求明确
- [ ] 指标选择合理
- [ ] 布局清晰易读
- [ ] 配色专业协调
- [ ] 图表类型合适
- [ ] 交互友好
- [ ] 性能达标
- [ ] 文档完善

## 红牌警告

- 指标过多无重点
- 数据不准确
- 更新延迟大
- 图表类型误用
- 配色混乱
- 加载过慢
- 缺乏异常标注
- 用户看不懂
