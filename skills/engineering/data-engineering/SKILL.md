# Data Engineering

数据工程与流水线设计。

## 何时使用

- 数据流水线设计
- 数据仓库建设
- ETL流程开发
- 数据质量保障
- 大数据处理

## 数据架构

### 数据架构分层

```markdown
## 数据架构

### 分层设计
| 层级 | 内容 | 技术 |
|------|------|------|
| ODS | 原始数据 | 日志/业务库 |
| DW | 数据仓库 | Hive/Spark |
| DM | 数据集市 | ClickHouse |
| APP | 应用层 | BI/报表 |

### 数据流向
```markdown
## 数据流向

业务库 → CDC → Kafka → ODS → 清洗 → DW → 聚合 → DM → 应用

### 各层职责
- ODS：原始数据保留
- DW：清洗转换建模
- DM：业务指标汇总
- APP：可视化分析
```
```

### Lambda架构
```markdown
## Lambda架构

### 层次
- Batch Layer：全量计算
- Speed Layer：实时增量
- Serving Layer：合并查询

### 适用场景
- 需要实时+历史
- 允许轻微延迟
- 查询复杂度高
```

```markdown
## 数据质量

### 六大维度
| 维度 | 说明 | 检测 |
|------|------|------|
| 完整性 | 数据不缺 | 非空检查 |

### 监控指标
| 指标 | 计算 | 标准 |
|------|------|------|
| 完整率 | 非空/总数 | >99% |
| 准确率 | 正确/总数 | >99% |
| 一致率 | 一致/总数 | >99% |
