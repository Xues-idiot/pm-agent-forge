# Performance Optimization

性能优化，提升系统性能。

## 何时使用

- 系统响应慢
- 性能瓶颈定位
- 高并发优化
- 缓存策略设计
- 性能测试调优

## 性能基础

### 性能指标

```markdown
## 性能指标

### 响应时间
| 指标 | 定义 | 目标 |
|------|------|------|
| 平均响应 | 请求耗时均值 | <200ms |
| P50响应 | 中位数响应时间 | <150ms |
| P95响应 | 95%请求耗时 | <500ms |
| P99响应 | 99%请求耗时 | <1s |

### 系统指标
| 指标 | 说明 | 目标 |
|------|------|------|
| CPU使用率 | 处理器占用 | <70% |
| 内存使用率 | 内存占用 | <80% |
| 磁盘IO | 读写速度 | IOPS合理 |
| 网络带宽 | 网络吞吐 | <50% |

### 业务指标
| 指标 | 说明 | 目标 |
|------|------|------|
| QPS | 每秒查询数 | 满足业务 |
| TPS | 每秒事务数 | 满足业务 |
| 并发用户 | 同时在线 | 预估峰值 |
| 错误率 | 失败比例 | <0.1% |
```

## 瓶颈定位

### 分析方法
| 方法 | 工具 | 适用 |

### 定位流程
```markdown
## 定位流程

1. 确定问题
   - 用户反馈响应慢
   - 监控发现异常
   - 测试发现瓶颈

2. 收集证据
   - 查看监控指标
   - 分析链路追踪
   - 提取日志

3. 分析原因
   - CPU瓶颈
   - 内存瓶颈
   - IO瓶颈
   - 网络瓶颈
   - 代码瓶颈

4. 验证假设
   - 本地复现
   - 灰度验证
   - A/B测试
```

### 常见瓶颈

```markdown
## 常见瓶颈

### CPU瓶颈
| 症状 | 原因 | 解决 |
|------|------|------|
| CPU高 | 死循环/计算密集 | 算法优化 |

| 症状 | 原因 | 解决 |
|------|------|------|
| OOM | 内存泄漏 | 代码排查 |

| 内存不足 | 资源耗尽 | 资源限制 |

### IO瓶颈
| 类型 | 症状 | 解决 |
|------|------|------|
| 磁盘IO | IO等待高 | SSD/索引优化 |

```

## 缓存策略

### 缓存类型

```markdown
## 缓存类型

### 多级缓存
| 级别 | 介质 | 延迟 | 容量 |
|------|------|------|------|
| L1缓存 | CPU Cache | ns级 | KB级 |
| L2缓存 | 本地内存 | 10ns级 | MB级 |
| L3缓存 | 分布式缓存 | 100ns级 | GB级 |
| L4缓存 | 数据库 | ms级 | TB级 |

### 缓存选择
| 缓存 | 特点 | 适用场景 |
|------|------|----------|
| Redis | KV/丰富数据结构 | 热点数据 |
| Memcached | 纯KV | 页面缓存 |
| 本地Cache | 超低延迟 | 单机热点 |
| CDN | 静态资源 | 媒体文件 |
```

### 缓存策略

```markdown
## 缓存策略

### 读写策略
| 策略 | 读 | 写 | 缺点 |
|------|------|------|------|
| Cache Aside | 先读缓存 | 先写DB | 缓存miss |
| Read Through | 读Cache | 写DB | 实现复杂 |
| Write Through | 写Cache | 写DB | 写入慢 |
| Write Behind | 写Cache | 异步写DB | 数据丢失 |

### 过期策略
| 策略 | 实现 | 特点 |
|------|------|------|
| TTL | 固定过期 | 简单 |
| LRU | 最近最少使用 | 内存敏感 |
| LFU | 最不经常使用 | 频率敏感 |
| 主动过期 | 事件触发 | 实时 |

### 缓存问题
| 问题 | 原因 | 解决 |
|------|------|------|
| 缓存穿透 | 恶意请求 | 布隆过滤器 |
| 缓存击穿 | 热键过期 | 永不过期 |
| 缓存雪崩 | 大量过期 | 随机TTL |
| 数据不一致 | 并发更新 | 分布式锁 |
```

## 并发优化

### 线程池

```markdown
## 线程池优化

### 参数配置
| 参数 | 公式 | 说明 |
|------|------|------|
| 核心线程 | CPU*2 | 保持活跃 |
| 最大线程 | CPU*2 | 突发处理 |
| 队列长度 | QPS*RT | 缓冲能力 |
| 拒绝策略 | 拒绝策略 | 保护系统 |

### 优化建议
- IO密集：线程数=CPU/(1-阻塞系数)
- CPU密集：线程数=CPU+1
- 避免线程过多
- 合理设置队列长度
```

## 异步处理
