# 缓存策略 (Caching Strategy)

## Overview

缓存策略是通过设计多级缓存体系提升系统访问性能的技能。涵盖缓存设计、缓存失效、更新策略、分布式缓存等核心环节。

## 何时使用

- 数据库压力过大
- 热点数据访问延迟高
- 需要提升系统吞吐
- 多级缓存架构设计

## 流程

1. **热点分析**：分析数据访问模式
2. **缓存设计**：确定缓存层次和选型
3. **Key设计**：设计缓存Key规则
4. **失效策略**：设计过期和更新策略
5. **一致性**：保证缓存与数据库一致
6. **监控**：监控缓存命中率和性能
7. **优化**：持续优化缓存效果

## 模板

### 缓存设计模板
```
## 缓存层次
- L1缓存: [本地缓存，如Caffeine/Guava Cache]
- L2缓存: [分布式缓存，如Redis]
- 命中率目标: >[X]%

## 缓存Key设计
- 格式: [业务前缀]:[业务标识]:[维度]
- 示例: user:info:{userId}

## 失效策略
| 数据类型 | TTL | 更新策略 |
|----------|-----|----------|
| 用户信息 | 30min | Cache Aside |
| 配置信息 | 24h | 主动刷新 |
| 热点数据 | 5min | LRU淘汰 |

## 缓存一致性问题
- 解决方案: [延迟双删/订阅binlog/直接删除]
- 主从延迟处理: [策略]
```

## 检查清单

- [ ] 缓存Key设计规范，避免冲突
- [ ] TTL设置合理，避免脏数据
- [ ] 有缓存穿透保护
- [ ] 有缓存雪崩防护
- [ ] 缓存与数据库一致性有保障
- [ ] 监控指标完善

## 红牌警告

- 缓存Key冲突导致数据错乱
- TTL设置过长导致数据过期
- 缺乏缓存保护导致雪崩
- 缓存与数据库不一致
