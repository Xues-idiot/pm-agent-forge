# 负载均衡设计 (Load Balancing)

## Overview

负载均衡设计是将流量均匀分布到多个服务节点，提升系统可用性和性能的技能。涵盖负载算法、健康检查、流量调度、故障转移等核心环节。

## 何时使用

- 单机性能无法满足需求
- 需要消除单点故障
- 多机房部署流量调度
- 服务弹性伸缩

## 流程

1. **场景分析**：分析流量特征和需求
2. **方案选型**：选择硬件/软件负载均衡
3. **负载算法**：设计流量分配策略
4. **健康检查**：设计节点健康检测
5. **故障处理**：设计故障转移机制
6. **监控运维**：监控负载和性能
7. **优化迭代**：持续优化调度策略

## 检查清单

- [ ] 负载算法选择合理
- [ ] 健康检查配置正确
- [ ] 有故障自动转移能力
- [ ] 会话保持满足业务需求
- [ ] 监控指标完善
- [ ] 有熔断降级机制

## 红牌警告

- 健康检查配置不当导致误切
- 负载不均导致部分节点过载
- 缺乏故障转移导致服务不可用
- 会话保持失效导致用户掉线
