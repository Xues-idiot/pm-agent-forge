# 事件驱动架构 (Event-Driven Architecture)

## Overview

事件驱动架构是通过事件解耦系统组件，实现异步通信和松耦合的技能。涵盖事件设计、消息中间件、事件溯源、CQRS等核心模式。

## 何时使用

- 系统间需要异步通信
- 需要事件溯源能力
- 微服务间强解耦
- 需要最终一致性
- 事件驱动业务场景

## 流程

1. **事件设计**：设计事件模型和类型
2. **中间件选型**：选择Kafka/RocketMQ等
3. **模式选择**：确定ES/CQRS模式
4. **消费者设计**：设计事件处理逻辑
5. **可靠性设计**：保证消息可靠投递
6. **监控运维**：监控消息延迟和积压
7. **优化迭代**：优化吞吐和延迟

## 检查清单

- [ ] 事件模型设计合理
- [ ] Topic分区规划合理
- [ ] 消息可靠性有保障
- [ ] 消费者幂等处理
- [ ] 有死信队列处理
- [ ] 监控消息积压和延迟

## 红牌警告

- 事件模型设计混乱
- 消息丢失无补偿机制
- 消费者幂等处理缺失
- Topic分区过少导致吞吐不足
- 缺乏监控无法发现消息积压
