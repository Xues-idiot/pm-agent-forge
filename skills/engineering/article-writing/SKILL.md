# Article Writing

技术文章写作，编写技术文档和博客内容。

## 何时使用

- 编写技术博客
- 撰写API文档
- 编写教程
- 创建技术指南
- 撰写发布说明

## 文章结构

### 常见结构

```markdown
## 技术文章结构

### 教程类
| 部分 | 内容 |
|------|------|
| 引言 | 介绍要解决的问题 |
| 前提 | 需要的知识和工具 |
| 步骤 | 逐步实现 |
| 总结 | 回顾要点 |
| 延伸 | 下一步学习 |

### 解释类
| 部分 | 内容 |
|------|------|
| 概述 | 是什么 |
| 原理 | 工作原理 |
| 使用 | 如何使用 |
| 注意事项 | 坑和最佳实践 |
| 参考 | 相关链接 |

### 最佳实践类
| 部分 | 内容 |
|------|------|
| 问题 | 常见问题 |
| 方案 | 解决方案 |
| 代码 | 示例代码 |
| 对比 | 方案对比 |
| 建议 | 推荐方案 |
```

## 写作技巧

### 标题技巧

```markdown
## 标题技巧

### 公式
- 数字+干货：5个技巧教你XXX
- 问题+方案：XXX怎么办？看这篇
- 对比+选择：XXX vs YYY 深度对比
- 教程：XXX从入门到精通

### 原则
- 清晰表达价值
- 包含关键词
- 不要标题党
```

### 开篇技巧

```markdown
## 开篇技巧

### 痛点开头
"你是否遇到过XXX问题？"
"在开发中，我们经常..."

### 数据开头
"据统计，90%的开发者..."
"调查显示，XXX是最大的痛点"

### 故事开头
"上周我遇到了一个有趣的问题..."
"一个深夜，我决定重构..."
```

### 代码展示

```markdown
## 代码展示

### 原则
- 完整可运行
- 注释清晰
- 简洁不冗余
- 语法高亮

### 示例结构
```python
# 导入
import xxx

# 主要代码
def main():
    # 注释说明
    pass

# 调用
if __name__ == "__main__":
    main()
```

### 注意事项
- 代码前说明语言
- 关键行加注释
- 过长代码要精简
- 及时展示运行结果
```

## 文档类型

### API文档

```markdown
## API文档

### 必含内容
| 内容 | 说明 |
|------|------|
| 概述 | 功能介绍 |
| 认证 | 如何认证 |
| 端点 | API列表 |
| 参数 | 参数说明 |
| 返回 | 响应格式 |
| 示例 | 请求示例 |
| 错误码 | 错误说明 |

### 示例

## 获取用户

### 请求
GET /api/users/{id}

### 参数
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 用户ID |

### 响应
```json
{
  "id": "123",
  "name": "张三"
}
```
```

## 质量检查

```markdown

### 内容
- [ ] 标题清晰有吸引力
- [ ] 结构合理层次分明
- [ ] 要点突出不冗余

```

## 好/坏对比

**好**：
```
- 标题清晰表达价值
- 结构清晰层次分明
- 代码完整有注释
- 要点突出易阅读
→ 读者收获大、传播广
```

**坏**：
```
- 标题党或模糊
- 结构混乱
- 代码残缺
- 冗长无重点
→ 读者流失、无价值
```
