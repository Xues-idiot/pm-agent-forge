# Observability

可观测性体系建设。

## 何时使用

- 日志系统建设
- 监控告警设计
- 链路追踪
- 故障排查
- 稳定性保障

## 可观测性三支柱

### 日志

```markdown
## 日志体系

### 日志级别
| 级别 | 用途 | 场景 |
|------|------|------|
| DEBUG | 开发调试 | 开发环境 |
| INFO | 一般信息 | 正常流程 |
| WARN | 警告信息 | 异常但正常 |
| ERROR | 错误信息 | 异常需处理 |
| FATAL | 严重错误 | 系统崩溃 |

### 日志规范
```markdown
## 日志规范

### 必须记录
- 请求入口/出口
- 业务关键节点
- 外部调用
- 异常信息
- 定时任务

### 禁止记录
- 密码/密钥
- 敏感个人信息
- 大量循环日志
- 调试冗余日志
```

### 日志方案
| 方案 | 工具 | 特点 |
|------|------|------|
| ELK | Elasticsearch+Logstash+Kibana | 功能全 |
| EFK | Elasticsearch+Fluentd+Kibana | K8s友好 |
| Loki | Loki+Promtail+Grafana | 成本低 |
| 商业 | Datadog/Splunk | 企业级 |
```

### 日志分析
```markdown
## 日志分析

### 常见分析
- 错误聚合
- 访问趋势
- 性能分析
- 安全审计
- 业务分析

### 查询技巧
- 关键词搜索
- 正则匹配
- 字段过滤
- 统计聚合
- 上下文关联
```

### 指标

```markdown
## 监控指标

### RED方法
| 指标 | 说明 | 计算 |
|------|------|------|
| Rate | 请求率 | QPS |
| Errors | 错误率 | 5xx/总 |
| Duration | 响应时间 | P99 |

### USE方法
| 指标 | 说明 | 示例 |
|------|------|------|
| Utilization | 利用率 | CPU 70% |

### 基础设施指标
| 类型 | 指标 | 工具 |

| 数据库 | 连接/慢查询 | MySQL Exporter |
| 缓存 | 命中率/内存 | Redis Exporter |

## 链路追踪

### 架构

```markdown
## 链路追踪

### 核心概念
| 概念 | 说明 | 示例 |
|------|------|------|
| Trace | 完整调用链 | 一次请求 |
| Span | 单个操作 | 一个函数 |
| Parent | 上游Span | 调用关系 |

### 追踪协议
| 协议 | 特点 | 适用 |
|------|------|------|
| OpenTelemetry | 厂商无关 | 通用 |
| Jaeger | CNCF项目 | K8s环境 |
| Zipkin | 简单轻量 | 小规模 |
| SkyWalking | APM完整 | 国产 |
```

### 埋点方式

```markdown
## 埋点方式

### 代码埋点
```java
// 手动埋点
Span span = tracer.startSpan("操作名");
try {
    // 业务逻辑
} finally {
    span.end();
}
```

### 自动埋点
- Spring Cloud Sleuth
- OpenTelemetry Agent
- Python OpenTelemetry

### SDK埋点
- HTTP框架拦截
- 数据库拦截
- 消息队列拦截
```

## 告警体系

### 告警设计

```markdown
## 告警设计

### 告警级别
| 级别 | 响应 | 定义 | 渠道 |
|------|------|------|------|
| P1 | 5分钟 | 服务不可用 | 电话+短信 |
| P2 | 15分钟 | 功能受损 | 短信+IM |
| P3 | 1小时 | 性能下降 | IM |
| P4 | 4小时 | 警告 | 邮件 |

### 告警收敛
| 策略 | 说明 | 适用 |
|------|------|------|
| 分组 | 相同问题合并 | 批量故障 |
| 抑制 | 高级别压制低级别 | 严重故障 |
| 静默 | 维护窗口 | 计划变更 |
| 聚合 | 时间窗口聚合 | 抖动 |

### 告警模板
```markdown
## 告警模板

### 格式
- 告警名称：[服务][级别]问题描述
- 告警内容：当前值、阈值、持续时间
- 影响范围：服务、用户、金额
- 建议操作：处理建议
- 紧急联系：值班人联系方式
```
```

## 故障管理

### 故障响应

```markdown
## 故障响应

### 响应流程
```markdown
## 故障处理

1. 发现
   - 监控告警
   - 用户反馈
   - 巡检发现

2. 评估
   - 影响范围
   - 严重程度
   - 紧急程度

3. 止血
   - 紧急限流
   - 降级处理
   - 切换备用

4. 排查
   - 查看日志
   - 查看链路
   - 定位根因

5. 修复
   - 代码修复
   - 配置调整
   - 数据修复

6. 复盘
   - 根因分析
   - 改进措施
   - 文档沉淀
```
```

### 故障分级

```markdown
## 故障分级

| 级别 | 影响 | 时长 | 响应 |
|------|------|------|------|
| P1 | 全站不可用 | >30min | CTO+ |
| P2 | 核心功能不可用 | >1h | VP+ |
| P3 | 非核心功能异常 | >4h | 总监+ |
| P4 | 轻微问题 | <8h | 团队内 |
```

### 复盘机制

```markdown
## 复盘要求

### 必须复盘
- P1/P2故障
- 超过30分钟故障
- 用户数据受影响

### 复盘内容
- 故障时间线
- 影响范围
- 根因分析
- 改进措施
- 行动项跟踪

### 复盘模板
1. 故障概述
2. 时间线
3. 影响分析
4. 根因分析
5. 改进措施
6. 行动项
```

## SRE实践

### SLO/SLI/SLA

```markdown
## SRE核心概念

### 指标定义
| 概念 | 说明 | 示例 |
|------|------|------|
| SLI | 指标 | 可用性99.9% |
| SLO | 目标 | 月可用性99.9% |
| SLA | 承诺 | 合同约定99.9% |

### 常见SLO
| 服务 | SLI | SLO |
|------|-----|-----|
| API | 可用性 | 99.9% |
| 延迟 | P99<500ms | 99% |
| 错误 | 5xx<0.1% | 99.9% |
```

### 错误预算

```markdown
## 错误预算

### 概念
- 月可用性99.9%
- 月错误预算：0.1%
- 43分钟/月允许故障

### 策略
| 场景 | 策略 |
|------|------|
| 预算充足 | 继续迭代 |
| 预算消耗快 | 暂停变更 |
| 预算耗尽 | 冻结发布 |

### 监控错误预算
- 错误预算消耗率
- 预测耗尽时间
- 告警阈值设置
```
