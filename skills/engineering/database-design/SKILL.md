## 规范化设计

| 范式 | 要求 | 示例 |
|------|------|------|

## 表结构设计

### 字段设计

```markdown
## 字段设计

### 字段类型选择
| 类型 | 大小 | 适用 | 注意 |
|------|------|------|------|
| INT | 4B | 数值 | 自增主键 |
| BIGINT | 8B | 大数值 | 分布式ID |
| VARCHAR | 1-65535B | 文本 | 长度评估 |
| TEXT | 64KB | 大文本 | 不用建索引 |
| DATETIME | 8B | 时间 | 时区处理 |
| DECIMAL | 变长 | 金额 | 不用Float |

### 字段规范
- 主键：使用BIGINT自增或UUID
- 时间：统一DATETIME，时区UTC
- 金额：DECIMAL(10,2)，不用Float
- 状态：TINYINT或ENUM
- 布尔：TINYINT(1)，不用BOOLEAN
- 文本：UTF8MB4，避免TEXT

### 敏感字段
| 字段 | 处理方式 |
|------|----------|
| 密码 | BCrypt哈希 |
| 手机号 | 脱敏存储 |
| 身份证 | 加密存储 |
| 银行卡 | 加密+脱敏 |
```

### 索引设计

```markdown
## 索引设计

### 索引类型
| 类型 | 特点 | 适用 |
|------|------|------|
| 主键索引 | 唯一且非空 | 主键 |
| 唯一索引 | 唯一可空 | 唯一约束 |
| 普通索引 | 普通查询 | 加速检索 |
| 复合索引 | 多列组合 | 联合查询 |
| 全文索引 | 文本搜索 | 文章搜索 |

### 索引原则
```markdown
## 索引原则

### 创建原则
- 等值查询列优先
- 查询频率高的列
- JOIN的连接列
- ORDER BY的排序列

### 不建索引
- 频繁更新的列
- 低选择性的列（性别/状态）
- 很少查询的列
- TEXT/BLOB大字段

### 复合索引原则
- 最左前缀匹配
- 区分度高的列在前
- 查询条件包含的列
```

### 索引优化
| 优化项 | 方法 | 效果 |
|--------|------|------|
| 覆盖索引 | 包含查询列 | 减少回表 |
| 前缀索引 | 字符串前缀 | 减少空间 |
| 索引下推 | 条件下推 | 减少扫描 |
| 索引合并 | 多索引合并 | 并行检索 |
```

## 红牌警告

- 不做索引规划
- 使用Text存储大文本
- 金额使用Float
- 深分页无优化
- 缺乏慢查询监控
- 无数据备份
- 密码明文存储
