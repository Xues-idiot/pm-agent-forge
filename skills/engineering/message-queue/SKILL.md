# Message Queue

消息队列实践。

## 何时使用

- 异步通信设计
- 消息队列选型
- 事件驱动架构
- 消息可靠性保障
- 队列性能优化

## 消息顺序

### 顺序保障

```markdown
## 消息顺序

### 顺序场景
| 场景 | 需求 | 实现 |
|------|------|------|
| 订单流程 | 创建→支付→发货 | 顺序消息 |
| 账户变动 | 开户→充值→消费 | 顺序消息 |
| 数据同步 | 增→改→删 | 顺序消息 |

### 实现方式
```markdown
## 顺序消息实现

### Kafka顺序
- 单分区：保证顺序
- 分区内有序
- 消费者串行处理

### RocketMQ顺序
- 顺序类型：全局顺序/分区顺序
- 消息组：相同ID到同一队列

### 实现要点
1. 消息分区key一致
2. 单消费者串行
3. 处理能力受限
```
```

|------|----------|

4. 记录处理日志

### 处理方式

## 红牌警告

- 不做消息确认
- 消费后不ACK
- 无死信队列
- 无积压监控
- 消息无幂等
- 无重试机制
- 无容量规划
