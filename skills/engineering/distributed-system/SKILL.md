# 分布式系统设计 (Distributed System Design)

## Overview

分布式系统设计是构建高可用、高性能分布式系统的技能。涵盖服务拆分、分布式事务、一致性、容错机制等核心环节。

## 何时使用

- 系统需要水平扩展
- 单点故障需要消除
- 微服务架构设计
- 分布式场景问题排查

## 流程

1. **业务拆分**：确定服务边界
2. **数据拆分**：确定数据分片策略
3. **一致性**：设计分布式一致性方案
4. **容错**：设计故障检测和恢复
5. **通信**：设计服务间通信机制
6. **治理**：设计监控和运维机制
7. **文档**：编写架构文档

## 检查清单

- [ ] 服务边界清晰，避免循环依赖
- [ ] 数据一致性方案可行
- [ ] 有完善的容错机制
- [ ] 服务间通信可靠
- [ ] 有全局监控和链路追踪
- [ ] 扩容方案可行

## 红牌警告

- 服务拆分不合理导致耦合
- 分布式事务范围过大影响性能
- 缺乏容错导致故障蔓延
- 监控不到位无法定位问题
