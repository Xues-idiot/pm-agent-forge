# Knowledge Management

## 何时使用

- 知识库建设

- 知识搜索优化

## 知识编写规范

## 内容规范

| 答案清晰 | 步骤明确 | 高 |

### 编写流程

```markdown
## 编写流程

### 标准流程
```
选题 → 编写 → 审核 → 发布 → 维护
```

### 各环节
| 环节 | 负责人 | 输出 | 时效 |
|------|--------|------|------|
| 选题 | 运营 | 选题表 | - |
| 编写 | 客服/专家 | 初稿 | 1-2天 |
| 审核 | 主管/产品 | 审核意见 | 1天 |
| 发布 | 运营 | 上线 | 即时 |
| 维护 | 原编写人 | 更新 | 按需 |
```

### 质量标准
| 标准 | 要求 | 检查 |
|------|------|------|
| 准确性 | 内容正确 | 测试验证 |
| 完整性 | 覆盖常见问题 | 对比工单 |
| 可操作性 | 步骤可执行 | 模拟操作 |
| 时效性 | 内容更新及时 | 版本检查 |
```
