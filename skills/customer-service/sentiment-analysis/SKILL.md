# Customer Service Sentiment Analysis Skill

分析用户情感，识别情绪状态，决定是否需要升级处理。

## 何时使用

- 处理投诉或负面反馈
- 用户表达不满或沮丧
- 需要评估用户满意度
- 判断是否需要人工介入
- 客服对话结束后的满意度分析

## 情感分析流程

```
1. 文本预处理 → 2. 情感识别 → 3. 强度评估 → 4. 关键词检测 → 5. 决策输出
```

### 1. 文本预处理

**预处理步骤：**
```markdown
## 文本预处理

### 清洗文本
- 去除表情符号（保留情感线索）
- 规范化大小写
- 处理重复字符（如"很很很生气"→"很生气"）

### 提取要素
- 用户名：[从上下文获取]
- 消息内容：[核心文本]
- 消息时间：[时间戳]
```

### 2. 情感识别

**情感分类体系：**

| 情感类别 | 描述 | 典型特征词 |
|---------|------|-----------|
| 积极 | 满意、感谢、愉快 | 谢谢、很好、满意、棒 |
| 中性 | 客观询问、陈述事实 | 请问、如何、是什么 |
| 消极 | 不满、失望、需要改善 | 不好、失望、不满意 |
| 愤怒 | 强烈不满、要求解释 | 非常生气、投诉、凭什么 |
| 焦虑 | 担心、急迫、需要帮助 | 急、怎么办、很担心 |
| 感激 | 感谢、认可、赞赏 | 非常感谢、太好了、赞 |

### 3. 强度评估

**情感强度量表：**

| 强度 | 分数 | 表现 | 处理建议 |
|------|------|------|---------|
| 极强 | 5 | 大写、感叹号、多次重复 | 立即升级 |
| 强 | 4 | 明显负面情绪 | 优先处理 |
| 中等 | 3 | 有一定负面倾向 | 标准处理 |
| 弱 | 2 | 轻微负面 | 可延后处理 |
| 无 | 1 | 客观/积极 | 正常流程 |

### 4. 关键词检测

**危险信号关键词：**

```markdown
## 危险信号（需要立即关注）

### 升级关键词
- "投诉" - 需要升级处理
- "律师" - 法律风险
- "媒体" - 舆情风险
- "退款" - 经济损失风险
- "曝光" - 舆情风险

### 紧急关键词
- "非常急" / "很急" - 急迫情绪
- "一直" / "总是" - 反复出现的问题
- "第N次" - 问题积累
- "不能用了" - 影响使用

### 负面情绪关键词
- "非常生气" / "气死了"
- "太失望了"
- "再也不用了"
- "太差了"
```

### 5. 决策输出

**分析结果模板：**
```markdown
## 情感分析结果

### 整体情感
- 情感类别：[类别]
- 情感强度：X/5
- 置信度：高/中/低

### 关键发现
- [发现1]
- [发现2]

### 处理建议
- [ ] 正常处理
- [ ] 优先处理
- [ ] 立即升级
- [ ] 人工介入

### 备注
[补充说明]
```

## 情感分析在工单路由中的应用

**基于情感调整优先级：**

| 情感检测结果 | 原始优先级调整 |
|-------------|--------------|
| 愤怒(强) | P+1 或立即处理 |
| 愤怒(极强) | 升级到P0 |
| 焦虑(强) | P+1 |
| 消极(中等) | 维持不变 |
| 积极 | P-1（可延后） |

**升级决策流程：**

```markdown
## 升级决策

### 升级条件（满足任一）
- [ ] 情感强度 ≥ 4
- [ ] 检测到危险信号关键词
- [ ] 用户明确要求升级
- [ ] 问题超过3轮未解决

### 升级路径
1. 一线客服 → 主管
2. 主管 → 经理
3. 经理 → 总监
4. 特殊 → 法务/公关
```

## 情感分析示例

**示例1：正常询问**
```
用户：请问如何修改密码？
分析：情感=中性，强度=1
处理：正常流程，按FAQ处理
```

**示例2：不满用户**
```
用户：这个功能太难用了，试了好几次都不行！
分析：情感=消极，强度=3，有"太...都不行"表达
处理：优先处理，可能需要人工介入
```

**示例3：愤怒投诉**
```
用户：你们这是什么破系统！钱扣了东西没收到！投诉你们！
分析：情感=愤怒，强度=5，检测到"投诉"、"钱"
处理：立即升级，P0处理
```

## 情感分析检查清单

- [ ] 文本已预处理
- [ ] 情感分类正确
- [ ] 强度评估合理
- [ ] 危险信号已检测
- [ ] 处理建议明确
- [ ] 工单已更新（如需要）

## 红牌警告

- 情感分析置信度低时强行判断
- 忽略危险信号关键词
- 情感与实际处理不符
- 用户情绪恶化但未升级
- 分析结果无记录
