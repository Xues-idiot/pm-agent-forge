# Customer Service FAQ Handling Skill

处理常见问题查询，快速准确地给出答案。

## 何时使用

- 用户询问常见问题
- 需要在知识库中搜索答案
- 处理重复性咨询
- 新客服人员培训

## FAQ 处理流程

```
1. 理解问题 → 2. 搜索知识库 → 3. 匹配答案 → 4. 生成回复 → 5. 确认解决
```

### 1. 理解问题

**关键要素：**
- 用户问的是什么？
- 问题的上下文是什么？
- 用户想达到什么目的？

**问题分类：**
| 类型 | 描述 | 处理方式 |
|------|------|---------|
| 咨询类 | 询问产品/服务信息 | 直接回答 |
| 操作类 | 询问如何完成某操作 | 步骤指引 |
| 故障类 | 报告问题或错误 | 诊断+解决 |
| 投诉类 | 表达不满或抱怨 | 倾听+安抚 |

### 2. 搜索知识库

**搜索策略：**
```markdown
## 搜索优先级

1. 精确匹配：关键词完全匹配
2. 同义词匹配：使用同义词搜索
3. 语义匹配：理解意图后搜索
4. 相关问题：查找相似问题
```

**搜索模板：**
```markdown
## 知识库搜索

### 搜索词
[用户问题的关键词]

### 搜索结果
| 文章标题 | 匹配度 | 适用性 |
|---------|--------|--------|
| | | |

### 选中的答案
[知识库文章内容摘要]
```

### 3. 匹配答案

**匹配度评估：**

| 匹配度 | 标准 | 行动 |
|--------|------|------|
| 完全匹配 | 问题完全相同，答案适用 | 直接使用 |
| 高匹配 | 问题核心相同，答案需要调整 | 调整后使用 |
| 部分匹配 | 答案部分相关 | 组合多个答案 |
| 无匹配 | 找不到相关答案 | 转人工/创建工单 |

### 4. 生成回复

**回复模板：**
```markdown
## FAQ 回复模板

### 确认理解
您好，关于您咨询的 [问题]，我来为您解答。

### 答案内容
[知识库答案，如果较长需要总结]

### 补充说明
[如有额外需要说明的内容]

### 进一步帮助
如果还有其他问题，欢迎随时咨询！
```

**回复语气：**
- 专业但友好
- 简洁明了
- 避免过度使用专业术语

### 5. 确认解决

**解决确认：**
```markdown
## 问题解决确认

- [ ] 用户表示理解
- [ ] 用户表示问题已解决
- [ ] 记录解决方案供后续参考
```

## 常见 FAQ 类型

### 产品咨询类

```markdown
**Q: 产品的核心功能是什么？**
A: [产品核心功能说明]

**Q: 如何开通/注册账号？**
A: 1. 访问 [注册页面]
   2. 填写 [信息]
   3. 完成 [验证]
   4. 开始使用
```

### 操作指引类

```markdown
**Q: 如何导出数据/报告？**
A: 导出步骤：
   1. 进入 [页面]
   2. 选择 [筛选条件]
   3. 点击 [导出按钮]
   4. 选择 [格式]
   5. 等待下载完成
```

### 故障处理类

```markdown
**Q: 页面打不开怎么办？**
A: 请尝试以下步骤：
   1. 刷新页面 (Ctrl+F5)
   2. 清除浏览器缓存
   3. 更换浏览器尝试
   4. 检查网络连接

   如果仍未解决，请联系客服。
```

## FAQ 质量标准

| 标准 | 要求 |
|------|------|
| 准确性 | 答案正确，有知识库依据 |
| 完整性 | 覆盖问题的各个方面 |
| 时效性 | 答案是最新的 |
| 可操作性 | 步骤清晰可执行 |

## FAQ 检查清单

- [ ] 正确理解用户问题
- [ ] 搜索了知识库
- [ ] 匹配度评估合理
- [ ] 回复内容准确
- [ ] 回复语气友好
- [ ] 确认问题已解决

## 红牌警告

- 知识库无匹配答案时直接猜测
- 用户情绪负面但未识别
- 问题复杂但强行 FAQ 回答
- 多轮对话后仍未解决问题
- 涉及安全/隐私问题未升级
