# Proactive Service Skill

主动服务，在客户开口前解决问题，降低投诉率。

## 何时使用

- 主动客户关怀
- 问题预防
- 服务体验优化
- 潜在风险识别
- 服务触达设计

## 主动服务流程

```
1. 场景识别 → 2. 触达设计 → 3. 内容准备 → 4. 执行监控 → 5. 效果评估
```

### 2. 触达设计

**触达方式选择：**

```markdown
## 触达方式

### 触达渠道
| 渠道 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 短信 | 到达率高 | 成本高 | 重要通知 |
| 推送 | 成本低 | 打开率低 | 日常提醒 |
| 邮件 | 内容丰富 | 打开率低 | 详细说明 |
| 微信 | 触达稳定 | 需要关注 | 私域用户 |
| 电话 | 即时沟通 | 成本高 | 重要客户 |

### 触达频率
```markdown
## 频率控制

### 避免过度打扰
| 用户类型 | 月度上限 | 说明 |
|----------|----------|------|
| 高价值 | 8-12次 | 重要触点多 |
| 中价值 | 4-8次 | 控制成本 |
| 低价值 | 2-4次 | 避免骚扰 |

### 禁止重复触达
- 同一内容24小时内不重复
- 同一用户1小时内不超2条
- 夜间22:00-次日8:00不触达
```
```

### 3. 内容准备

**内容策略：**

```markdown
## 内容策略

### 内容类型
| 类型 | 作用 | 占比 |
|------|------|------|
| 实用信息 | 提供价值 | 50% |
| 优惠活动 | 促进转化 | 25% |
| 情感关怀 | 增强粘性 | 15% |
| 品牌故事 | 提升认知 | 10% |

### 内容原则
- [ ] 有用：用户提供价值
- [ ] 相关：与用户相关
- [ ] 及时：在恰当时机
- [ ] 简洁：一眼看懂
- [ ] 真诚：不套路用户
```

### 5. 效果评估

**效果评估模型：**

```markdown
## 效果评估

### 评估维度
| 维度 | 指标 | 说明 |
|------|------|------|
| 服务效益 | 问题解决率 | 主动服务减少的投诉 |
| 用户体验 | NPS变化 | 主动服务前后对比 |
| 成本效益 | ROI | 投入vs减少的投诉成本 |
| 效率提升 | 被动转主动 | 主动vs被动服务占比 |

### ROI计算
```markdown
## ROI计算

主动服务投入：
- 人工成本：[X]元
- 系统成本：[X]元
- 内容成本：[X]元
- 总投入：[Y]元

主动服务收益：
- 减少投诉：[A]件 × [B]元/件 = [C]元
- 减少主动咨询：[D]次 × [E]元/次 = [F]元
- 提升复购：[G]元
- 总收益：[H]元

ROI = (H - Y) / Y × 100%
```
```

**持续优化：**

```markdown
## 优化迭代

### 优化方向
1. 时机优化：什么时间触达效果最好

3. 人群优化：什么样用户更值得主动服务
4. 渠道优化：什么渠道成本效益更好

```markdown

### 实验2：内容风格

## 红牌警告

- 过度打扰用户
- 内容过于营销
- 数据不准确
- 承诺无法兑现
- 隐私数据滥用
- 触达时间不当
- 不跟踪效果
- 只触达不跟进
