# Customer Service Response Drafting Skill

生成专业、友好的客服回复内容。

## 何时使用

- 需要回复用户消息
- 起草工单回复
- 写邮件/短信/推送通知
- 批量生成模板回复
- 回复语气需要调整

## 回复起草流程

```
1. 理解上下文 → 2. 确定回复类型 → 3. 收集必要信息 → 4. 起草草稿 → 5. 审核优化 → 6. 最终确认
```

### 1. 理解上下文

**必须了解：**
```markdown
## 上下文信息

### 用户信息
- 用户类型：[新用户/老用户/VIP]
- 历史交互：[无/有投诉记录/多次咨询]

### 问题信息
- 问题类型：[咨询/故障/投诉]
- 问题状态：[处理中/已解决/待确认]
- 已提供的解决方案：

### 情感状态
- 用户情感：[积极/中性/消极/愤怒]
- 紧迫程度：[一般/急/很急]
```

### 2. 确定回复类型

| 回复类型 | 场景 | 语气要求 |
|---------|------|---------|
| 问候类 | 首次回复 | 友好热情 |
| 确认类 | 收到用户消息 | 简洁确认 |
| 解答类 | 提供答案 | 专业清晰 |
| 进展类 | 问题处理中 | 透明进度 |
| 解决方案类 | 提供最终方案 | 完整有礼 |
| 致歉类 | 道歉/补偿 | 真诚恳切 |
| 挽留类 | 挽留要离开的用户 | 诚恳有吸引力 |
| 告别类 | 结束对话 | 祝福期待 |

### 3. 收集必要信息

**信息清单：**
```markdown
## 起草所需信息

### 必要信息
- [ ] 用户姓名/称呼
- [ ] 问题/订单/账户相关信息
- [ ] 具体解决方案（如已有）

### 可选信息
- [ ] 订单号/工单号
- [ ] 相关时间节点
- [ ] 补偿/优惠内容
```

### 4. 起草草稿

**回复结构模板：**

```markdown
## 回复模板

### 第一段：称呼 + 确认理解
您好[称呼]，关于您咨询的[问题]，我已了解。

### 第二段：核心内容
[根据回复类型调整]

### 第三段：行动说明
您可以[具体操作指引]。

### 第四段：后续联系
如有其他问题，请随时联系我。

### 结尾
祝您[祝福语]！
```

**不同场景的回复示例：**

### 场景1：初次问候
```markdown
您好，[用户名]！

感谢您选择[产品/服务]！
我是您的专属客服[名字]，很高兴为您服务。

请问有什么可以帮助您的？
```

### 场景2：问题已解决
```markdown
您好，[用户名]！

很高兴通知您，您的问题已经解决了！

[简述解决方案]

再次感谢您的耐心等待，
如有任何其他问题，欢迎随时联系我们。

祝您生活愉快！
[客服签名]
```

### 场景3：提供临时方案
```markdown
您好，[用户名]！

关于您反馈的[问题]，我理解给您带来的不便。

目前我们的技术团队正在紧急处理中。
作为临时方案，您可以尝试：
1. [步骤1]
2. [步骤2]

预计在[时间]内可以完全解决。
给您带来的不便，敬请谅解！

如有问题请随时联系我。
```

### 场景4：道歉+补偿
```markdown
您好，[用户名]！

非常抱歉给您带来了不好的体验。

经过核实，您遇到的问题是[问题描述]。
这是由于[原因]导致的，对此我们深表歉意。

我们已经为您准备了以下补偿：
- [补偿内容1]
- [补偿内容2]

同时，我们承诺[改进措施]，
确保此类问题不再发生。

再次感谢您的理解和包容！
```

### 场景5：挽留用户
```markdown
您好，[用户名]！

看到您最近使用频率有所下降，我们非常关心您的情况。

如果您在使用过程中遇到任何问题，我们可以提供：
- 专属1对1使用指导
- 优先技术支持
- [VIP专属权益]

我们非常希望能继续为您提供服务。
您有哪些建议或想法，也欢迎告诉我们！

期待您的回复！
```

### 5. 审核优化

**审核检查点：**

```markdown
## 回复审核

### 内容审核
- [ ] 准确回答了用户问题
- [ ] 信息完整无遗漏
- [ ] 解决方案可行

### 语气审核
- [ ] 语气符合用户情感
- [ ] 称呼恰当
- [ ] 无不专业用语

### 格式审核
- [ ] 长度适中
- [ ] 段落清晰
- [ ] 无错别字
```

### 6. 最终确认

**确认清单：**
- [ ] 回复内容准确
- [ ] 语气合适
- [ ] 用户信息正确
- [ ] 附件/链接已检查
- [ ] 准备发送

## 回复语气指南

| 用户情感 | 建议语气 | 避免 |
|---------|--------|------|
| 积极 | 友好热情 | 过度热情 |
| 中性 | 专业友好 | 过于正式 |
| 消极 | 真诚关心 | 敷衍 |
| 愤怒 | 理解道歉 | 辩解 |
| 焦虑 | 安抚+行动 | 空洞承诺 |

## 回复检查清单

- [ ] 正确理解上下文
- [ ] 选择了正确的回复类型
- [ ] 语气与用户情感匹配
- [ ] 信息完整准确
- [ ] 没有语法/拼写错误
- [ ] 发送前已审核

## 红牌警告

- 未核实信息就发送回复
- 回复内容与实际问题不符
- 语气与用户情感不匹配
- 承诺了无法兑现的事情
- 发送了错误的内容给错误的人
- 敏感信息泄露

## 模板库管理

建议维护的模板库：

```markdown
## 模板库结构

templates/

├── solution.md        # 解决方案模板
├── apology.md        # 道歉模板
├── retention.md      # 挽留模板
└── farewell.md       # 告别模板
```
