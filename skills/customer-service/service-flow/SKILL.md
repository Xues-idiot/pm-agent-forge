# Customer Service Flow

客服服务流程，设计客服流程、处理请求和升级管理。

## 何时使用

- 设计客服服务流程
- 处理客户请求
- 管理升级投诉
- 优化客服效率
- 培训客服人员

## 客服服务流程

## 问题诊断

### 诊断框架

```markdown
## 问题诊断

### 5W1H分析法
| 问题 | 内容 |
|------|------|
| What | 发生了什么问题 |
| When | 什么时候发生 |
| Where | 哪里出现问题 |
| Who | 谁遇到的问题 |
| Why | 为什么会出现 |
| How | 如何解决问题 |

### 诊断步骤
1. 倾听：让用户完整描述问题
2. 复述：确认理解问题
3. 提问：补充必要信息
4. 诊断：判断问题原因
```

## 满意度管理

### 满意度评估

```markdown
## 满意度

### 评价维度
| 维度 | 说明 |
|------|------|
| 响应速度 | 是否及时回复 |
| 服务态度 | 是否友好耐心 |
| 专业程度 | 能否解决问题 |
| 处理结果 | 结果是否满意 |

### 满意度计算
- 非常满意：10分
- 满意：7-9分
- 一般：4-6分
- 不满意：1-3分
- 满意度 = (7-10分用户数 / 总用户数) × 100%
```

### 满意度提升

```markdown
## 满意度提升

### 低满意度原因
| 原因 | 占比 | 改进 |
|------|------|------|
| 问题未解决 | 40% | 提升处理能力 |
| 等待时间长 | 30% | 优化流程 |
| 态度不好 | 20% | 培训加强 |
| 其他 | 10% | 分析改进 |

### 改进措施
1. 定期培训：服务态度+产品知识
2. 优化流程：减少等待时间
3. 激励考核：满意度与绩效挂钩
```

## 红牌警告

- 踢皮球不处理
- 态度恶劣
- 承诺不兑现
- 泄露用户信息
- 过度承诺

## 好/坏对比

**好**：
```
客服处理：
1. 快速响应，确认问题
2. 倾听用户，专业解答
3. 超出权限，升级处理
4. 全程跟进，结果反馈
→ 用户满意，效率高
```

**坏**：
```
客服处理：
"这个问题不归我管"
→ 踢皮球
"我也没办法"
→ 态度消极
→ 用户不满升级
```
