# Complaint Handling Skill

客户投诉处理，管理负面情绪，化解客户不满。

## 何时使用

- 客户投诉处理
- 情绪安抚
- 升级事件处理
- 危机公关配合
- 投诉数据分析

## 投诉处理流程

```
1. 接收确认 → 2. 情绪安抚 → 3. 问题诊断 → 4. 方案提供 → 5. 闭环跟踪
```

**投诉接收规范：**

```markdown
## 投诉接收

- [ ] 控制语速和语气

### 标准开场
"您好，我是[姓名]，非常抱歉给您带来困扰，我会认真处理您的问题，请问可以先了解一下具体情况吗？"

**紧急度判断：**

```markdown
## 紧急度分级

### P0 - 紧急
| 情况 | 定义 | 响应时间 |
|------|------|----------|
| 媒体曝光风险 | 可能发微博/媒体 | 立即 |
| 大量用户受影响 | 50人以上 | 1小时 |
| 涉及资金安全 | 账户异常/欺诈 | 立即 |

### P1 - 高优先级
| 情况 | 定义 | 响应时间 |
|------|------|----------|
| VIP客户投诉 | 高价值用户 | 2小时 |
| 重复投诉 | 同一问题2次+ | 4小时 |
| 公开投诉 | 朋友圈/群 | 2小时 |

### P2 - 普通
| 情况 | 定义 | 响应时间 |
|------|------|----------|
| 一般投诉 | 单个用户 | 24小时 |
| 建议反馈 | 非紧急问题 | 48小时 |

### 2. 情绪安抚

**情绪识别：**

```markdown
## 客户情绪识别

### 愤怒期
| 表现 | 策略 |
|------|------|
| 声音变大 | 先倾听不打断 |
| 语速加快 | 适当放慢自己的语速 |
| 重复表达 | 表示理解，复述要点 |
| 使用感叹号 | 避免对抗性语言 |

### 失望期
| 表现 | 策略 |
|------|------|
| 叹气 | 表示理解原因 |
| 沉默 | 主动询问需求 |
| 语气低落 | 温和回应，积极解决 |

### 焦虑期
| 表现 | 策略 |
|------|------|
| 反复询问 | 耐心解答 |
| 担心后果 | 明确保障措施 |
| 犹豫不决 | 提供最优方案 |
```

**情绪安抚话术：**

```markdown
## 安抚话术

### 表达理解
"我非常理解您的心情，换作是我也会着急"
"让您遇到这样的问题，真的非常抱歉"

### 表示重视
"您反映的这个问题非常重要"
"我会亲自跟进处理，确保给您一个满意的答复"

### 肢体语言（电话）
- 语速：比平时稍慢
- 语气：温和且坚定
- 音量：比平时稍低
- 停顿：适当留白让客户表达

### 禁止用语
❌ "您冷静一下"
❌ "这个问题很简单"
❌ "不是我们的问题"
❌ "您理解错了"
```
```

### 3. 问题诊断

**问题分析框架：**

```markdown
## 问题诊断

### 5Why分析法
```markdown
## 问题：客户投诉配送慢

Why1：为什么投诉？（配送太慢）
Why2：为什么会慢？（订单堆积）
Why3：为什么会堆积？（促销订单多）
Why4：为什么促销订单多？（预估备货不足）
Why5：为什么预估不足？（历史数据参考不够）

根因：促销前备货预测机制不完善
```
```

**问题分类：**

```markdown
## 问题分类

### 产品问题
| 类型 | 定义 | 责任方 |
|------|------|--------|
| 质量问题 | 产品本身缺陷 | 供应商/生产 |
| 描述不符 | 实际与宣传差 | 产品/运营 |
| 过期问题 | 临期或过期 | 仓储/物流 |

### 服务问题
| 类型 | 定义 | 责任方 |
|------|------|--------|
| 响应慢 | 回复超时 | 客服 |
| 态度差 | 服务态度问题 | 客服 |
| 错误解答 | 信息不准确 | 客服/培训 |

### 物流问题
| 类型 | 定义 | 责任方 |
|------|------|--------|
| 延迟 | 未按时送达 | 物流 |
| 丢失 | 包裹丢失 | 物流 |
| 破损 | 运输损坏 | 物流 |
| 错发 | 型号/数量错 | 仓储 |

### 承诺问题
| 类型 | 定义 | 责任方 |
|------|------|--------|
| 虚假宣传 | 夸大功能 | 运营/法务 |
| 承诺不兑现 | 说过的话没做到 | 客服/管理 |
```
```

### 4. 方案提供

**解决方案模板：**

```markdown
## 解决方案

### 方案结构
1. 道歉：真诚道歉
2. 解释：说明原因（如有）
3. 补偿：给出具体补偿
4. 承诺：保证不再犯

### 补偿标准
```markdown
## 补偿参考

| 问题类型 | 轻度 | 中度 | 重度 |
|----------|------|------|------|
| 配送延迟 | 5-10元券 | 20-30元券+道歉 | 退款+补偿 |
| 产品损坏 | 退款/补发 | 退款+补偿 | 退款+高额补偿 |
| 服务态度 | 道歉 | 道歉+小额券 | 道歉+补偿+处理 |
| 虚假宣传 | 说明澄清 | 退款+补偿 | 退款+赔偿 |

### 补偿形式
- 优惠券/红包
- 补发产品
- 退款处理
- 额外赠品
- VIP权益
```
```

**方案提出话术：**

```markdown
## 方案话术

### 自我介绍+道歉
"您好，我是[姓名]，非常抱歉给您带来不便，这确实是我们的责任。"

### 说明原因（如适用）
"经过了解，是因为[原因]导致了这个问题的发生。"

### 提出解决方案
"为了弥补给您造成的困扰，我们这边可以为您提供：[方案]"

### 确认接受
"这个方案您看可以吗？如果您还有其他需要，我们可以再商量。"

### 承诺改进
"您的反馈我们会认真反思，避免类似问题再次发生。"
```
```

### 5. 闭环跟踪

**跟踪流程：**

```markdown
## 投诉跟踪

### 处理时效
| 阶段 | 时效要求 | 动作 |
|------|----------|------|
| 首次响应 | 立即/<2h | 告知已收到 |
| 方案提出 | 24h内 | 提供具体方案 |
| 方案确认 | 48h内 | 达成一致 |
| 执行完成 | 方案约定时间内 | 补偿兑现 |
| 满意度回访 | 完成后3天 | 确认满意 |

**升级处理：**

```markdown
## 升级处理

### 升级条件
- [ ] 客户明确要求升级
- [ ] 投诉次数≥3次
- [ ] 涉及金额>5000元
- [ ] 可能引发媒体曝光
- [ ] VIP客户投诉

### 升级流程
```markdown
## 升级流程

客服 → 客服组长 → 主管 → 经理 → 总监

### 各层级处理权限
| 层级 | 补偿权限 | 处理时限 |
|------|----------|----------|
| 客服 | 0-100元 | 24h |
| 组长 | 100-500元 | 12h |
| 主管 | 500-2000元 | 6h |
| 经理 | 2000-10000元 | 2h |
| 总监 | 10000元+ | 立即 |
```
```

## 投诉处理检查清单

- [ ] 响应及时不推诿
- [ ] 情绪安抚到位
- [ ] 问题诊断准确
- [ ] 方案合理有诚意
- [ ] 承诺兑现及时
- [ ] 记录完整可查
- [ ] 回访确认满意
- [ ] 经验总结归档

## 红牌警告

- 推诿甩锅
- 与客户争吵
- 虚假承诺
- 泄露客户信息
- 拖延不处理
- 补偿不兑现
- 投诉数据造假
- 不分析不改进
