# Customer Service Knowledge Base Search Skill

在知识库中搜索相关文章和文档，支撑客服问题解答。

## 何时使用

- 回答用户咨询前搜索答案
- 找不到问题时搜索类似案例
- 需要引用官方文档
- 培训新员工了解产品
- 创建 FAQ 内容

## 知识库搜索流程

```
1. 理解搜索需求 → 2. 选择搜索策略 → 3. 执行搜索 → 4. 评估结果 → 5. 提取内容 → 6. 整理输出
```

### 1. 理解搜索需求

**搜索需求分析：**
```markdown
## 搜索需求

### 核心问题
[用户问题的核心是什么]

### 关键实体
- 产品/功能：
- 操作：
- 问题/错误：

### 搜索目标
- [ ] 直接答案
- [ ] 操作步骤
- [ ] 政策说明
- [ ] 类似案例

### 约束条件
- 语言：[中文/英文/双语]
- 时效：[最新/当前有效]
- 范围：[产品/运营/政策]
```

### 2. 选择搜索策略

**搜索策略选择：**

| 策略 | 适用场景 | 搜索词示例 |
|------|---------|-----------|
| 精确搜索 | 问题明确、关键词清晰 | "密码找回" |
| 同义词搜索 | 精确搜索无结果 | 密码→登录码→账户恢复 |
| 语义搜索 | 问题模糊、需要理解意图 | "账号进不去" → "登录问题" |
| 分类浏览 | 不知道具体关键词 | 按产品分类浏览 |
| 相关搜索 | 找到类似问题 | 基于已有文档的相关推荐 |

**搜索词扩展：**

```markdown
## 搜索词扩展

### 核心词
[原始搜索词]

### 同义词
- [同义词1]
- [同义词2]

### 相关词
- [相关词1]
- [相关词2]

### 用户常用表达
- [表达1]
- [表达2]
```

**搜索执行模板：**

```markdown
- 搜索词：[核心词]
- 结果数量：X条
- 命中：X条

### 第二轮搜索（如需）
- 搜索词：[扩展词]
- 结果数量：X条
- 命中：X条
```

### 4. 评估结果

**结果评估标准：**

| 评估维度 | 权重 | 说明 |
|---------|------|------|
| 相关性 | 40% | 内容与问题的匹配程度 |
| 准确性 | 30% | 信息是否正确 |
| 时效性 | 15% | 是否为最新内容 |
| 完整性 | 10% | 是否完整回答问题 |
| 可读性 | 5% | 内容是否易读易懂 |

**结果选择矩阵：**

| 相关性 | 准确性 | 行动 |
|--------|--------|------|
| 高 | 高 | 直接使用 |
| 高 | 低 | 标记不可靠，慎用 |
| 中 | 高 | 部分采用，需补充 |
| 低 | 高 | 参考角度，另找 |
| 低 | 低 | 不采用 |

### 5. 提取内容

**内容提取模板：**

```markdown

### 文档信息

- 链接：
- 更新时间：
```

## 高级搜索技巧

**搜索语法：**

| 语法 | 含义 | 示例 |
|------|------|------|
| "精确短语" | 完全匹配 | "密码找回" |
| OR | 或关系 | 密码 OR 登录码 |
| AND | 与关系 | 登录 AND 失败 |
| NOT | 排除 | 登录 NOT 注册 |

**搜索优化：**

```markdown
## 搜索优化

### 提高召回率
- 使用同义词
- 使用上位概念（如"登录问题"代替具体错误码）
- 模糊搜索

### 提高精度
- 使用精确短语
- 使用AND组合
- 限制分类范围
```

## 红牌警告

- 搜索结果明显不相关但仍使用
- 引用过时政策/文档
- 未核实内容准确性就使用
- 搜索词过于宽泛导致结果噪音大
- 只看标题不读内容
- 多个结果冲突时未做判断

## 知识库建设建议（供参考）

```markdown
## 知识库最佳实践

### 内容标准
- 每篇文章解决一个具体问题
- 使用用户语言（非技术语言）
- 包含操作截图/视频
- 定期更新维护

### 分类标准
- 按用户场景分类
- 按产品模块分类
- 交叉引用相关文档

### 质量控制
- 上线前审核
- 用户反馈机制
- 定期内容审计
```
