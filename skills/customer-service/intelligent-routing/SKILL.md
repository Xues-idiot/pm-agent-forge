## 负载均衡

### 负载计算

```markdown
## 负载计算

### 负载因素
| 因素 | 权重 | 说明 |
|------|------|------|
| 当前会话 | 1.0 | 同时接待数 |
| 排队优先级 | 0.5 | 等待中 |
| 技能要求 | 0.3 | 匹配度 |
| 预计时长 | 0.2 | 预估处理 |

### 负载公式
```
当前负载 = Σ(会话数×权重) + Σ(等待数×权重)

可接待 = 最大接待数 - 当前负载
```
```

### 均衡策略

```markdown
## 均衡策略

### 均衡原则
- 同一客服接待量差距不过大
- 等待时间长的优先分配
- 紧急问题优先处理

### 动态调整
| 策略 | 说明 | 触发 |
|------|------|------|
| 扩容 | 增加接待 | 高峰期 |
| 缩容 | 减少接待 | 低峰期 |
| 转移 | 转移会话 | 负载不均 |
```

```markdown
## 路由决策

### 决策因素
1. 客户特征（等级、类型）
2. 问题特征（类型、紧急度）
3. 客服特征（技能、负载）
4. 上下文（时段、渠道）

### 决策树
```
客户类型 = VIP?
  → YES: 分配高级客服
  → NO: 问题类型 = 专业?
      → YES: 分配专业客服
      → NO: 等待时间 > 阈值?
          → YES: 分配最快响应
          → NO: 轮询分配
```
```
