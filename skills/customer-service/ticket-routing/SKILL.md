# Customer Service Ticket Routing Skill

智能分配工单，将问题路由到合适的团队或人员。

## 何时使用

- 收到新的客服工单
- 需要确定工单处理优先级
- 需要将工单转交给其他团队
- 处理高负载的工单积压

## 工单路由流程

```
1. 读取工单 → 2. 分析问题 → 3. 确定类别 → 4. 评估优先级 → 5. 选择路由 → 6. 分配执行
```

### 2. 分析问题

**分析维度：**

| 维度 | 内容 |
|------|------|
| 问题类型 | 咨询/投诉/故障/建议 |
| 产品模块 | 功能/模块/集成 |
| 紧急程度 | 用户描述的紧急性 |
| 影响范围 | 单用户/多用户/全局 |

### 3. 确定类别

**工单分类体系：**

| 一级分类 | 二级分类 | 处理团队 |
|---------|---------|---------|
| 产品问题 | 功能故障 | 技术支持 |
| 产品问题 | 使用咨询 | 产品运营 |
| 产品问题 | 功能建议 | 产品经理 |
| 账户问题 | 登录异常 | 安全团队 |
| 账户问题 | 权限问题 | 技术支持 |
| 账单问题 | 支付失败 | 财务 |
| 账单问题 | 发票问题 | 财务 |
| 其他 | 合作咨询 | 商务 |
| 其他 |媒体采访 | 市场 |

### 4. 评估优先级

**优先级矩阵：**

| 级别 | 定义 | 响应时间 | 处理方式 |
|------|------|---------|---------|
| P0 | 全局故障、多用户受影响 | 15分钟 | 立即处理 |
| P1 | 单用户严重问题 | 1小时 | 优先处理 |
| P2 | 功能异常、可 workaround | 4小时 | 标准队列 |
| P3 | 一般咨询、小问题 | 24小时 | 普通队列 |
| P4 | 简单问题、建议 | 72小时 | 低优先级 |

**优先级评估因素：**

```markdown
## 优先级评估

### 加分项（优先级提升）
- [ ] 多用户报告同一问题 (+1)
- [ ] 用户是付费大客户 (+1)
- [ ] 影响核心功能 (+1)
- [ ] 用户情绪激动 (+1)
- [ ] 问题持续超过24小时 (+1)

### 减分项（优先级降低）
- [ ] 问题描述模糊 (-1)
- [ ] 非工作时间 (-1)
- [ ] 简单FAQ问题 (-2)
```

### 5. 选择路由

**路由决策树：**

```markdown
## 路由决策

### 问题类型判断
- [ ] 是产品故障？ → 技术支持
- [ ] 是使用问题？ → 产品运营
- [ ] 是账户问题？ → 安全团队
- [ ] 是账单问题？ → 财务

### 特殊情况
- [ ] 涉及多个团队？ → 主导团队 + 抄送
- [ ] 需要升级？ → 主管/经理
- [ ] 紧急？ → 直连负责人
```

### 6. 分配执行

**分配执行记录：**
```markdown
## 工单分配

### 分配结果
- 分配给：[团队/人员]

- 预期处理时间：

### 通知
- [ ] 发送分配通知
- [ ] 更新工单状态
- [ ] 记录路由原因
```

## 路由检查清单

- [ ] 工单信息完整读取
- [ ] 问题类型判断正确
- [ ] 优先级评估合理
- [ ] 路由目标正确
- [ ] 分配通知已发送
- [ ] 工单状态已更新

## 红牌警告

- 优先级评估与实际情况不符
- 路由到错误团队导致二次转交
- 高优先级工单未及时处理
- 路由决策无记录
- 用户问题涉及多团队但未协调
