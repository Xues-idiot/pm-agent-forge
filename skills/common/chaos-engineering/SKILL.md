# 混沌工程 (Chaos Engineering)

## 触发条件
- 故障注入
- 系统韧性
- 容错测试
- 主动防御

## 输入
- 系统架构
- 故障场景
- 稳态定义
- 风险控制

## 输出
- 实验设计
- 故障场景
- 防护措施
- 结果评估

## Prompt模板
```
请进行混沌工程实验。

背景：
- 系统架构：[架构]
- 故障场景：[场景]
- 稳态定义：[稳态]
- 风险控制：[控制]

请输出：
1. 实验计划
2. 故障场景
3. 假设验证
4. 防护措施
5. 结果评估
```

## 示例
### 示例1
- 输入：电商系统
- 输出：混沌实验，含服务器宕机、网络延迟、数据库故障等场景验证

## 注意事项
- 要小范围开始
- 要有防护措施
- 要持续进行
- 要团队共识
