# 可观测性 (Observability)

## 触发条件
- 故障诊断
- 性能分析
- 链路追踪
- 运维效率

## 输入
- 系统架构
- 监控现状
- 故障场景
- 团队能力

## 输出
- 可观测架构
- 指标体系
- 日志规范
- 链路追踪

## Prompt模板
```
请建设可观测性。

背景：
- 系统架构：[架构]
- 监控现状：[现状]
- 故障场景：[场景]
- 团队能力：[能力]

请输出：
1. 可观测架构
2. 指标设计
3. 日志规范
4. 链路追踪
5. 告警策略
```

## 示例
### 示例1
- 输入：分布式微服务
- 输出：可观测性建设，含Metrics+Logs+Traces三位一体，OpenTelemetry标准

## 注意事项
- 要标准化
- 要低开销
- 要全链路
- 要持续改进
