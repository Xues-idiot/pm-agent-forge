# 服务连续性 (Service Continuity)

## 触发条件
- 业务连续性
- 灾难恢复
- 应急响应
- 预案演练

## 输入
- 关键业务
- 恢复目标
- 潜在威胁
- 资源需求

## 输出
- 连续性计划
- 恢复策略
- 演练计划
- 改进机制

## Prompt模板
```
请保障服务连续性。

背景：
- 关键业务：[业务]
- 恢复目标：[目标]
- 潜在威胁：[威胁]
- 资源需求：[资源]

请输出：
1. 业务影响分析
2. 恢复策略
3. 应急预案
4. 演练计划
5. 持续改进
```

## 示例
### 示例1
- 输入：数据中心
- 输出：服务连续性，同城双活、RTO 2小时、RPO 1小时

## 注意事项
- 要关键业务优先
- 要切实可行
- 要定期演练
- 要持续更新
