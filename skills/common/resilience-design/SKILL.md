# 韧性设计 (Resilience Design)

## 触发条件
- 高可用
- 故障容错
- 灾备设计
- 快速恢复

## 输入
- 可用性目标
- 故障场景
- 恢复要求
- 成本约束

## 输出
- 韧性策略
- 容错设计
- 恢复方案
- 测试计划

## Prompt模板
```
请设计系统韧性。

背景：
- 可用性目标：[目标]
- 故障场景：[场景]
- 恢复要求：[要求]
- 成本约束：[约束]

请输出：
1. 韧性目标
2. 容错设计
3. 熔断降级
4. 恢复方案
5. 混沌测试
```

## 示例
### 示例1
- 输入：核心交易系统
- 输出：韧性设计，含主备切换、熔断降级、限流保护、故障注入测试

## 注意事项
- 要面向失败
- 要优雅降级
- 要快速恢复
- 要持续测试
