# SPACE-thesis-writing：学术论文全流程写作

## 论文类型与要求差异

不同论文类型有不同的要求标准：

### 各类型论文对比

| 维度 | 本科论文 | 硕士论文 | 博士论文 | 期刊论文 |
|------|---------|---------|---------|---------|
| 字数要求 | 8千-1.5万 | 3万-5万 | 8万+ | 8千-1.5万（核心期刊更少） |
| 文献要求 | ≥20篇 | ≥80篇 | ≥150篇 | ≥30篇（核心期刊50+） |
| 研究深度 | 基础应用 | 理论+应用 | 原创贡献 | 创新性 |
| 方法要求 | 基本掌握 | 熟练运用 | 方法论贡献 | 严谨可重复 |
| 创新要求 | 适度创新 | 有所创新 | 实质创新 | 显著创新 |
| 周期时长 | 3-6月 | 1-2年 | 3-5年 | 6-18月 |

### 各学科论文写作要点

#### 理工科
- **结构特点**：IMRAD结构（Introduction, Methods, Results, And Discussion）
- **方法要求**：实验设计要详细可重复
- **结果呈现**：数据+图表为主
- **特别要求**：材料与方法章节详细

#### 社会科学/管理学
- **结构特点**：理论框架+实证验证
- **方法要求**：问卷调查+数据分析
- **结果呈现**：统计结果+讨论
- **特别要求**：理论贡献和实践意义

#### 人文社科
- **结构特点**：论证为主
- **方法要求**：文献分析+文本解读
- **结果呈现**：论述为主
- **特别要求**：学术史梳理

#### 医学/生命科学
- **结构特点**：结构化实验报告
- **方法要求**：严格对照实验
- **结果呈现**：临床数据+统计
- **特别要求**：伦理审批

### 摘要写法规范

```markdown
## 摘要结构（五要素）

### 本科/硕士论文摘要
1. **研究背景**（1-2句）：为什么做这项研究
2. **研究目的**（1句）：要解决什么问题
3. **研究方法**（1-2句）：如何做的
4. **主要结果**（2-3句）：发现了什么
5. **结论**（1-2句）：结论是什么

**示例（300字）**：
[研究背景] 随着数字化转型的深入推进，中小企业面临组织惯性的挑战。[研究目的] 本研究旨在探讨数字化转型背景下组织惯性对中小企业创新绩效的影响机制。[研究方法] 基于动态能力理论和组织惯性理论，收集了328份有效问卷，使用层级回归分析和Bootstrap方法进行实证检验。[主要结果] 研究发现：（1）组织惯性负向影响创新绩效（β=-.35, p<.001）；（2）动态能力在组织惯性与创新绩效间起部分中介作用（间接效应=-.12, 95%CI[-.19,-.05]）；（3）环境动荡性正向调节上述关系。[结论] 研究结论为中小企业克服组织惯性、提升创新绩效提供了理论依据和实践指导。

### 期刊论文摘要（结构化摘要）
1. **Background**（背景）
2. **Objectives**（目的）
3. **Methods**（方法）
4. **Results**（结果）
5. **Conclusions**（结论）

### 摘要避坑指南
| 错误写法 | 正确写法 |
|---------|---------|
| "本文研究了..." | "本研究探讨..." |
| "结果表明..." | "研究结果显示..." |
| 无具体数据 | 包含关键统计量 |
| 主观评价 | 客观陈述 |
| 引用文献 | 不引用文献 |
```

### 本科论文特点
- 侧重基础知识运用
- 结构相对标准
- 导师指导频率低
- 查重要求一般≤30%
- 答辩时间10-15分钟

### 硕士论文特点
- 要求理论框架
- 需要文献评述
- 数据分析要规范
- 查重要求一般≤20%
- 答辩时间15-20分钟
- 需要盲审

### 博士论文特点
- 要求原创性贡献
- 理论要有突破
- 方法要有创新
- 查重要求一般≤15%
- 答辩时间30-60分钟
- 需外审盲审

### 期刊论文特点
- 创新性最重要
- 结构精简（IMRAD）
- 多次修改
- 同行评审
- 见刊周期长

## 阶段一：选题与规划（Topic & Planning）

### 做什么

帮助用户明确研究问题、评估研究可行性、制定写作计划。

### 1.1 研究问题确定

通过以下问题引导用户明确研究问题：

**必须回答（阻塞动笔的）：**
- 你的研究主题是什么？（尽量具体到可研究的变量）
- 你想解决什么问题？/ 回答什么研究问题？
- 为什么这个问题值得研究？（研究意义）
- 你的研究视角是什么？（理论视角）

**影响深度的：**
- 有没有初步的假设或预期结论？
- 你打算用哪种研究方法？
- 预计多长时间完成？

### 1.2 研究可行性评估

| 评估维度 | 评估要点 | 评估结果 |
|---------|---------|---------|
| 文献可行性 | 能否找到足够的参考文献？ | □充分 □一般 □不足 |
| 数据可行性 | 能否获取研究所需数据？ | □可获取 □部分可 □不可 |
| 时间可行性 | 是否在规定时间内完成？ | □充足 □紧张 □不足 |
| 能力匹配 | 研究方法是否在能力范围内？ | □匹配 □需学习 □超出 |

## 阶段二：文献综述（Literature Review）

### 做什么

系统梳理研究领域的已有研究，评述研究现状，找出研究缺口。

### 2.1 文献搜索策略

| 搜索维度 | 建议资源 | 搜索词示例 |
|---------|---------|-----------|
| 中文文献 | CNKI、万方、Web of Science | 研究主题 + 研究方法 |
| 英文文献 | Google Scholar、PubMed、Scopus | Topic + Method + Review |
| 经典文献 | 导师推荐、高被引文献 | - |

### 2.2 文献整理方法

建议使用以下分类框架整理文献：

```markdown
## 文献分类框架

### A. 理论研究类
- [作者1, 年份] 研究了X，提出Y理论/模型
- [作者2, 年份] 对Y理论进行扩展，研究了Z

### B. 实证研究类
- [作者3, 年份] 使用X方法研究Y问题，发现Z
- [作者4, 年份] 对Y进行实证，发现结果与Z一致/不一致

### C. 方法研究类
- [作者5, 年份] 提出X方法，用于研究Y问题

### D. 研究缺口（Gap）
- 现有研究很少关注XXX
- 现有研究缺乏XXX视角
- 现有方法无法解决XXX问题
```

### 2.3 文献综述结构

标准的文献综述应包含：

1. **引言**：说明本部分的写作目的、结构安排
2. **国内研究现状**：按主题/流派/方法分类评述
3. **国外研究现状**：同上
4. **研究缺口**：指出现有研究的不足
5. **本章小结**：总结本部分对后续研究的启示

### 2.4 文献评述要点

- **综合评述**：不罗列文献，而是综合比较分析
- **批判性思考**：指出已有研究的贡献与局限
- **逻辑衔接**：每段最后一句要引出研究缺口

## 阶段三：研究设计（Research Design）

### 做什么

明确研究方法、研究框架、数据收集方案。

### 3.1 研究方法选择

| 研究目的 | 定量研究 | 定性研究 | 混合研究 |
|---------|---------|---------|---------|
| 探索现象 | 问卷调查 | 访谈、案例 | 问卷+访谈 |
| 验证假设 | 实验、回归 | - | SEM |
| 理解过程 | 追踪调查 | 扎根理论 | 纵向+访谈 |
| 评价效果 | 效果评估 | 实地评价 | 多方法 |

#### 定量研究方法详解

**问卷调查法**：
- 适用场景：验证变量关系、描述群体特征
- 样本计算公式：N = Z²×σ²/E²（Z=置信水平，σ=标准差，E=误差）
- 常用量表：李克特5级/7级量表、语义差异量表
- 质量指标：Cronbach's α ≥ 0.7，KMO ≥ 0.7，Bartlett检验显著

**实验法**：
- 适用场景：因果关系验证
- 设计类型：被试间/被试内/混合设计
- 内部效度：控制额外变量
- 外部效度：样本代表性

**回归分析**：
- 线性回归：连续因变量
- Logistic回归：二分类因变量
- 多元回归：多个自变量

#### 定性研究方法详解

**访谈法**：
- 结构化访谈：预设问题，灵活性低
- 半结构化访谈：灵活调整
- 深度访谈：开放性问题

**案例研究法**：
- 单案例：深入分析
- 多案例：对比分析

**扎根理论**：
- 开放编码：提炼概念
- 主轴编码：建立关联
- 选择编码：核心范畴

### 3.5 问卷设计规范

```markdown
## 问卷设计规范

### 问卷结构
1. 封面信：研究目的、保密承诺、感谢语
2. 基本信息：性别、年龄、学历等
3. 主体问题：量表题+开放题
4. 结束语：感谢、联系方式

### 问题设计原则
- 清晰简洁：每题只问一件事
- 用词准确：避免模糊表述
- 中立无偏：避免引导性提问
- 逻辑连贯：同类问题集中

### 量表设计
| 量表类型 | 选项数 | 适用场景 |
|---------|-------|---------|
| 李克特5级 | 1-5 | 一般研究 |
| 李克特7级 | 1-7 | 精细测量 |
| 语义差异 | 1-7 | 态度测量 |

### 问卷预测试
- 样本量：20-50人
- 检验内容：题目理解、信度、效度
- 访谈反馈：理解偏差、填写时间
```

### 3.6 统计学规范

```markdown
## 统计学写作规范

### 描述性统计
- 连续变量：均值±标准差，或中位数（P25, P75）
- 分类变量：频数（百分比）

**正确示例**：
- 年龄：M=25.3, SD=2.1
- 性别：男124人（52.1%），女114人（47.9%）

### 推断性统计
- t检验：t(df)=值, p=值
- 方差分析：F(df1,df2)=值, p=值
- 相关分析：r=值, p=值
- 回归分析：β=值, SE=值, t=值, p=值

**正确示例**：
- t(98)=3.24, p<.001
- F(2, 197)=5.67, p<.01
- r=.45, p<.001

### 效应量报告
| 检验方法 | 效应量指标 | 判断标准 |
|---------|-----------|---------|
| t检验 | Cohen's d | .2小/.5中/.8大 |
| 方差分析 | η² | .01小/.06中/.14大 |
| 相关 | r | .1小/.3中/.5大 |
| 卡方 | Cramer's V | .1小/.3中/.5大 |

### 4.2 各章写作要点

**绪论**：3-5页，吸引读者，说明"为什么做"和"怎么做"

**文献综述**：8-15页，系统评述，找出缺口，不要罗列要评述

**理论框架**：5-10页，构建理论逻辑，推导研究假设

**研究设计**：5-8页，详细具体，可重复验证

**研究结果**：8-15页，用数据说话，表格规范，统计结果清晰

**结论讨论**：5-10页，回应研究问题，讨论理论贡献和实践意义

## 阶段五：修改定稿（Finalize）

### 做什么

查重降重、格式规范、逻辑优化、最终润色。

### 5.1 查重原理与降重策略

**查重系统工作原理**：
- 按连续N字（通常13字）相同即判定重复
- 识别引用格式，但超过阈值仍算重复
- 数据库包含：论文、网页、书籍、专利等

| 查重率区间 | 风险等级 | 处理建议 |
|-----------|---------|---------|
| <15% | 低风险 | 通过 |
| 15%-25% | 中风险 | 轻度改写 |
| 25%-40% | 高风险 | 中度改写+部分重写 |
| >40% | 极高风险 | 大幅改写或重构 |

**降重技巧（按效果排序）**：

**技巧1：核心词替换法**（降重效果：★★☆，保持原意：★★★★★）
```
原文：研究结果表明，A对B有显著正向影响
改写：A对B的促进作用明显（效应显著）
```

**技巧2：主动变被动/倒装**（降重效果：★★★☆，保持原意：★★★★☆）
```
原文：本研究采用问卷调查法收集数据
改写：数据收集采用问卷调查法
```

**技巧3：拆分合并法**（降重效果：★★★★☆，保持原意：★★★☆☆）
```
原文：随着互联网技术的快速发展，网络购物成为消费者主要购物方式之一
改写：互联网技术近十年发展迅速。在此背景下，网络购物逐渐成为主流消费渠道。
```

**技巧4：扩句缩句法**（降重效果：★★★☆☆，保持原意：★★★★☆）
```
原文：用户粘性是衡量平台价值的重要指标
改写：用户粘性作为衡量互联网平台商业价值的核心指标之一，在学术界和产业界均受到广泛关注
```

**技巧5：图表转换法**（降重效果：★★★★★，保持原意：★★★★★）
- 将文字描述的流程/对比转换为流程图/表格
- 将连续文字描述转为分点列举
- 将数据对比转为图表展示

**技巧6：翻译转换法**（降重效果：★★★★☆，保持原意：★★☆☆☆）
```
原文（英文文献）：The results indicate...
中译后再中译：结果表明...
```

**降重优先级**：
1. 删除非核心重复段落
2. 图表替代文字描述
3. 核心句改写
4. 引用格式调整
5. 增加个人论述

论文数据分析模板

import pandas as pd
import numpy as np
from scipy import stats
import matplotlib.pyplot as plt
import seaborn as sns
from scipy.stats import pearsonr, spearmanr
from sklearn.preprocessing import StandardScaler
import statsmodels.api as sm

# 设置中文显示
plt.rcParams['font.sans-serif'] = ['SimHei', 'Arial Unicode MS']
plt.rcParams['axes.unicode_minus'] = False

# 1. 数据导入
df = pd.read_excel('data.xlsx')

# 2. 数据清洗
df = df.dropna()  # 删除缺失值
df = df[df['age'] >= 18]  # 异常值处理

# 3. 描述性统计
desc_stats = df[['var1', 'var2', 'var3']].describe()
print(desc_stats)

# 4. 信度分析（Cronbach's α）
def cronbach_alpha(df):
    """计算Cronbach's α系数"""
    items = df.shape[1]
    var_sum = df.var(axis=0, ddof=1).sum()
    total_var = df.sum(axis=1).var(ddof=1)
    return (items / (items-1)) * (1 - var_sum/total_var)

alpha = cronbach_alpha(df[['item1', 'item2', 'item3']])
print(f"Cronbach's α = {alpha:.3f}")

# 5. 相关分析
corr, p_value = pearsonr(df['var1'], df['var2'])
print(f"Pearson r = {corr:.3f}, p = {p_value:.3f}")

# 6. 回归分析
X = sm.add_constant(df[['var1', 'var2']])
y = df['outcome']
model = sm.OLS(y, X).fit()
print(model.summary())
