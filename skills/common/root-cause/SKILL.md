# 根因分析 (Root Cause Analysis)

## 触发条件
- 问题诊断
- 质量改进
- 缺陷分析
- 持续改进

## 输入
- 问题现象
- 发生频率
- 影响程度
- 历史数据

## 输出
- 问题分析
- 根因识别
- 解决方案
- 预防措施

## Prompt模板
```
请进行根因分析。

背景：
- 问题现象：[现象]
- 发生频率：[频率]
- 影响程度：[程度]
- 历史数据：[数据]

请输出：
1. 问题定义
2. 数据收集
3. 原因分析
4. 根因识别
5. 解决方案
6. 预防措施
```

## 示例
### 示例1
- 输入：产品缺陷率高，需要找根因
- 输出：根因分析报告，含5Why、鱼骨图、解决方案

## 注意事项
- 要追根溯源
- 要数据说话
- 要防再发
