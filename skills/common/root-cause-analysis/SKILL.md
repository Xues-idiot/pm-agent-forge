# 根因分析 (Root Cause Analysis)

## 触发条件
- 问题分析
- 质量改进
- 故障排查
- 流程优化

## 输入
- 问题现象
- 发生频率
- 影响程度
- 相关数据

## 输出
- 原因分析
- 根因识别
- 解决方案
- 预防措施

## Prompt模板
```
请进行根因分析。

背景：
- 问题现象：[现象]
- 发生频率：[频率]
- 影响程度：[程度]
- 相关数据：[数据]

请输出：
1. 问题描述
2. 可能原因
3. 根因识别
4. 解决方案
5. 预防措施
```

## 示例
### 示例1
- 输入：生产线不良率上升
- 输出：根因分析，使用5Why方法，定位到原材料供应商更换

## 注意事项
- 要追根溯源
- 要数据验证
- 要避免表象
- 要系统思考
