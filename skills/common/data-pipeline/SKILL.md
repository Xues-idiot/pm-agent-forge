# 数据管道 (Data Pipeline)

## 触发条件
- 数据同步
- ETL流程
- 数据入湖
- 实时处理

## 输入
- 数据源
- 数据目标
- 实时性要求
- 数据质量

## 输出
- 管道架构
- 同步策略
- 质量监控
- 故障处理

## Prompt模板
```
请设计数据管道。

背景：
- 数据源：[源]
- 数据目标：[目标]
- 实时性要求：[要求]
- 数据质量：[质量]

请输出：
1. 管道架构
2. 数据同步
3. 转换规则
4. 质量监控
5. 故障处理
```

## 示例
### 示例1
- 输入：业务数据库到数据仓库
- 输出：数据管道，含CDC同步、实时ETL、数据校验、失败重试

## 注意事项
- 要高可靠性
- 要数据一致性
- 要可监控
- 要易于维护
