# 消息架构 (Messaging Architecture)

## 触发条件
- 异步通信
- 系统解耦
- 事件驱动
- 可靠性

## 输入
- 业务场景
- 消息类型
- 可靠性要求
- 规模预期

## 输出
- 架构设计
- 消息模式
- 可靠性方案
- 监控运维

## Prompt模板
```
请设计消息架构。

背景：
- 业务场景：[场景]
- 消息类型：[类型]
- 可靠性要求：[要求]
- 规模预期：[预期]

请输出：
1. 架构设计
2. 消息模式
3. 可靠性方案
4. 顺序保证
5. 监控运维
```

## 示例
### 示例1
- 输入：订单处理流程
- 输出：消息架构，含Kafka消息队列、可靠传输、消费确认、幂等处理

## 注意事项
- 要解耦服务
- 要保证可靠
- 要顺序处理
- 要可观测
