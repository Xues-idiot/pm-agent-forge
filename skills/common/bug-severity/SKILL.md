# 缺陷定级 (Bug Severity)

## 触发条件
- 缺陷管理
- 优先级
- 质量控制
- 发布决策

## 输入
- 缺陷描述
- 影响范围
- 复现概率
- 修复成本

## 输出
- 严重等级
- 优先级
- 修复计划
- 发布决策

## Prompt模板
```
请评估缺陷等级。

背景：
- 缺陷描述：[描述]
- 影响范围：[范围]
- 复现概率：[概率]
- 修复成本：[成本]

请输出：
1. 严重等级
2. 优先级
3. 影响分析
4. 修复计划
5. 发布决策
```

## 示例
### 示例1
- 输入：软件缺陷
- 输出：缺陷定级，P1-Crash类立即修复，P2-功能影响本周修复

## 注意事项
- 要客观评估
- 要考虑影响
- 要平衡质量
- 要及时修复
