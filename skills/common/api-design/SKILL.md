# API设计 (API Design)

## 触发条件
- 接口设计
- 服务集成
- 开发者体验
- API治理

## 输入
- 业务场景
- 用户需求
- 技术架构
- 安全要求

## 输出
- API设计
- 协议选择
- 安全策略
- 文档规范

## Prompt模板
```
请设计API。

背景：
- 业务场景：[场景]
- 用户需求：[需求]
- 技术架构：[架构]
- 安全要求：[要求]

请输出：
1. API设计
2. 接口规范
3. 认证授权
4. 版本管理
5. 文档规范
```

## 示例
### 示例1
- 输入：电商订单服务
- 输出：API设计，含RESTful风格、OpenAPI规范、JWT认证、限流策略

## 注意事项
- 要RESTful规范
- 要版本管理
- 要安全第一
- 要文档完善
