# 容器编排 (Container Orchestration)

## 触发条件
- 容器部署
- 服务管理
- 自动扩缩
- 滚动更新

## 输入
- 应用架构
- 容器数量
- 资源需求
- 发布策略

## 输出
- 编排方案
- 服务配置
- 扩缩策略
- 监控体系

## Prompt模板
```
请设计容器编排。

背景：
- 应用架构：[架构]
- 容器数量：[数量]
- 资源需求：[资源]
- 发布策略：[策略]

请输出：
1. 编排方案
2. 服务配置
3. 扩缩策略
4. 监控体系
5. 日志管理
```

## 示例
### 示例1
- 输入：微服务容器
- 输出：容器编排，Kubernetes、服务网格、自动扩缩

## 注意事项
- 要高可用
- 要自动扩缩
- 要滚动更新
- 要资源隔离
