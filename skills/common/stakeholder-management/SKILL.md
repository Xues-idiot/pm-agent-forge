# 干系人管理 (Stakeholder Management)

## 触发条件
- 识别干系人
- 分析干系人
- 制定沟通策略
- 维护干系人关系

## 输入
- 项目/业务背景
- 涉及部门
- 关键人物
- 历史关系

## 输出
- 干系人分析
- 沟通管理计划
- 关系维护策略
- 冲突解决机制

## Prompt模板
```
请进行[项目/业务]的干系人管理规划。

背景：
- 项目/业务：[背景]
- 涉及部门：[部门]
- 关键人物：[人物]
- 历史关系：[关系]

请输出：
1. 干系人识别清单
2. 干系人分析（权力-利益矩阵）
3. 干系人分类和优先级
4. 沟通策略（各干系人）
5. 期望管理计划
6. 冲突预警和处理机制
7. 关系维护计划
8. 干系人参与度评估
```

## 示例
### 示例1
- 输入：ERP系统实施，涉及IT部门、各业务部门、高管、外部供应商
- 输出：干系人管理计划，含沟通策略、冲突处理

## 注意事项
- 不要遗漏关键干系人
- 沟通要个性化
- 关系要持续维护
