# 微服务设计 (Microservices Design)

## 触发条件
- 服务拆分
- 架构设计
- 服务治理
- 运维复杂度

## 输入
- 业务领域
- 团队结构
- 技术选型
- 运维能力

## 输出
- 服务划分
- 架构设计
- 治理策略
- 运维方案

## Prompt模板
```
请设计微服务架构。

背景：
- 业务领域：[领域]
- 团队结构：[结构]
- 技术选型：[选型]
- 运维能力：[能力]

请输出：
1. 服务划分
2. 架构设计
3. 服务通信
4. 数据管理
5. 运维方案
```

## 示例
### 示例1
- 输入：电商平台
- 输出：微服务设计，含用户、商品、订单、支付、物流等服务的领域划分

## 注意事项
- 要领域驱动
- 要服务独立
- 要考虑团队
- 要平衡粒度
