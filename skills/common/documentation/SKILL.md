# Documentation Skills

文档写作与维护。

## 何时使用

- 技术文档编写
- 产品文档编写
- 知识库维护
- 文档规范制定
- 文档质量提升

## 文档结构

### 结构设计

```markdown
## 文档结构

### 文档结构模板
```markdown
## 文档模板

1. 概述
   - 背景
   - 目的
   - 范围

2. 主体内容
   - 详细章节

3. 附录
   - 名词解释
   - 参考资料
   - 版本历史
```

### 层次结构
| 层次 | 说明 | 标记 |
|------|------|------|
| H1 | 文档标题 | # |
| H2 | 一级章节 | ## |
| H3 | 二级章节 | ### |
| H4 | 三级章节 | #### |

### 段落结构
```markdown
## 段落组织

### 好的段落
- 主题句（开头）
- 支持句（解释）
- 结尾句（过渡）

### 段落长度
- 一般段落：3-5句
- 复杂段落：可更长
- 简洁为要
```
```

## 写作技巧

### 语言风格

```markdown
## 语言风格

### 简洁
| 问题 | 修改 |
|------|------|
| 冗长句子 | 拆分 |
| 重复表达 | 删除 |
| 专业术语 | 解释或替换 |
| 被动语态 | 主动语态 |

### 清晰
| 技巧 | 说明 |
|------|------|
| 具体 | 不用模糊词 |
| 准确 | 用词精准 |
| 完整 | 无歧义 |
| 逻辑 | 连贯流畅 |

### 一致性
| 方面 | 要求 |
|------|------|
| 术语 | 全文统一 |
| 格式 | 风格统一 |
| 时态 | 叙述一致 |
| 人称 | 避免混乱 |
```

- 内容不准确
- 结构混乱
- 格式不统一
- 长期不更新
- 无维护机制
- 协作不顺畅
- 文档找不到
