# 最小可行产品 (MVP)

## 触发条件
- 快速验证
- 精益创业
- 资源效率
- 学习迭代

## 输入
- 核心假设
- 用户痛点
- 资源约束
- 验证目标

## 输出
- MVP设计
- 核心功能
- 验证指标
- 迭代计划

## Prompt模板
```
请设计MVP。

背景：
- 核心假设：[假设]
- 用户痛点：[痛点]
- 资源约束：[约束]
- 验证目标：[目标]

请输出：
1. MVP定义
2. 核心功能
3. 验证指标
4. 假设验证
5. 迭代计划
```

## 示例
### 示例1
- 输入：新创项目
- 输出：MVP设计，Landing Page→预约演示→Beta测试渐进

## 注意事项
- 要聚焦核心
- 要快速验证
- 要低成本
- 要持续学习
