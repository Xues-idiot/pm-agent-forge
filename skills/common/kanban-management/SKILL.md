# 看板管理 (Kanban Management)

## 触发条件
- 流程可视化
- 限制在制品
- 流动优化
- 持续交付

## 输入
- 流程阶段
- 团队配置
- 瓶颈识别
- 交付目标

## 输出
- 看板设计
- WIP限制
- 流动分析
- 改进计划

## Prompt模板
```
请管理看板。

背景：
- 流程阶段：[阶段]
- 团队配置：[配置]
- 瓶颈识别：[瓶颈]
- 交付目标：[目标]

请输出：
1. 看板设计
2. WIP限制
3. 流动分析
4. 瓶颈识别
5. 改进计划
```

## 示例
### 示例1
- 输入：持续交付团队
- 输出：看板管理，含To Do-In Progress-Review-Test-Deploy、瓶颈可视化

## 注意事项
- 要可视化
- 要限制WIP
- 要管理流动
- 要持续改进
