# 微服务架构 (Microservices Architecture)

## 触发条件
- 系统拆分
- 服务治理
- 架构演进
- 运维复杂度

## 输入
- 业务边界
- 服务拆分
- 技术选型
- 团队结构

## 输出
- 架构设计
- 服务划分
- 通信机制
- 部署策略

## Prompt模板
```
请设计微服务架构。

背景：
- 业务边界：[边界]
- 服务拆分：[拆分]
- 技术选型：[选型]
- 团队结构：[团队]

请输出：
1. 架构设计
2. 服务划分
3. 通信机制
4. 部署策略
5. 监控体系
```

## 示例
### 示例1
- 输入：电商系统
- 输出：微服务架构，用户、订单、商品、支付服务拆分

## 注意事项
- 要服务自治
- 要独立部署
- 要技术多样
- 要治理完善
