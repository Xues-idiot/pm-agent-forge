# 产品待办管理 (Backlog Grooming)

## 触发条件
- 需求梳理
- 优先级排序
- 工作量评估
- Sprint准备

## 输入
- 用户故事
- 优先级
- 估算标准
- 团队能力

## 输出
- 故事完善
- 估算确认
- 优先级调整
- Sprint准备

## Prompt模板
```
请管理产品待办。

背景：
- 用户故事：[故事]
- 优先级：[优先]
- 估算标准：[标准]
- 团队能力：[能力]

请输出：
1. 故事完善
2. 验收标准
3. 工作量估算
4. 优先级确认
5. Sprint准备
```

## 示例
### 示例1
- 输入：产品待办列表
- 输出：待办管理，含User Story完善、INVEST原则、故事点估算、优先级排序

## 注意事项
- 要持续进行
- 要团队参与
- 要INVEST原则
- 要明确验收
