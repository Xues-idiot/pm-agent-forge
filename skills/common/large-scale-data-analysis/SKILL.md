# 大规模数据分析与可视化报告 (Large-Scale Data Analysis & Visualization Report)

## 触发条件
- 数据量达到万级以上（10万+行）
- 需要深度数据分析
- 需要生成可视化报告
- 需要确保数据完整性和可追溯性

## 输入
- **数据源**：CSV/Excel/数据库/API等原始数据
- **分析目标**：业务问题、假设验证、趋势探索
- **数据量级**：行数、列数、数据字段定义
- **保密级别**：公开/内部/机密（最低优先级）

## 输出
- **数据分析报告**：
  - 执行摘要（结论先行）
  - 数据概览（行列数、字段说明）
  - 描述性统计
  - 趋势分析
  - 异常发现
  - 结论与建议
- **可视化图表**：趋势图、分布图、对比图、关系图等
- **数据追溯清单**：每条数据结论对应的原始数据行列坐标

## 数据完整性保障

### 行列坐标标注规范
所有数据分析结论必须标注数据来源的具体位置：

| 结论类型 | 标注格式 | 示例 |
|---------|---------|------|
| 汇总值 | `[Sheet/Table]!Row[行号]:Col[列号]` | `Sheet1!Row5:Col3` |
| 样本数据 | `[Sheet/Table]![起始行]-[结束行],[列名]` | `Data!100-105,销售额` |
| 统计指标 | `[Sheet/Table]![统计范围]` | `Sales![均值计算区间]` |

### 数据覆盖率要求
- **100%数据覆盖**：每个分析结论必须可追溯
- **异常值标注**：每条异常数据需标注 `[Row:行号, Col:列名]`
- **缺失值标注**：缺失数据需标注 `[Row:行号, Col:列名] = NULL/NA`

## 注意事项

### 数据完整性（最高优先级）
- **禁止跳过任何数据**：即使异常值也要标注后保留
- **行列坐标必须精确**：每条结论都能在原始数据中找到对应行
- **使用索引而非推断**：不基于采样推断，基于全量数据计算

### 分析严谨性
- 统计结论需注明样本范围：`[Row:起始-结束]`
- 异常判断需注明判断标准：如"超过3个标准差"
- 显著性结论需注明置信水平：如"95%置信度下显著"

### 可视化规范
- 图表必须有标题、坐标轴标签、数据来源标注
- 图表中关键数据点建议直接标注数值
- 不同图表类型适用于不同分析场景：
  | 分析目的 | 推荐图表 |
  |---------|---------|
  | 趋势变化 | 折线图 |
  | 占比构成 | 饼图/堆叠柱图 |
  | 对比分析 | 柱图/条形图 |
  | 分布情况 | 直方图/箱线图 |
  | 相关关系 | 散点图 |
  | 地理分布 | 地图热力图 |

### 保密处理（最低优先级）
- 报告中可隐藏敏感字段（如手机号、身份证）
- 隐藏方式：部分打码如 `138****5678`
- 隐藏字段需在报告开头注明
- 整体报告保密级别仅影响展示层，不影响分析流程

### 性能优化（10万+行数据）
- 分批处理：每批1-2万行，避免内存溢出
- 增量分析：先采样探索，再全量验证
- 聚合优先：能聚合计算的不逐行处理
- 向量化：使用pandas/numpy的向量化操作
