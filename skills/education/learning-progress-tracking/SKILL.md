# Education Learning Progress Tracking Skill

追踪学习进度，识别学习问题，生成进度报告。

## 何时使用

- 监控学习进度
- 识别学习问题
- 生成学习报告
- 制定改进计划
- 激励学习者

## 进度追踪流程

```
1. 建立基线 → 2. 数据收集 → 3. 进度分析 → 4. 问题识别 → 5. 干预建议 → 6. 报告输出
```


```markdown

### 与计划对比
| 阶段 | 计划进度 | 实际进度 | 差距 |
|------|---------|---------|------|
```

### 4. 问题识别

**学习问题分类：**

| 问题类型 | 表现 | 原因分析 |
|---------|------|---------|
| 参与度低 | 登录少、学习时间短 | 动力不足/时间不够 |
| 理解困难 | 测验得分低 | 基础薄弱/方法不当 |
| 进度落后 | 完成率低于计划 | 难度过高/时间不够 |
| 遗忘 | 已学内容回生 | 缺少复习 |

**问题识别模板：**

```markdown
## 问题识别

### 学习者：[ID/姓名]
### 分析周期：[时间范围]

#### 问题1
- 表现：[具体表现]
- 原因：[分析原因]
- 严重程度：[高/中/低]

#### 问题2
...
```

### 5. 干预建议

**干预策略：**

| 问题类型 | 干预策略 |
|---------|---------|
| 参与度低 | 提醒激励、学习计划调整 |
| 理解困难 | 补充材料、辅导答疑 |
| 进度落后 | 调整计划、优先级排序 |
| 遗忘 | 复习提醒、间隔练习 |

## 进度追踪指标

| 指标类别 | 指标名称 | 计算方式 |
|---------|---------|---------|
| 参与度 | 活跃天数 | 登录天数/总天数 |
| 完成度 | 内容完成率 | 已完成内容/总内容 |
| 表现度 | 测验得分率 | 得分/满分 |
| 效率 | 学习投入产出比 | 得分/学习时长 |

## 预警机制

```markdown
## 预警规则

### 黄色预警（需关注）
- 进度差距 > 10%
- 测验得分 < 60%
- 连续3天未学习

### 红色预警（需干预）
- 进度差距 > 20%
- 测验得分 < 40%
- 连续7天未学习
```

## 进度追踪检查清单

- [ ] 基线评估完成
- [ ] 数据收集完整
- [ ] 进度计算准确
- [ ] 问题识别及时
- [ ] 干预建议有效
- [ ] 报告输出清晰

## 红牌警告

- 数据采集不完整
- 问题识别滞后
- 干预建议不针对
- 报告与实际不符
- 预警机制失效
- 学习者不知情
