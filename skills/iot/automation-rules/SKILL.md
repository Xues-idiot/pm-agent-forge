# IoT Automation Rules Skill

设计 IoT 自动化规则，实现设备联动和智能控制。

## 何时使用

- 设计设备联动场景
- 配置自动化规则
- 创建触发-动作规则
- 设置场景模式
- 优化自动化流程
- 规则效果评估

## 自动化设计流程

```
1. 需求分析 → 2. 场景设计 → 3. 规则配置 → 4. 测试验证 → 5. 上线运行 → 6. 效果优化
```

### 1. 需求分析

**自动化场景分类：**

| 类别 | 描述 | 示例 |
|------|------|------|
| 告警联动 | 异常触发告警和处理 | 温度>50°触发告警+降温 |
| 场景联动 | 场景触发设备动作 | 开门开灯 |
| 定时动作 | 定时执行控制 | 每天8点开关 |
| 数据驱动 | 阈值触发动作 | PM2.5>100开净化器 |

### 2. 场景设计

**场景设计模板：**

```markdown
## 场景设计

### 场景概述
- 场景名称：
- 触发设备：
- 目标设备：
- 应用场景：

### 触发条件
```yaml
条件类型: data  # data/time/event/device
条件表达式: >
  - 变量: device.temperature
  - 值: 30

持续时间: 5s  # 条件需持续多久才触发

附加条件:
  - device.status == "online"
  - time.hour in [9, 18]
```

### 执行动作
| 步骤 | 设备 | 动作 | 参数 | 延迟 |
|------|------|------|------|------|
| 1 | 空调 | 开启 | 模式: 制冷, 温度: 26℃ | 0s |
| 2 | 风扇 | 开启 | 档位: 2 | 2s |
| 3 | 告警灯 | 闪烁 | 次数: 3 | 0s |

### 状态管理

### 退出条件

- 用户手动关闭

### 互斥设计

### 3. 规则配置

**规则配置模板：**

```markdown
## 自动化规则配置

### 规则基本信息
| 字段 | 内容 |
|------|------|
| 规则名称 | |
| 规则ID | |
| 优先级 | P1/P2/P3 |
| 启用状态 | 是/否 |

### 触发配置
```yaml
触发类型: data_change  # data_change/time/composite

触发条件:
  - 数据点: temperature
    操作符: ">"
    值: 30
    设备: sensor_001

持续条件: 30s  # 条件持续30秒才触发
```

### 动作配置
```yaml
动作列表:
  - 序号: 1
    设备: ac_001
    命令: set_mode
    参数:
      mode: cool
      temperature: 26
    延迟: 0s

  - 序号: 2
    设备: fan_001
    命令: turn_on
    参数:
      level: 2
    延迟: 2s
```

### 条件判断
```yaml
执行条件:
  - 设备在线: [ac_001, fan_001]
  - 时间范围: 08:00-22:00
  - 星期: [1,2,3,4,5]  # 工作日
```

### 异常处理
| 异常情况 | 处理方式 |
|---------|---------|
| 设备不在线 | 跳过该动作/告警通知 |
| 动作执行失败 | 重试3次/跳过/告警 |
| 条件频繁触发 | 增加冷却时间30s |
```

## 常用自动化场景

### 场景1：智能办公

```markdown
## 智能办公场景

### 场景1：人来灯亮
- 触发：人体感应器检测到人
- 动作：打开灯光
- 退出：无人持续5分钟，关闭灯光

### 场景2：温控节能
- 触发：温度>28℃且有人
- 动作：开启空调26℃
- 退出：温度<26℃或无人持续30分钟

### 场景3：下班巡检
- 触发：每天18:00
- 动作：关闭所有灯光、空调
- 条件：非工作日不执行
```

### 场景2：智能工厂

```markdown
## 智能工厂场景

### 场景1：设备异常告警
- 触发：设备温度>80℃或振动异常
- 动作：发送告警+记录日志
- 处理：通知维护人员

### 场景2：能耗优化
- 触发：功率>设定阈值
- 动作：关闭非必要设备
- 退出：功率恢复正常

### 场景3：质量检测
- 触发：检测到缺陷品
- 动作：标记+分流+告警
- 数据：记录缺陷图片和参数
```

## 红牌警告

- 触发条件过于宽泛
- 动作可能导致设备损坏
- 缺少互斥机制
- 异常处理不完善
- 规则产生死循环
- 缺少安全限位
- 未经充分测试上线
