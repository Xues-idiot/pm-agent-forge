# System Integration

系统集成，物联网系统与企业应用的集成。

## 何时使用

- IoT平台搭建
- 设备接入
- 数据集成
- 协议选型
- 系统架构

## 集成架构

### 典型架构

```markdown
## IoT集成架构

### 分层架构
| 层级 | 功能 | 组件 |
|------|------|------|
| 设备层 | 感知/执行 | 传感器、执行器 |
| 接入层 | 协议转换 | 网关、边缘计算 |
| 平台层 | 数据处理 | IoT平台 |
| 应用层 | 业务应用 | 应用系统 |

### 数据流
```
设备 → 网关 → IoT平台 → 数据处理 → 应用系统
              ↓
          数据存储
```
```

### 集成模式

```markdown
## 集成模式

### 点对点集成
- 设备直连应用
- 简单但耦合高
- 适用于小规模

### 星型集成
- 通过IoT平台中转
- 解耦好
- 适用中等规模

### 总线集成
- 通过消息总线
- 扩展性好
- 适用大规模
```

## 协议选择

### 设备协议

```markdown
## 设备协议

### 常见协议
| 协议 | 特点 | 适用 |
|------|------|------|
| MQTT | 轻量、发布订阅 | 低功耗 |
| CoAP | RESTful、UDP | 受限设备 |
| HTTP | 通用、Web友好 | 资源充足 |
| LwM2M | 设备管理 | 运营商设备 |

### MQTT vs HTTP
| 对比 | MQTT | HTTP |
|------|------|------|
| 功耗 | 低 | 高 |
| 效率 | 高 | 中 |
| 可靠性 | 可配置 | 请求-响应 |
| 场景 | 设备上报 | 命令下发 |
```

## 消息路由

### 路由设计

```markdown
## 消息路由

### 路由规则
| 规则 | 说明 | 示例 |
|------|------|------|
| 主题路由 | 按主题 | device/+/data |
| 内容路由 | 按内容 | temperature>30 |
| 标签路由 | 按标签 | location=beijing |

### 路由配置
```yaml
## 路由规则示例

routes:
  - name: temperature-alert
    topic: device/+/temperature
    filter: value > 30
    target: alert-system

  - name: storage-route
    topic: device/+/data
    target: storage-service
```
```
