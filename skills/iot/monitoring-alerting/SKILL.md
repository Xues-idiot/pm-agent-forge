# IoT Monitoring and Alerting Skill

监控 IoT 系统，设置告警规则，处理异常。

## 何时使用

- 配置监控看板
- 设置告警规则
- 分析告警原因
- 处理设备异常
- 优化告警策略
- 根因分析

## 监控告警流程

```
1. 确定监控目标 → 2. 设计监控指标 → 3. 配置告警规则 → 4. 告警响应 → 5. 根因分析 → 6. 优化改进
```

### 1. 确定监控目标

**监控层次：**

| 层次 | 监控内容 | 指标示例 |
|------|---------|---------|
| 设备层 | 单个设备状态 | 在线率、电量、温度 |
| 网关层 | 网关连接状态 | 连接数、流量 |
| 平台层 | 系统整体健康 | API响应、消息延迟 |
| 业务层 | 业务指标 | 数据采集量、告警处理率 |

### 2. 设计监控指标

**设备监控指标：**

```markdown
## 设备监控指标

### 基础状态指标
| 指标 | 计算方式 | 告警阈值 | 严重程度 |
|------|---------|---------|---------|
| 在线率 | 在线数/总数×100% | <95% | 高 |
| 响应超时率 | 超时数/请求数×100% | >5% | 高 |
| 数据上报率 | 实际上报/应上报×100% | <90% | 中 |

### 性能指标
| 指标 | 正常范围 | 告警阈值 | 严重程度 |
|------|---------|---------|---------|
| 信号强度 | >-80dBm | <-90dBm | 中 |
| 电池电量 | >30% | <10% | 高 |
| 温度 | 20-45°C | >50°C | 高 |
| 湿度 | 30-70% | >80% | 中 |
```

**系统监控指标：**

```markdown

### 平台指标

|------|-------|---------|---------|

### 3. 配置告警规则

**告警规则模板：**

```markdown
## 告警规则配置

### 规则基本信息
| 字段 | 内容 |
|------|------|
| 规则名称 | |
| 规则ID | |
| 优先级 | P1/P2/P3 |
| 启用状态 | 是/否 |

### 触发条件
```yaml
条件类型: threshold  # threshold/complex/event
指标: device.temperature
操作符: >  # >/</>=/<=/==
阈值: 50
持续时间: 5m  # 持续多久才触发
```

### 告警动作
| 动作类型 | 配置 |
|---------|------|
| 平台通知 | 告警列表 + 推送 |
| 短信通知 | P1告警发送给XX |
| 邮件通知 | 发送给YY |
| Webhook | 回调URL: http://... |
| 自动处理 | 重启设备/切换模式 |
```

**告警分级：**

| 级别 | 定义 | 响应时间 | 通知方式 |
|------|------|---------|---------|
| P1 | 严重，影响业务 | 15min | 短信+电话 |
| P2 | 较重，需要处理 | 1h | 推送+邮件 |
| P3 | 一般，关注即可 | 4h | 告警列表 |
| P4 | 提示，无需处理 | - | 日志记录 |

### 6. 优化改进

**告警优化策略：**

```markdown
## 告警优化

### 当前问题
- 告警过多，噪音大
- 重复告警频繁
- 告警处理时间长

### 优化措施

#### 告警收敛
| 策略 | 说明 | 预期效果 |
|------|------|---------|
| 告警聚合 | 同一设备多个告警聚合 | 减少30% |
| 告警抑制 | 父级告警抑制子级 | 减少20% |
| 告警升级 | 一定时间未处理自动升级 | 提高处理率 |

#### 告警智能
| 策略 | 说明 | 预期效果 |
|------|------|---------|
| 动态阈值 | 根据历史数据动态调整 | 减少误报 |
| 关联分析 | 关联多个指标判断 | 提高准确性 |
| 预测告警 | 基于趋势预测告警 | 提前发现问题 |
```

## 监控告警检查清单

- [ ] 监控指标覆盖全面
- [ ] 告警阈值合理
- [ ] 告警分级清晰
- [ ] 通知机制有效
- [ ] 处理流程规范
- [ ] 根因分析完整
- [ ] 优化措施落实

## 红牌警告

- 告警阈值设置不当（过高或过低）
- 告警未及时处理
- 重复告警未优化
- 告警根因未找到
- 告警风暴未处理
- 监控盲区未发现
- 告警配置变更无记录
