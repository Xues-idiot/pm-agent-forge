# IoT Data Collection Skill

设计和管理 IoT 数据采集、传输、存储流程。

## 何时使用

- 设计数据采集方案
- 配置数据采集参数
- 数据传输优化
- 数据质量验证
- 数据存储设计
- 数据生命周期管理

## 数据采集流程

```
1. 需求分析 → 2. 方案设计 → 3. 采集配置 → 4. 传输处理 → 5. 存储管理 → 6. 质量保障
```

### 1. 需求分析

**数据需求收集：**

```markdown
## 数据采集需求

### 业务需求
| 需求项 | 描述 | 优先级 |
|--------|------|--------|
| 实时监控 | 设备状态实时展示 | P0 |
| 历史分析 | 趋势分析和报表 | P1 |
| 故障追溯 | 问题根因分析 | P1 |
| 机器学习 | AI模型训练数据 | P2 |

### 数据规格
| 规格项 | 要求 |
|--------|------|
| 数据类型 | 数值/文本/二进制 |
| 采集频率 | 实时/分钟/小时 |
| 数据量级 | 单设备X条/天，总量X万条/天 |
| 延迟要求 | <X秒 |
| 可用性要求 | >99.9% |
```

### 2. 方案设计

**数据采集架构：**

```markdown
## 数据采集架构

### 架构设计
```
设备 → 网关 → 消息队列 → 处理层 → 存储层
              ↓
         实时计算
              ↓
         告警触发
```

### 协议选择
| 协议 | 适用场景 | 特点 |
|------|---------|------|
| MQTT | 设备→平台 | 轻量、低功耗 |
| CoAP | 资源受限设备 | UDP、RESTful |
| HTTP | 通用场景 | 广泛支持 |
| LwM2M | 电信行业 | 标准化 |

### 消息格式
```json
{
  "deviceId": "xxx",
  "timestamp": 1234567890,
  "type": "telemetry",
  "data": {
    "temperature": 25.5,
    "humidity": 60
  }
}
```
```

### 3. 采集配置

**设备数据配置：**

```markdown
## 数据点配置

### 数据点定义
| 数据点 | 类型 | 单位 | 采集间隔 | 存储策略 |
|--------|------|------|---------|---------|
| temperature | float | ℃ | 60s | 全量存储 |
| humidity | float | % | 60s | 全量存储 |
| battery | int | % | 300s | 全量存储 |
| status | string | - | 60s | 全量存储 |

### 采集参数
| 参数 | 值 | 说明 |
|------|---|---|
| 采集间隔 | 60s | 设备主动上报 |
| 上报策略 | 变化上报+周期上报 | 变化时立即上报 |
| 压缩 | 启用 | 减少传输量 |
```

### 4. 传输处理

**消息队列配置：**

```markdown
## 消息队列配置

### Topic 设计
| Topic | 用途 | QoS |
|-------|------|-----|
| device/{id}/telemetry | 设备遥测数据 | 1 |
| device/{id}/attribute | 设备属性 | 1 |
| device/{id}/command | 设备命令 | 2 |
| device/{id}/event | 设备事件 | 1 |

### 消息处理
| 处理阶段 | 处理内容 | 组件 |
|---------|---------|------|
| 接入层 | 协议转换、认证 | 网关/Broker |
| 消息层 | 消息路由、过滤 | Kafka/RabbitMQ |
| 计算层 | 数据解析、聚合 | Flink/Spark |
| 存储层 | 数据写入 | InfluxDB/TSDB |
```

**数据处理规则：**

```markdown
## 数据处理规则

### 实时处理
| 规则 | 条件 | 动作 |
|------|------|------|
| 告警触发 | temperature > 50 | 产生告警事件 |
| 状态变更 | status != lastStatus | 更新设备状态 |
| 数据聚合 | 每5分钟 | 计算平均值 |

### 离线处理
| 规则 | 执行频率 | 输出 |
|------|---------|-----|
| 小时聚合 | 每小时 | 每小时均值 |
| 天聚合 | 每天 | 每天统计 |
| 月聚合 | 每月 | 每月报表 |
```

### 5. 存储管理

**存储方案设计：**

```markdown
## 存储方案

### 存储层次
| 层次 | 存储类型 | 保留时间 | 用途 |
|------|---------|---------|------|
| 热数据 | TSDB/InfluxDB | 30天 | 实时查询 |
| 温数据 | 时序数据库 | 6个月 | 分析报表 |
| 冷数据 | 对象存储 | 2年 | 历史归档 |

### 数据压缩
| 策略 | 方法 | 压缩比 |
|------|------|--------|
| 死区间压缩 | 变化小于阈值时不存储 | 30-50% |
| 差分编码 | 存储差值而非绝对值 | 40-60% |
| 时间对齐 | 合并同时间段数据 | 20-30% |
```

### 6. 质量保障

**数据质量检查：**

```markdown
## 数据质量检查

### 质量维度
| 维度 | 定义 | 检查方法 |
|------|------|---------|
| 完整性 | 无数据丢失 | 统计对比 |
| 准确性 | 数据正确 | 范围校验 |
| 一致性 | 多源数据一致 | 交叉验证 |
| 时效性 | 数据及时 | 时间戳检查 |

### 质量监控
| 指标 | 目标值 | 告警阈值 |
|------|-------|---------|
| 数据完整率 | >99.9% | <99% |
| 数据准确率 | >99.5% | <99% |
| 数据延迟 | <5s | >30s |

### 异常处理
```markdown
## 数据异常处理

### 异常类型
| 类型 | 检测方法 | 处理方式 |
|------|---------|---------|
| 缺失值 | 统计对比 | 插值/清除 |
| 异常值 | 3σ原则/IQR | 标记/过滤 |
| 重复值 | 主键去重 | 删除重复 |
| 格式错误 | 格式校验 | 修复/丢弃 |
```
```

## 数据生命周期管理

**数据生命周期：**

```markdown
## 数据生命周期

### 阶段管理
| 阶段 | 时间范围 | 操作 | 存储成本 |
|------|---------|------|---------|
| 生成 | T+0 | 实时处理 | - |
| 短期存储 | T+0~30d | 高性能存储 | 高 |
| 中期存储 | 30d~6m | 普通存储 | 中 |
| 归档 | 6m~2y | 低成本存储 | 低 |
| 删除 | >2y | 数据销毁 | - |

### 数据归档规则
- [ ] 按时间自动归档
- [ ] 归档前数据验证
- [ ] 归档后元数据保留
- [ ] 支持按需查询归档数据
```

## 数据采集检查清单

- [ ] 数据需求分析完整
- [ ] 采集方案设计合理
- [ ] 协议选择合适
- [ ] 传输可靠性保证
- [ ] 存储成本优化
- [ ] 数据质量有保障
- [ ] 生命周期管理规范

## 红牌警告

- 采集频率过高导致带宽浪费
- 未考虑数据量增长
- 存储方案成本失控
- 数据质量未监控
- 数据传输未加密
- 敏感数据未脱敏
- 备份恢复方案缺失
