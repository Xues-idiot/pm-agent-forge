# IoT Security Skill

保护物联网系统，管理设备安全，实施物联网网络安全措施。

## 何时使用

- IoT安全评估
- 设备安全管理
- 安全策略制定
- 漏洞管理
- 安全监控
- 合规审计

## IoT安全流程

```
1. 安全评估 → 2. 策略设计 → 3. 防护实施 → 4. 监控运营 → 5. 应急响应
```

### 4. 监控运营

**安全监控：**

```markdown
## 安全监控

### 监控指标
| 指标 | 检测方法 | 告警阈值 |
|------|----------|----------|
| 异常登录 | 登录日志分析 | 失败>5次 |
| 异常流量 | NetFlow分析 | 流量突增>50% |
| 设备离线 | 心跳检测 | 离线>5分钟 |
| 固件异常 | 版本校验 | 版本不一致 |
| 证书过期 | 证书监控 | <30天过期 |

### 日志管理
```markdown
## 日志规范

### 必须记录
- [ ] 设备登录
- [ ] 配置变更
- [ ] 固件更新
- [ ] 告警事件
- [ ] 系统错误

### 日志要求
- 内容：时间+设备+用户+操作+结果
- 存储：加密存储，保留>6个月
- 传输：安全通道传输
```
```

### 5. 应急响应

**应急预案：**

```markdown
## 应急响应

### 事件分级
| 等级 | 事件类型 | 响应时间 | 响应级别 |
|------|----------|----------|----------|
| P0 | 大规模设备被控 | 立即 | 公司级 |
| P1 | 核心系统入侵 | 1小时 | 部门级 |
| P2 | 单点安全事件 | 4小时 | 团队级 |
| P3 | 安全隐患 | 24小时 | 日常处理 |

### 响应流程
```markdown
## 响应流程

发现事件
    ↓
初步评估 → 确认事件
    ↓
启动响应 → 遏制措施
    ↓
调查分析 → 定位原因
    ↓
恢复系统 → 验证安全
    ↓
复盘总结 → 改进措施
```
```

- 设备弱密码

- 网络未隔离
- 数据未加密
- 日志未记录
- 漏洞未修复
- 安全事件未响应
- 重大数据泄露
