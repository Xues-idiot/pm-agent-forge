# Performance Management

绩效管理。

## 何时使用

- 绩效考核
- 绩效评估
- 绩效面谈
- 绩效改进
- OKR管理

## 绩效体系

### 绩效框架

```markdown
## 绩效框架

### 绩效组成部分
| 部分 | 说明 | 权重 |
|------|------|------|
| 目标达成 | 结果导向 | 40-60% |
| 能力发展 | 能力提升 | 20-30% |
| 价值观 | 文化践行 | 10-20% |

### 绩效周期
| 周期 | 说明 | 适用 |
|------|------|------|
| 年度绩效 | 一年一次 | 综合评估 |
| 半年度 | 半年一次 | 复盘调整 |
| 季度绩效 | 季度 | 快速迭代 |
| 月度考核 | 月度 | 结果导向 |

### OKR与绩效
```markdown
## OKR vs 绩效考核

### OKR
- 目标对齐
- 挑战性目标
- 不直接挂钩薪酬
- 适用于创新业务

### 绩效考核
- 结果评价
- 与薪酬挂钩
- 适用于执行层
```
```

## 目标管理

### 目标设定

```markdown
## 目标设定

### 目标原则
| 原则 | 说明 | 应用 |
|------|------|------|
| SMART | 具体/可衡量 | 目标质量 |
| 上下对齐 | 与公司目标对齐 | 方向一致 |
| 挑战性 | 略高于能力 | 激发潜力 |
| 透明 | 目标公开 | 协同 |

### 目标分解
```markdown
## 目标分解

### 分解逻辑
公司目标 → 部门目标 → 个人目标

### 分解维度
- 时间维度：年度→季度→月度
- 职责维度：各岗位职责
- 协作维度：需要配合的目标

### 目标对齐
- 纵向对齐：上下级
- 横向对齐：跨部门
```
```

### 目标跟踪

```markdown
## 目标跟踪

### 跟踪机制
| 频率 | 内容 | 形式 |
|------|------|------|
| 周review | 本周进展 | 1v1 |
| 月复盘 | 月度进展 | 会议 |
| 季度评估 | 季度总结 | 评估 |

### 跟踪要点
- 进度跟踪
- 风险识别
- 资源协调
- 目标调整
```

## 绩效评估

### 评估方法

```markdown
## 绩效评估

### 评估方式
| 方式 | 特点 | 适用 |
|------|------|------|
| 目标考核 | 结果导向 | 可量化岗位 |
| 360评估 | 多方反馈 | 管理岗位 |
| 行为评估 | 行为表现 | 软技能 |
| 目标+行为 | 综合评估 | 大多数岗位 |

### 评估流程
```markdown
## 评估流程

1. 目标确认
   - 年初目标确认
   - 变更记录

2. 中期回顾
   - 进展跟踪
   - 调整确认

3. 年终评估
   - 结果收集
   - 自评
   - 他评

4. 校准会议
   - 结果校准
   - 分布控制

5. 结果反馈
   - 面谈
   - 签字确认
```
```

### 评估校准

```markdown
## 评估校准

### 校准目的
- 确保标准一致
- 控制分布比例
- 减少主观偏差

### 校准会议
| 环节 | 内容 | 参与者 |
|------|------|--------|
| 目标回顾 | 目标达成情况 | 管理者 |
| 绩效阐述 | 被评估者表现 | HR/管理者 |
| 校准讨论 | 结果校准 | 管理层 |
| 结果确认 | 最终结果 | HR |

### 分布控制
| 等级 | 比例 | 说明 |
|------|------|------|
| A | 10-15% | 超出期望 |
| B | 60-70% | 达到期望 |
| C | 15-20% | 低于期望 |
| D | 5%以内 | 不合格 |
```

## 绩效面谈

### 面谈准备

```markdown
## 绩效面谈

### 面谈准备
| 角色 | 准备内容 |
|------|----------|
| 管理者 | 评估数据/案例/反馈 |
| 员工 | 自评/问题/发展期望 |

### 面谈结构
```markdown
## 面谈结构

1. 开场（5min）
   - 建立氛围
   - 明确目的

2. 自评分享（10min）
   - 员工自评
   - 亮点/不足

3. 评估反馈（15min）
   - 评估结果
   - 具体案例

4. 发展讨论（15min）
   - 优势发挥
   - 改进方向
   - 下年目标

5. 总结收尾（5min）
   - 共识确认
   - 后续安排
```
```

### 面谈技巧

```markdown
## 面谈技巧

### 反馈技巧
| 技巧 | 说明 | 示例 |
|------|------|------|
| B-I-F | 行为-影响-期待 | 具体行为 |
| 汉堡法则 | 正面-改进-正面 | 一般反馈 |
|  SBI | 情境-行为-影响 | 反馈 |

### 负面反馈
- 具体事实
- 避免指责
- 共同分析原因
- 制定改进计划
- 表达信任支持
```

## 绩效改进

### 改进计划

```markdown
## 绩效改进

### 改进对象
| 情况 | 处理 | 说明 |
|------|------|------|
| 绩效不达标 | PIP | 绩效改进计划 |
| 行为问题 | 辅导 | 行为改进 |
| 能力不足 | 培训 | 能力提升 |

### PIP流程
```markdown
## PIP流程

1. 绩效问题确认
   - 问题表现
   - 期望标准

2. 原因分析
   - 能力/动机/环境

3. 改进计划
   - 目标
   - 行动
   - 时间

4. 辅导执行
   - 定期review
   - 支持资源

5. 结果评估
   - 达标：继续
   - 不达标：处理
```
```

### 绩效结果应用

```markdown
## 绩效结果应用

### 应用方向
| 应用 | 说明 | 占比 |
|------|------|------|
| 薪酬调整 | 年薪调整 | 30-50% |
| 奖金分配 | 绩效奖金 | 50-70% |
| 晋升 | 晋升 | 重要依据 |
| 培训 | 培训需求 | 发展依据 |
| 优化 | 末尾淘汰 | 5-10% |

### 应用原则
- 强相关：直接关联
- 差异化：拉开差距
- 及时性：尽快兑现
- 透明性：规则公开
```

## 检查清单

## 红牌警告

- 目标不清晰
- 评估不客观
- 面谈走过场
- 改进无跟踪
- 结果不应用
- 分布失控
- 体系不透明
