# HR Resume Screening Skill

筛选简历，评估候选人资质，快速匹配合适人选。

## 何时使用

- 新职位开放招聘
- 批量筛选简历
- 评估候选人匹配度
- 预筛选电话面试
- 优化简历评估流程

## 简历筛选流程

```
1. 理解职位需求 → 2. 建立评估标准 → 3. 筛选简历 → 4. 评估匹配度 → 5. 候选人排序
```

### 2. 建立评估标准

**候选人评估维度：**

| 维度 | 权重 | 评估要点 |
|------|------|---------|
| 硬性条件 | 30% | 学历/经验/技能 |
| 专业能力 | 30% | 岗位相关能力 |
| 发展潜力 | 20% | 学习能力/成长性 |
| 文化匹配 | 20% | 价值观/团队适配 |

**评分标准：**

```markdown
## 评分标准

### 硬性条件评分
| 等级 | 描述 | 分值 |
|------|------|------|
| A | 完全符合要求 | 100 |
| B | 基本符合 | 80 |
| C | 部分不匹配 | 60 |
| D | 明显不符合 | 30 |

### 经验匹配评分
| 等级 | 描述 | 分值 |
|------|------|------|
| A | 经验非常匹配 | 100 |
| B | 经验较匹配 | 80 |
| C | 经验部分相关 | 60 |
| D | 经验不相关 | 30 |
```

### 3. 筛选简历

**快速筛选清单：**

```markdown
## 快速筛选

### 必须通过（硬性）
- [ ] 学历符合要求
- [ ] 工作年限达标
- [ ] 技能要求满足

### 加分项
- [ ] 知名公司经验
- [ ] 相关行业经验
- [ ] 突出的项目成果
- [ ] 证书/培训经历

### 排除项
- [ ] 频繁跳槽（<1年）
- [ ] 长期空白期
- [ ] 薪资期望过高
- [ ] 简历明显夸大
```

## 简历分析要点

### 技术类职位

| 评估点 | 关注内容 |
|--------|---------|
| 技术栈 | 编程语言、框架、工具 |
| 项目经验 | 项目复杂度、角色、技术挑战 |
| 成果 | 性能提升、用户增长、业绩 |
| 开源贡献 | GitHub、Star数、项目 |
| 算法能力 | 竞赛、论文 |

### 产品/运营类职位

| 评估点 | 关注内容 |
|--------|---------|
| 行业经验 | 行业知识、洞察 |
| 成功案例 | 主导项目、成果数据 |
| 技能 | 分析工具、方法论 |
| 学历背景 | 专业、院校 |

### 管理类职位

| 评估点 | 关注内容 |
|--------|---------|
| 管理经验 | 团队规模、管理年限 |
| 业务成果 | 业务增长、团队业绩 |
| 培养人才 | 下属成长、晋升 |
| 领导力 | 团队评价、认可 |

## 简历筛选检查清单

- [ ] 职位需求理解准确
- [ ] 评估标准明确
- [ ] 硬性条件筛选完成
- [ ] 简历信息核实
- [ ] 匹配度评估客观
- [ ] 候选人排序合理
- [ ] 评估记录完整

## 红牌警告

- 硬性条件未核实就推荐
- 忽略简历中的时间空白
- 薪资期望明显不符
- 频繁跳槽未关注
- 评估过于主观
- 候选人信息不一致
- 推荐明显不符合的人选
