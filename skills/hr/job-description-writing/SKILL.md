# HR Job Description Writing Skill

撰写职位描述，清晰定义岗位职责和要求。

## 何时使用

- 创建新职位描述
- 更新现有职位描述
- 撰写招聘 JD
- 职位说明书归档
- 职位体系梳理

## JD 撰写流程

```
1. 了解职位背景 → 2. 收集职位信息 → 3. 撰写 JD → 4. 内部评审 → 5. 发布使用
```

```markdown
- 平级协作：

### 招聘紧迫度
- 原因：
- 时间要求：

```

### 2. 收集职位信息

**信息收集方法：**

| 方法 | 内容 | 适用场景 |
|------|------|---------|
| 访谈上级 | 期望、要求 | 新职位 |
| 访谈在职者 | 实际工作内容 | 完善现有 JD |
| 工作观察 | 日常任务 | 准确性验证 |
| 文档分析 | 历史文档 | 信息补充 |

### 3. 撰写 JD

**JD 标准模板：**

```markdown

2. [核心职责2，主导...]
3. [核心职责3，协同...]

## 具体工作内容

### 日常工作
- [ ] 负责...
- [ ] 执行...

### 项目工作
- [ ] 参与...
```

### JD 撰写技巧

**好 JD 特征：**

```markdown
## JD 撰写技巧

### 职责描述
✓ 好：[具体动词] + [具体内容] + [产出/目标]
   "负责用户增长策略制定，通过A/B测试优化获客漏斗，目标提升月活20%"

✗ 不好：[模糊动词] + [笼统内容]
   "负责用户增长相关工作"

### 任职要求
✓ 好：具体、可衡量
   "3年以上互联网产品经验，有从0到1产品经验优先"

✗ 不好：模糊、主观
   "有相关工作经验，沟通能力强"
```

### 4. 内部评审

**JD 评审清单：**

```markdown
## JD 评审

### 准确性
- [ ] 职责描述与实际工作一致
- [ ] 无遗漏重要工作内容
- [ ] 无夸大或虚高要求

### 清晰性
- [ ] 职责描述具体明确
- [ ] 要求可理解、可评估
- [ ] 无歧义表述

### 吸引力
- [ ] 亮点突出
- [ ] 发展路径清晰
- [ ] 薪资有竞争力

### 合规性
- [ ] 无歧视性内容
- [ ] 无违法要求
- [ ] 性别/年龄表述中性
```

### 5. 发布使用

**JD 适配不同渠道：**

```markdown
## JD 版本适配

### 官网版本（完整专业）
- 完整职责描述
- 详细任职要求
- 完整福利介绍

### 招聘平台版本（简洁吸引）
- 3-5条核心职责
- 关键要求
- 突出亮点

### 内推版本（简洁直接）
- 核心职责
- 硬性要求
- 联系方式
```

## 常见职位 JD 示例

### 技术类 JD 示例

```markdown
# 高级后端工程师

## 职位概述
加入增长技术团队，负责高并发交易系统架构设计与优化，支撑业务日均千万级订单处理。

## 核心职责
1. 设计并实现高可用、高性能后端服务
2. 优化系统性能瓶颈，提升系统稳定性
3. 主导技术方案评审，推动技术债务治理

## 任职要求
### 必要条件
- 本科及以上，计算机相关专业
- 5年以上后端开发经验
- 精通Go/Java，精通微服务架构
- 有高并发系统经验（QPS 1万+）

### 加分条件
- 有电商/支付系统经验
- 有分布式系统中间件开发经验
- 开源项目贡献者
```

### 产品类 JD 示例

```markdown
# 产品经理

## 职位概述
负责用户增长方向产品设计，通过数据驱动和用户洞察，提升用户获取和留存。

## 核心职责
1. 制定用户增长策略，优化获客漏斗
2. 设计并推动增长实验落地
3. 跨部门协调，推动增长项目实施

## 任职要求
### 必要条件
- 本科及以上
- 3年以上互联网产品经验
- 数据分析能力强，熟练使用SQL
- 有增长产品经验

### 加分条件
- 有从0到1产品经验
- 熟悉增长黑客方法论
- 有A/B测试实践经验
```

## JD 检查清单

- [ ] 职位背景清晰
- [ ] 职责具体可执行
- [ ] 要求可衡量评估
- [ ] 发展路径明确
- [ ] 薪资范围合理
- [ ] 无歧视性内容
- [ ] 内部评审通过

## 红牌警告

- 职责描述过于笼统
- 要求过高或过低
- 与实际工作不符
- 存在歧视性内容
- 薪资范围不合理
- 复制竞品 JD 不做本地化
- 职位名称不准确
