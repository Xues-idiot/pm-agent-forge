# Team Building

团队建设与文化。

## 何时使用

- 团队组建
- 团队融合
- 团队文化
- 团队协作
- 团队诊断

## 团队组建

### 团队设计

```markdown
## 团队设计

### 团队结构
| 类型 | 特点 | 适用 |
|------|------|------|
| 职能型 | 按职能划分 | 稳定业务 |
| 项目型 | 按项目组织 | 创新业务 |
| 矩阵型 | 双重汇报 | 资源复用 |
| 平台型 | 中台+业务 | 中台战略 |

### 团队规模
| 规模 | 人数 | 协作效率 |
|------|------|----------|
| 小团队 | 3-7人 | 高 |
| 中团队 | 8-12人 | 中 |
| 大团队 | 13-20人 | 低 |
| 超大团队 | 20+人 | 需分层 |

### 角色配置
```markdown
## 团队角色

### 角色类型
| 角色 | 职责 | 占比 |
|------|------|------|
| 核心执行 | 具体执行 | 60% |
| 专家 | 专业深度 | 15% |
| 协调者 | 跨团队协调 | 10% |
| 创新者 | 创新突破 | 10% |
| 领导 | 方向把控 | 5% |
```
```

## 团队协作

### 协作机制

```markdown
## 团队协作

### 协作制度
| 制度 | 内容 | 频率 |
|------|------|------|
| 站会 | 进展同步 | 每日 |
| 周会 | 周度复盘 | 每周 |
| Review | 成果审视 | 迭代末 |
| 复盘会 | 项目复盘 | 项目结束 |

### 协作规范
- 信息同步及时
- 求助不羞耻
- 主动承担责任
- 尊重他人时间
- 闭环反馈
```

### 沟通协作

```markdown
## 沟通协作

### 沟通方式
| 方式 | 适用 | 特点 |
|------|------|------|
| 即时沟通 | 快速讨论 | 效率高 |
| 会议沟通 | 复杂决策 | 充分讨论 |
| 文档沟通 | 信息沉淀 | 正式 |
| 一对一 | 深度沟通 | 信任建立 |

### 跨团队协作
- 明确责任人
- 清晰接口
- 利益分配
- 冲突解决机制
```

### 团队诊断

```markdown
## 团队诊断

### 诊断维度
| 维度 | 问题 | 表现 |
|------|------|------|
| 目标 | 目标不清晰 | 方向混乱 |
| 角色 | 职责不清 | 推诿/重叠 |
| 流程 | 流程混乱 | 效率低 |
| 关系 | 关系紧张 | 冲突 |
| 资源 | 资源不足 | 抱怨 |
| 领导 | 领导不力 | 执行差 |

### 诊断工具

- 访谈
- 观察
- 数据分析
```

## 检查清单

- [ ] 团队结构合理
- [ ] 人员配置互补
- [ ] 文化塑造有效
- [ ] 融合阶段顺利
- [ ] 协作机制健全
- [ ] 效能持续提升
- [ ] 问题及时解决

## 红牌警告

- 团队结构混乱
- 人员配置不合理
- 文化形同虚设
- 融合阶段停滞
- 协作障碍
- 效能低下
- 问题长期积累
