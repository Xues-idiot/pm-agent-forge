# Organizational Design Skill

设计组织架构，规划团队配置，优化组织效能。

## 何时使用

- 组织架构设计
- 团队规划
- 岗位职责优化
- 汇报关系梳理
- 组织效能提升
- 变革管理

## 组织设计流程

```
1. 现状分析 → 2. 架构设计 → 3. 职责划分 → 4. 人员规划 → 5. 落地实施
```

**组织架构类型：**

### 常见类型
| 类型 | 特点 | 适用场景 | 优缺点 |

### 3. 职责划分

**部门职责设计：**

```markdown
## 部门职责

### 职责矩阵
| 部门 | 核心职责 | 主要产出 | 关键指标 |
|------|----------|----------|----------|
| 产品部 | 产品规划/设计 | PRD/原型 | 产品交付率 |
| 技术部 | 技术实现 | 系统/功能 | 系统稳定性 |
| 运营部 | 用户/内容运营 | 活跃/留存 | DAU/留存 |
| 市场部 | 品牌/获客 | 线索/CAC | 线索量/CAC |
| 销售部 | 客户转化 | 收入 | GMV/回款 |
| 客服部 | 客户服务 | 满意度 | 响应/解决率 |
| 财务部 | 财务管理 | 报表/合规 | 准确/及时 |
| HR部 | 人力资源 | 人才/文化 | 人效/流失率 |

### 职责边界
```markdown
## 职责边界

### 产品部 vs 技术部
- 产品部：做什么（需求）
- 技术部：怎么做（实现）
- 决策点：技术选型/架构

### 运营部 vs 市场部
- 运营部：存量用户运营
- 市场部：增量用户获取
- 交叉点：品牌曝光/活动配合
```
```

## 组织设计检查清单

- [ ] 业务战略对齐
- [ ] 架构类型合适
- [ ] 职责划分清晰
- [ ] 人员规划合理
- [ ] 汇报关系顺畅
- [ ] 实施计划可行
- [ ] 变革管理到位
- [ ] 效果跟踪机制

## 红牌警告

- 组织架构与战略脱节
- 职责重叠或空白
- 管理幅度不合理
- 核心人才流失
- 变革推进过快
- 沟通不充分
- 阻力不化解
- 效果不跟踪
