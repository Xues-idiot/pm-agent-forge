# Brainstorming Workflow

头脑风暴阶段，明确问题和方向。

## 何时使用

- 收到新的需求或问题
- 产品规划初期
- 探索新方向
- 解决现有问题

## 流程

```
1. 问题澄清 → 2. 现状分析 → 3. 目标设定 → 4. 方案发散 → 5. 方案收敛
```

### 1. 问题澄清

**澄清问题：**
- 真正要解决的问题是什么？
- 问题的背景是什么？
- 谁遇到了这个问题？
- 问题的严重程度？

**澄清模板：**
```markdown
## 问题澄清

### 核心问题
[用一句话描述问题]

### 问题背景
- 时间：
- 地点：
- 涉及人员：
- 触发事件：

### 相关方
- 受益者：
- 受损者：
- 决策者：
- 执行者：

### 问题边界
- 包含：
- 不包含：
```

### 2. 现状分析

**现状了解：**
- 当前是怎么做的？
- 现有的解决方案是什么？
- 数据表现如何？

**分析工具：**
- 5W1H 分析
- 鱼骨图
- 用户旅程图

### 3. 目标设定

**SMART 目标：**

| 要素 | 说明 | 示例 |
|------|------|------|
| Specific | 具体 | 提升首单转化率 |
| Measurable | 可衡量 | 从 5% 提升到 8% |
| Achievable | 可实现 | 基于历史数据 |
| Relevant | 相关 | 业务目标一致 |
| Time-bound | 有时限 | 3 个月内 |

**目标模板：**
```markdown
## 目标设定

### 核心目标
[一句话描述]

### 分解目标
- 目标1：[具体指标]
- 目标2：[具体指标]

### 成功标准
- [ ] 标准1
- [ ] 标准2
```

### 4. 方案发散

**发散思维：**
- 不要批判，先收集所有想法
- 鼓励大胆创新的想法
- 参考竞品怎么做
- 参考其他行业怎么做

**发散工具：**
```markdown
## 方案发散

### 用户需求角度
- 需求1：
- 需求2：
- 需求3：

### 技术角度
- 方案1：
- 方案2：
- 方案3：

### 业务角度
- 方案1：
- 方案2：
- 方案3：
```

### 5. 方案收敛

**收敛标准：**

| 标准 | 说明 |
|------|------|
| 可行性 | 技术能实现吗？ |
| 价值 | 用户真的需要吗？ |
| 成本 | 开发成本高吗？ |
| 风险 | 风险可控吗？ |

**收敛工具：**
- 二八法则
- MVP 思维
- ROI 评估

## 输出物

```markdown
# 头脑风暴结果

## 问题定义
[清晰的问题描述]

## 目标
- 核心目标：
- 衡量标准：

## 初步方案
### 方案A
- 描述：
- 优点：
- 缺点：
- 成本估算：

### 方案B
...

## 下一步
- [ ] 方案A 深入调研
- [ ] 方案B 深入调研
- [ ] 召集评审
```

## 红牌警告

- 问题定义模糊就开始讨论
- 过早收敛到一个方案
- 忽略数据支撑
- 假设未经验证
- 没有考虑约束条件
