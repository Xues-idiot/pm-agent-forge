# Evolving Workflow

进化阶段，持续改进和自我优化。

## 何时使用

- 项目/任务完成后复盘
- 定期自我检查
- 用户反馈积累后优化
- 发现更好的方法

## 进化循环

```
1. 自我评估 → 2. 学习新知识 → 3. 制定改进计划 → 4. 实施改进 → 5. 验证效果 → 6. 记录进化
```

### 1. 自我评估

**评估维度：**

| 维度 | 问题 |
|------|------|
| 效率 | 完成速度是否够快？ |
| 质量 | 产出是否够好？ |
| 用户 | 用户反馈如何？ |
| 成长 | 有学到新东西吗？ |

**自我评估模板：**
```markdown
## 自我评估

### 本次表现
- **做得好的**：
  - 点1
  - 点2

- **需要改进的**：
  - 点1
  - 点2

### 用户/相关方反馈
- 正面反馈：
- 负面反馈：

### 数据表现
| 指标 | 目标 | 实际 | 差距 |
|------|------|------|------|
| | | | |
```

### 3. 制定改进计划

**改进优先级：**

| 优先级 | 定义 | 示例 |
|--------|------|------|
| P0 | 必须改，影响核心 | 流程导致效率低 |
| P1 | 应该改，提升效率 | 工具可以更好 |
| P2 | 可以改，有条件 | 有资源再做 |

**改进计划模板：**
```markdown
## 改进计划

### P0 优先级
| 改进项 | 当前问题 | 改进方案 | 预期效果 | 负责人 | 时间 |
|--------|---------|---------|---------|--------|------|
| | | | | | |

### P1 优先级
| 改进项 | 当前问题 | 改进方案 | 预期效果 | 负责人 | 时间 |
|--------|---------|---------|---------|--------|------|
```

### 4. 实施改进

**改进原则：**
- 小步快跑
- 快速验证
- 持续迭代

**改进执行：**
```markdown
## 改进执行

### 改进1：[名称]
- 开始时间：
- 结束时间：
- 改动范围：
- 验证方式：

### 改进2：[名称]
...
```

### 5. 验证效果

**验证方式：**

| 改进类型 | 验证方式 |
|---------|---------|
| 流程改进 | 效率提升数据 |
| 质量改进 | 缺陷率下降 |
| 用户体验 | 用户反馈 |
| 性能优化 | 性能指标 |

**验证模板：**
```markdown
## 效果验证

### 改进1
- 预期效果：
- 实际效果：
- 达成情况：[ ]达成 [ ]未达成

### 改进2
...
```

### 2x2 复盘矩阵

```
                    做得好的
                       ↑
                       │   ★ 保持推广      ★ 继续加强
                       │   （成功+保持）   （成功+强化）
                       │
                       │
  ─────────────────────┼──────────────────────→
                       │   ○ 开始停止       ○ 快速改进
                       │   （失败+停止）    （失败+改进）
                       │
                       └──────────────────────→
                                            做得不好的
```

## Agent 进化检查

对于 Agent 本身的进化：
- [ ] 哪些 Skill 使用频率最低？考虑优化或删除
- [ ] 哪些 Skill 反馈最多？持续优化
- [ ] 有新的场景需要新 Skill？
- [ ] 现有 Skill 是否需要更新？
- [ ] 工作流是否需要调整？

## 红牌警告

- 改动太大，超过 200 行
- 涉及多个模块/功能
- 没有测试验证
- 改动没有记录
- 改进不可持续
- 忽略用户反馈
