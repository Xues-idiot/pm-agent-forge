# Executing Workflow

执行阶段，将计划转化为实际产出。

## 何时使用

- 项目计划确定后
- 开始开发/运营任务
- 执行既定方案

## 流程

```
1. 启动 → 2. 执行 → 3. 监控 → 4. 调整
```

### 1. 启动

**启动检查：**
```markdown
## 启动检查

### 需求确认
- [ ] PRD 已评审
- [ ] 需求理解一致
- [ ] 疑问已澄清

### 资源确认
- [ ] 人员已到位
- [ ] 环境已准备
- [ ] 工具已就绪

### 沟通确认
- [ ] 沟通渠道建立
- [ ] 会议节奏确定
- [ ] 汇报机制明确
```

### 2. 执行

**执行原则：**

| 原则 | 说明 |
|------|------|
| 小步快跑 | 分阶段交付 |
| 持续集成 | 频繁合并代码 |
| 随时沟通 | 遇到问题立即同步 |
| 文档同步 | 决策和进展记录 |

**每日站会：**
```markdown
## 每日站会

### 格式
1. 昨天完成了什么？
2. 今天计划做什么？
3. 遇到什么阻碍？

### 常见问题处理
| 问题 | 处理方式 |
|------|---------|
| 阻塞 | 立即升级 |
| 风险 | 记录并跟踪 |
| 变更 | 评估影响 |
```

**进度同步：**
```markdown
## 进度同步

### 周报模板
## Week X：[日期]

### 进度
| 任务 | 计划 | 实际 | 状态 |
|------|------|------|------|
| 任务1 | 100% | 100% | 完成 |
| 任务2 | 50% | 40% | 延期 |

### 问题
- 问题1：描述 → 解决方案
- 问题2：

### 下周计划
- 计划1
- 计划2
```

### 3. 监控

**监控指标：**

| 类型 | 指标 | 监控方式 |
|------|------|---------|
| 进度 | 完成度、里程碑偏差 | 项目管理系统 |
| 质量 | Bug数、代码规范 | CI/CD 报告 |
| 风险 | 新风险、风险变化 | 周报 |
| 资源 | 人力投入、预算消耗 | 财务系统 |

**预警机制：**
```markdown
## 预警机制

### 黄色预警（偏差 10-20%）
- 通知：项目经理
- 处理：分析原因，制定对策

### 红色预警（偏差 >20%）
- 通知：项目经理 + 业务负责人
- 处理：紧急评审，调整计划
```

### 4. 调整

**变更管理：**
```markdown
## 变更申请

### 变更描述
[描述变更内容]

### 变更原因
[为什么要变]

### 变更影响
| 维度 | 影响 |
|------|------|
| 进度 | 增加 X 天 |
| 成本 | 增加 X 万 |
| 质量 | 影响 X |

### 变更方案
[具体调整内容]

### 审批
- [ ] 项目经理
- [ ] 业务负责人
```

## 执行检查清单

- [ ] 每日进度同步
- [ ] 遇到问题及时升级
- [ ] 变更走审批流程
- [ ] 关键决策有记录
- [ ] 交付物符合质量标准
- [ ] 文档同步更新

## 红牌警告

- 遇到问题不沟通
- 进度延迟不报告
- 随意变更范围
- 忽略质量只赶进度
- 缺少关键评审节点
